## 🍅 番茄21：大模型工作原理（通俗版）

### 📖 费曼三句话

1. **大模型像一个"超级搜索+超级组合"引擎**——它不是真的"思考"，而是在它训练过的海量数据里，找到最符合你要求的文字组合。你问"北欧风客厅怎么设计"，它把所有关于北欧风客厅的文本碎片组合成一段通顺的回答。
2. **训练大模型就像教一个天才婴儿**：给它看互联网上几乎所有公开的文字（读书、看文章、看设计论坛），然后让它做"接龙游戏"——根据上文猜下文，猜错了就调整，反复千亿次，直到它学会"像人一样说话"。
3. **大模型的三个局限**：①它不知道"不知道"——胡说八道时也很自信（这叫"幻觉"）；②它没有审美——你觉得好看的设计它不一定懂；③它记不住最近的事——除非你告诉它。**知道它的局限，才能更好地用它**。

### 🛠 刻意练习

1. 给AI一个"陷阱问题"，观察它怎么反应：
   ```
   请推荐一种不存在的装修材料"碳纤维纳米墙布"的优缺点。
   ```
2. 再给它一个明确的问题：
   ```
   请推荐3种常见的客厅地面材料，对比优缺点和价格。
   ```
3. 对比两个回答，总结AI什么时候可靠、什么时候不可靠

- [ ] 知道什么时候该相信AI、什么时候该查证

## 🍅 番茄23：提示词黄金框架——角色 + 任务 + 细节 + 格式

### 📖 费曼三句话

1. **好的提示词 = 给实习生写的任务说明书**——越具体越好。不要说"设计个客厅"，要说"一个20平米、采光一般、三口之家、预算3万的北欧风客厅改造方案"。
2. **黄金四要素缺一不可**：
   - 🎭 **角色**：你是谁？（"你是一个有10年经验的装修设计师"）
   - 📋 **任务**：你要AI做什么？（"请设计一个改造方案"）
   - 🔍 **细节**：关键约束条件（面积/预算/风格/人群）
   - 📤 **输出格式**：怎么呈现（表格/清单/分段/多少字）
3. **最常见的错误是"只给任务不给细节"**——就像跟设计师说"帮我设计个好看的家"，设计师无从下手。细节越多，AI少猜，结果越好。

| B | +面积（80平米老房） | 回答更具体 |

## 🍅 番茄25：装饰行业提示词库建设（下）——预算与材料类

### 📖 费曼三句话

1. **预算编制是AI最擅长的**——只要你给出面积、项目和材料等级，AI能按市场均价快速生成预算表。关键是你告诉它"本地市场价"而非"全国均价"。
2. **材料选配提示词的关键**：给AI"约束条件"比给"想要的"更重要。"环保等级E0级，每平米不超过200元，耐污易清洁"比"我要好材料"精准10倍。
3. **AI预算的"人机协作"模式**：AI出初版 → 你根据本地实际价格调整 → 保存调整后的版本作为模板 → 下次AI在模板基础上微调。循环迭代，预算越来越准。

## 🍅 番茄26：提示词高级技巧——从提示词到可复用SKILL

### 📖 费曼三句话
1. **提示词的终极形态不是"写一次用一次"，而是"写一次用无数次"**——把写好的提示词封装成"SKILL"（技能模块），像App一样随时调用。一个装饰方案提示词SKILL = 角色定义 + 任务模板 + 细节参数 + 输出格式，以后每次只需填参数就能复用。
2. **从提示词到SKILL的三步进化**：①写好提示词（单个场景）→ ②模板化（提取可变参数，如`[空间]`、`[预算]`）→ ③封装为SKILL（命名+版本+触发词，通过Claudian或自定义插件调用）。你积累的SKILL越多，你的AI能力越强。
3. **装饰行业SKILL示例**：`方案设计SKILL`（输入:空间/风格/预算 → 输出:完整方案）、`预算编制SKILL`（输入:项目清单 → 输出:三档报价）、`客户沟通SKILL`（输入:客户问题 → 输出:应答话术）。每个SKILL都是一个封装好的提示词模板。

### 🛠 刻意练习

## 🍅 番茄27：方案生成实战——从客户需求到初步方案

### 📖 费曼三句话

1. **用AI出方案的正确顺序**：先让AI帮你"问问题"而不是直接"给答案"——"我客户有20万预算翻新80平米老房，一对退休夫妇，我应该问他们什么问题才能给出好方案？"AI的问题清单会帮你想到你忽略的细节。
2. **AI方案是"初稿"，不是"终稿"**——告诉AI"输出一个能直接给客户看的版本"，然后你花10分钟做三件事：改掉AI说错的本地信息、加你的个人风格、删掉太空泛的套话。
3. **方案的核心价值不是"设计"，而是"沟通"**——客户看不懂设计图，但看得懂"这里改成这样，你早上起来会更方便"。AI写的专业术语要翻译成客户能听懂的话。
4. **用SKILL加速方案生成**——不需要每次都从零写提示词。调用你封装好的`方案设计SKILL`，填入参数即可出方案。以后积累10个SKILL，覆盖装饰行业90%的常见场景。

## 🍅 番茄28：Hermes多Agent系统入门——概念与架构

### 📖 费曼三句话

1. **AI Agent = "有角色的AI"**——普通AI聊天是"你问它答"，Agent是给AI一个固定角色和任务，让它自主工作。就像给Hermes模型设置一个"你是一个专业装修设计师，你的任务是从客户描述中提取所有需求并结构化输出"的系统提示词，它就变成了一个"需求分析Agent"。
2. **多Agent系统 = "多个专家各司其职"**——不是用一个AI做所有事，而是让多个专业Agent协作：需求Agent接待客户 → 设计Agent出方案 → 预算Agent算报价 → 审查Agent检查冲突。每个Agent只做自己最擅长的事，互相传递结果。比单个AI更稳定、更专业。
3. **在装饰行业的落地方式**：Hermes模型（本地运行，数据安全）+ Ollama管理多个模型实例 + n8n编排Agent协作流程。你不需要写复杂的代码，只需要：①为每个Agent写系统提示词 ②在n8n中把Agent连接起来。三个Agent就能覆盖从获客到方案的核心流程。

### 🧩 MCP工具——让Agent拥有"手脚"

**MCP（Model Context Protocol）** 是AI模型与外部工具之间的通信协议。简单说：之前Agent只能"说话"，有了MCP后Agent可以"做事"——查数据库、算价格、调用其他API。

装饰行业MCP工具示例：
- **价格查询MCP**：Agent自动查询本地建材市场价格数据库
- **面积计算MCP**：Agent根据户型图数据自动计算墙面/地面面积
- **规范检查MCP**：Agent自动检查设计方案是否符合建筑规范

## 🍅 番茄29：n8n工作流自动化——多Agent编排实战

### 📖 费曼三句话

1. **n8n + Hermes = 多Agent的"指挥系统"**——单独用Hermes Agent需要手动复制粘贴输出。n8n把Agent串联起来：客户需求 → n8n自动调用需求Agent → 结果自动传给设计Agent → 方案自动传给预算Agent。全程无人值守。
2. **n8n编排多Agent的核心模式**：一个Trigger节点（触发工作流）→ 多个AI节点（调用不同Ollama模型）→ 条件节点（判断输出质量）→ 输出节点（保存/发消息）。每个AI节点配置一个不同的Agent提示词。
3. **装饰行业最有价值的自动化工作流**：客户填写需求表单 → n8n触发 → 需求Agent结构化信息 → 设计Agent生成方案 → 预算Agent生成三档报价 → 汇总成一份《方案建议书》→ 自动保存到项目文件夹 → 通知你审核。10分钟完成以前3小时的工作。

### 🛠 刻意练习（25分钟）

1. **在n8n中构建"需求→方案→预算"工作流**：

   **步骤1**：添加 "Manual Trigger"（手动触发）

   **步骤2**：添加 "HTTP Request" 节点 - 调用需求分析Agent：
   ```
   URL: http://localhost:11434/api/generate
   Method: POST
   Body (JSON):
   {
     "model": "demand-agent",
     "prompt": "{{$json.input}}",
     "stream": false,
     "options": { "temperature": 0.3 }
   }
   ```
   这个节点用你刚创建的demand-agent处理输入。

   **步骤3**：添加第二个 "HTTP Request" 节点 - 调用设计Agent：
   ```
   URL: http://localhost:11434/api/generate
   Method: POST
   Body (JSON):
   {
     "model": "design-agent",
     "prompt": "根据以下需求分析结果，生成初步设计方案：\n{{$json.response}}",
     "stream": false,
     "options": { "temperature": 0.5 }
   }
   ```
   注意：`$json.response` 引用了上一个节点的输出。

   **步骤4**：添加 "Code" 节点 - 合并结果：
   ```javascript
   const demandOutput = $input.all()[0].json.response;
   const designOutput = $input.all()[1].json.response;
   ```
