## 🍅 番茄1：Python环境搭建

### 📖 费曼三句话

1. **Python = AI世界的"普通话"**——几乎所有AI工具、本地模型、自动化脚本都用Python写。你不是要成为Python程序员，而是要学会"用Python运行AI工具"。就像会开汽车不需要会造发动机，但至少要会点火挂挡。
2. **Python环境搭建就三步**：①下载Python安装包 → ②安装（勾选"Add to PATH"）→ ③验证（打开终端输入`python --version`看到版本号）。全程不超过10分钟。
3. **虚拟环境是你的"项目隔离舱"**——每个AI项目有自己独立的Python包，互不干扰。就像每个装修项目有独立的材料清单。创建命令：`python -m venv venv`，激活后安装的所有包只在这个项目里生效。

### 🛠 刻意练习（25分钟）

1. **下载安装Python**（3.12+ 最新稳定版）
   - 官网：python.org → Downloads
   - 安装时**务必勾选** "Add Python to PATH"
   - Win用户注意勾选 "Install launcher for all users"

2. **验证安装**：打开终端（CMD/PowerShell/Terminal），执行：
   ```bash
   python --version    # 应该看到 Python 3.12.x
   pip --version       # Python包管理器
   ```

3. **创建第一个虚拟环境**：
   ```bash
   mkdir my-ai-project
   cd my-ai-project
   python -m venv venv
   ```
   - Windows激活：`venv\Scripts\activate`
   - Mac/Linux激活：`source venv/bin/activate`
   - 看到终端前面出现 `(venv)` 说明激活成功

4. **安装第一个Python包**（在虚拟环境中）：
   ```bash
   pip install requests    # 最常用的HTTP请求库
   ```

## 🍅 番茄2：Node.js与npm生态入门

### 📖 费曼三句话

1. **Node.js让JavaScript能跑在电脑上**——原来JS只能在浏览器里运行，Node.js把它解放出来，可以用来写自动化脚本、搭建Web服务、运行开发工具。很多AI前端工具（包括Remotion视频渲染）都依赖Node.js。
2. **npm是"JavaScript的应用商店"**——全世界开发者写了300万个Node.js包（工具库），用`npm install 包名`就能安装。比如装个`npm install express`就获得了搭建Web服务器的能力。
3. **推荐用fnm（Fast Node Manager）管理Node版本**——不同项目需要不同Node版本。fnm让你一键切换：`fnm use 18`（用18版本）、`fnm use 20`（用20版本）。比直接安装Node更灵活。

### 🛠 刻意练习

1. **安装fnm（Node版本管理器）**：
   - Win（用winget）：`winget install Schniz.fnm`
   - Mac：`brew install fnm`
   - 安装后重启终端

2. **安装Node.js LTS版本**：
   ```bash
   fnm install --lts
   fnm use lts-latest
   fnm default lts-latest
   ```

3. **验证安装**：
   ```bash
   node --version    # 应该看到 v20.x.x 或 v22.x.x
   npm --version     # 应该看到 10.x.x
   ```

4. **第一个npm练习**：创建一个项目并安装包
   ```bash
   mkdir my-node-project
   cd my-node-project
   npm init -y       # 创建 package.json
   npm install axios # 安装HTTP请求库（和Python的requests类似）
   ```

### ✅ 完成标准

- [ ] fnm安装成功，能切换Node版本
- [ ] Node.js LTS版本运行正常
- [ ] npm可用，成功创建了一个项目并安装了包

## 🍅 番茄3：Git版本控制与协作基础

### 📖 费曼三句话

1. **Git = 你的代码"时光机"**——每次修改代码都可以保存一个"快照"（commit），出错了可以随时回到之前任何一个版本。就像装修过程中每个阶段拍照留底，拆错了还能看之前的样子。
2. **Git的核心工作流就三个命令**：`git add`（把修改放进暂存区）、`git commit`（保存一个版本快照）、`git push`（把本地版本同步到云端GitHub）。记住这三个就能应付90%的场景。
3. **GitHub = "代码的微信朋友圈"**——把你的代码发布到GitHub，别人可以看到、下载、甚至帮你改进。对于装饰行业AI教程，GitHub是获取开源AI工具的第一渠道。

### 🛠 刻意练习

1. **安装Git**：
   - 官网：git-scm.com → Downloads
   - Win：下载exe安装，全部默认选项
   - Mac：`brew install git`
   - 验证：`git --version`

2. **配置你的身份**（让Git知道是谁在提交）：
   ```bash
   git config --global user.name "你的名字"
   git config --global user.email "你的邮箱"
   ```

3. **注册GitHub账号**（github.com）并配置SSH Key：
   ```bash
   # 生成SSH Key（一路回车）
   ssh-keygen -t ed25519 -C "你的邮箱"
   # 查看公钥
   cat ~/.ssh/id_ed25519.pub
   ```
   复制输出的内容 → GitHub网站 → Settings → SSH and GPG keys → New SSH key → 粘贴保存

4. **第一个Git练习**：
   ```bash
   # 在番茄2创建的项目中初始化Git
   cd my-node-project
   git init
   git add .
   git commit -m "第一次提交：初始化Node项目"
   ```

- [ ] Git安装成功，`git --version`正常输出
- [ ] 配置了姓名和邮箱
- [ ] 注册了GitHub账号，配置了SSH Key

## 🍅 番茄4：Claude API与CCSwitch模型路由

### 📖 费曼三句话

1. **API = "让程序帮你调用AI"**——你在网页上和ChatGPT对话，那是手动模式。用API，你的代码可以直接调用AI，实现批量处理、自动回复、工作流集成。就像从"手工单件生产"升级到"自动化流水线"。
2. **CCSwitch（Claude Code Switch） = "AI模型的智能路由器"**——你不需要在多个AI平台之间手动切换。CCSwitch统一管理多个模型（Claude/DeepSeek/GPT/本地模型），根据任务类型自动选择最合适、最经济的模型。写方案用它，写代码用它，画图用它——一次配置，到处调用。
3. **装饰行业的典型API调用流**：客户需求 → 自动调用大模型生成方案 → 调用绘图API生成效果图 → 自动保存到项目文件夹。全程不需要打开任何网页，一键完成。

### 🛠 刻意练习

1. **获取Claude API Key**：
   - 访问 console.anthropic.com → 注册/登录
   - 在API Keys页面 → Create Key → 复制并保存好（**离开页面后就看不到了**）
   - ⚠️ 把API Key保存在环境变量中，不要写在代码里

2. **配置CCSwitch**（假设备份已安装）：
   ```bash
   # 安装CCSwitch
   npm install -g ccswitch
   
   # 配置模型（编辑配置文件）
   ccswitch config add claude --key "你的API Key" --model claude-sonnet-4-20250514
   ccswitch config add deepseek --key "你的Key" --model deepseek-chat
   ```

3. **测试调用**：
   ```bash
   # 用CCSwitch调用Claude
   ccswitch chat "请用一句话解释什么是装修中的'隐蔽工程'"
   
   # 切换模型
   ccswitch use deepseek
   ccswitch chat "请用一句话解释什么是装修中的'隐蔽工程'"
   ```

### ✅ 完成标准

- [ ] 获取了至少一个AI模型的API Key（Claude/DeepSeek）
- [ ] 配置了CCSwitch并添加了至少一个模型
- [ ] 通过CCSwitch成功调用了AI模型
- [ ] API Key已保存在环境变量中（未写在代码里）

## 🍅 番茄5：本地模型部署——Hermes + DeepSeek（Ollama）

### 📖 费曼三句话

1. **本地模型 = "在你的电脑上运行的AI"**——不需要联网、不需要付费、数据不出门。Ollama让你像安装App一样安装AI模型：`ollama run hermes3` 或 `ollama run deepseek-r1`。等几分钟下载完，就可以在终端里和AI对话了。
2. **两条本地模型路线**：**Hermes系列**（Nous Research）擅长推理和创意写作，装饰行业方案设计首选；**DeepSeek系列**（深度求索）在中文理解、数学推理和代码生成上极为出色，适合预算计算、数据分析。**两者互补而非替代**，装饰公司建议两个都装。
3. **本地模型的核心价值不是"替代云端"，而是"私有化+离线化"**——客户的装修方案中包含户型图、预算、个人信息，不适合上传到公共AI平台。用本地模型处理敏感数据，用云端模型处理创意内容，各有分工。

### 🛠 刻意练习

## 🍅 番茄6：模型管理与硬件配置

### 📖 费曼三句话

1. **不同模型适合不同任务**——Hermes适合通用对话和创意写作、DeepSeek中文理解最好且数学推理极强、Qwen2.5（通义千问）中文能力也很出色、CodeLlama专攻代码。Ollama里已经预置了200+模型，`ollama list`查看已下载的，`ollama pull 模型名`下载新的。
2. **模型的三个关键参数**：温度（temperature, 0-1，越高越有创意越低越保守）、上下文长度（context length, 越大能记住越多对话历史）、量化级别（Q4/Q5/Q8，越低体积越小但精度略降）。装饰行业建议：temperature=0.3（方案要可靠）、context=8192（足够记住一个完整的客户需求）。
3. **DeepSeek部署的硬件门槛比想象中低**——DeepSeek-R1蒸馏版7B只需要8GB显存/16GB内存就能流畅运行，一块RTX 3060（¥2000）或M系列Mac都够用。如果预算充足上14B/33B版本（24GB显存），效果接近GPT-4。本地部署的月成本只有电费（约¥30-80/月），相比云端API订阅，一年能省¥2000+。

### 🛠 刻意练习

1. **深度了解DeepSeek系列**（与Hermes互补）：
   ```bash

   ollama pull deepseek-r1:7b
   
   # DeepSeek-R1 蒸馏14B（推荐进阶，16GB显存）
   ollama pull deepseek-r1:14b
   
   # DeepSeek-V3（671B，需要云端，了解即可）
   ```

2. **硬件配置评估**——对照你的电脑配置，选择适合的模型：

   | 配置等级 | 显存 | 内存 | 推荐模型 | 硬件成本 | 月电费 |
   |:--------|:----:|:----:|:---------|:--------:|:------:|
   | 💻 办公本 | 共享 | 16GB | Hermes 8B / DeepSeek 7B | ¥0（已有） | ¥30 |
   | 🖥 入门独显 | 8GB | 16GB | Hermes 8B / DeepSeek 7B/14B | ¥2000（RTX 3060） | ¥50 |
   | 💪 中端独显 | 12GB | 32GB | DeepSeek 14B / Hermes 12B | ¥3500（RTX 4070） | ¥60 |
   | 🚀 高端独显 | 24GB | 32GB | DeepSeek 33B / Hermes 70B | ¥8000（RTX 4090） | ¥80 |
   | ☁️ 纯云端 | — | — | 全部模型（通过API） | ¥0-200/月 | — |

3. **常见问题排查**：
   - 显存不足 → 用更小的量化版本（如 `deepseek-r1:7b-q4` 比默认小30%）
   - 内存不足 → 关闭其他程序，设置Ollama并发数为1
   - 速度慢 → 降低上下文长度（`--num-ctx 4096`）
   - Mac用户 → M系列芯片统一内存，等效显存=总内存÷2

4. **Benchmark测试**——用同一个问题测试不同模型的性能：
   ```bash

   瓷砖单价80元/平米，水泥砂浆辅料10元/平米，请计算总费用"
   
   # 对比Hermes的回答，哪个算得更准？
   ```

## 🍅 番茄7：n8n工作流引擎搭建

### 📖 费曼三句话

1. **n8n = "AI自动化流水线"**——它是一个开源的工作流引擎，让你用可视化的方式把不同工具连接起来。比如：收到小红书评论 → AI自动回复 → 记录到CRM → 通知你。全部自动化，不需要写代码（但支持写代码扩展）。
2. **n8n的核心理念是"节点"**——每个节点做一件事：Trigger（触发，如"收到邮件"）、Action（操作，如"发一条微信消息"）、AI（调用模型，如"用Claude分析客户需求"）。把节点连起来就是一个工作流。
3. **对装饰行业，n8n能自动化的6件事**：①客户咨询自动回复 ②内容多平台分发 ③评论数据采集分析 ④施工进度自动提醒 ⑤客户回访定时触发 ⑥设计方案的批量生成。每个工作流搭建一次，永久使用。

### 🛠 刻意练习

1. **用Docker安装n8n**（Docker是最简单的安装方式）：
   ```bash
   # 先安装Docker Desktop（docker.com → Download）
   # 验证：docker --version
   
   # 运行n8n（一条命令就够了）
   docker run -d --name n8n -p 5678:5678 \
     -v n8n_data:/home/node/.n8n \
     -e N8N_SECURE_COOKIE=false \
     n8nio/n8n
   ```

2. **访问n8n**：打开浏览器 → http://localhost:5678 → 注册账号（本地使用）

3. **创建第一个工作流**（不做复杂操作，先熟悉界面）：
   - 点击 "New Workflow"
   - 左侧节点面板看看有哪些触发器（Trigger）和操作（Action）
   - 添加一个 "Manual Trigger" 节点（手动触发）
   - 添加一个 "AI → OpenAI" 节点（配置API Key）
   - 尝试连接两个节点
   - 点击 "Execute Workflow" 运行

### ✅ 完成标准

- [ ] Docker Desktop安装成功
- [ ] n8n通过Docker成功运行（浏览器能打开n8n界面）
- [ ] 创建了第一个工作流（手动触发→AI节点）

## 🍅 番茄8：Obsidian安装与核心配置

### 📖 费曼三句话

1. **Obsidian = "你的第二大脑"**——它不只是笔记软件，更是一个知识管理系统。所有的笔记都是纯文本Markdown文件（不用担心厂商锁定），通过双向链接把知识连接成网。装修行业的知识积累（案例/材料/供应商/客户偏好）放在Obsidian里，随时检索，越用越值钱。
2. **Obsidian的三大核心能力**：①**双向链接**（`[[笔记名]]` 建立笔记间的关联，一个笔记被引用了会自动显示"反向链接"）②**图谱视图**（可视化看到所有笔记的关联网络，发现"原来这两个知识是有关系的"）③**插件生态**（2000+社区插件，几乎可以扩展成任何工具）。
3. **装饰行业用Obsidian管理四类知识**：客户案例库（每个项目一条笔记，链接风格/材料/供应商）、材料知识库（链接到案例和供应商）、模板库（提示词模板/方案模板/预箅模板）、内容库（已发布的内容和选题池）。

3. **创建第一条笔记**：

## 🍅 番茄9：Obsidian+Claudian知识管理实战

### 📖 费曼三句话

1. **Claudian（Claude AI助手） = 你的"知识库协作者"**——它不是独立的软件，而是在你写笔记时随时可调用的AI助手。能帮你总结文章、提取洞见、生成笔记框架、建立知识关联。你在Obsidian里写笔记遇到"这个知识点我好像以前写过"，问Claudian，它立刻告诉你笔记在哪里。
2. **Obsidian+Claudian = "笔记+AI"的协奏**——Obsidian负责存储和链接（知识的外骨骼），Claudian负责理解和生成（知识的大脑）。一个典型的协作场景：你读完一篇装修行业文章 → 粘贴到Obsidian → 让Claudian提取3个关键洞见 → 自动链接到已有的相关笔记 → 生成双向链接。

3. **知识管理的"黄金三步法"**：
   - **收集** → 看到有用的内容（文章/图片/灵感），立即存进Obsidian，不要分类（"先存后理"）
   - **整理** → 每天花10分钟，给笔记加标签、建链接、写摘要
   - **创造** → 需要用时，通过链接网络找到所有相关内容，产出方案或内容
   - Claudian在这三步中都能提供帮助：收集时自动摘要、整理时建议链接、创造时生成初稿。

## 🍅 番茄10：综合考核——搭建完整开发环境

### 📖 费曼三句话

1. **一个完整的AI开发环境 = Python + Node.js + Git + AI模型 + n8n + Obsidian**——它们就像装修队的工具箱：Python是"电动工具"（处理数据）、Node.js是"精密仪器"（运行前端工具）、Git是"工程日志"（记录所有修改）、本地模型是"现场加工"（离线处理敏感数据）、n8n是"自动化流水线"、Obsidian是"项目知识库"。
2. **环境搭建的"最小可行方案"**：不是每个工具都要完美配置，而是每个工具都能"跑通一个Hello World"。后续用到时再深入配置。今天的目标是"能用"，不是"精通"。
3. **保存你的环境配置信息**：把所有工具的安装路径、API Key（加密保存）、配置文件位置写进Obsidian笔记。下次电脑坏了或换新电脑，照着笔记半小时就能恢复全部环境。
