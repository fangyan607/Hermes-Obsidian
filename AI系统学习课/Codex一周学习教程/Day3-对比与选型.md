## 番茄钟5：同一任务，不同表现（25分钟）

### 5.1 场景测试：搭建一个 REST API

**任务：** 用 FastAPI 创建一个待办事项 API

**Cursor 的表现：**
```
体验：IDE 内操作
- Tab 补全：写 `from fast` 自动补全 import
- Composer：⌘+I → "创建 Todo API，GET/POST/PUT/DELETE"
- 内联 Diff：每行改动都能看到绿色/红色标记
- 终端：内建终端运行 uvicorn
优点：流畅，视觉反馈好，修改方便
缺点：依赖 IDE 环境
```

**Claude Code 的表现：**
```
体验：终端操作
$ claude "用 FastAPI 创建一个 Todo API"
- 先问：用 SQLite 还是 PostgreSQL？有认证吗？
- 规划：创建 main.py, models.py, schemas.py, database.py
- 写代码：逐文件生成
- 解释：每段代码附说明
优点：规划详细，考虑周全，善于处理模糊需求
缺点：速度慢，Opus 回答要等一会儿
```

**Codex CLI 的表现：**
```
体验：终端操作
$ codex -a full-auto "用 FastAPI 创建一个 Todo API"
- 直接开干：检查目录 → 创建文件 → 写代码
- 速度快：几乎瞬间完成
- 沙箱运行：自动尝试运行测试
优点：极快，一次性完成，适合明确需求
缺点：交互少，不主动提问澄清
```

### 5.2 场景测试：重构老旧代码库

**任务：** 把一个 500 行的 `utils.py` 拆成模块

| 工具 | 表现 | 时间 | 评价 |

| **Claude Code** | 先分析依赖关系，规划拆分方案，逐步执行 | 5-10分钟 | ⭐⭐⭐⭐⭐ |

| Codex CLI | 直接拆，速度快但对复杂依赖可能欠考虑 | 1-2分钟 | ⭐⭐⭐ |

### 5.3 场景测试：批量生成测试

| 工具 | 表现 | 时间 | 评价 |

| **Codex CLI** | "开源的高速引擎，批量任务的王者" | CI/CD、测试生成、脚手架、安全场景 |

**核心洞察：**
- 没有一个工具在所有领域最强
- **速度 vs 深度**是 Codex 和 Claude Code 的核心权衡
- Cursor 的壁垒是 **IDE 体验和生态**
- Codex CLI 的壁垒是 **开源和低边际成本**
- Claude Code 的壁垒是 **推理深度和准确率**
