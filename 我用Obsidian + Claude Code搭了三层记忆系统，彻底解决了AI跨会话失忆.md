为什么是Obsidian + Claude Code这个组合？
 
Obsidian负责「存」——
纯本地运行，你的所有知识、脚本、风格文件，
全部以Markdown格式存在你自己的硬盘里。
不上云，不依赖第三方服务，数据永远是你的。
 
Claude Code负责「用」——
它能直接读取你的Obsidian知识库，
不是复制粘贴，而是真正「住进」你的文件夹，
调取你积累的所有内容来工作。
 
两个工具打通之后，我设计了三层记忆分级结构：
 
Layer 1，永久记忆层：
Obsidian里你已有的知识文件夹——
Context（品牌定位+用户画像）、
Script（历史脚本）、Reading（读书笔记）……
这是Claude了解你创作风格的「知识底座」。
 
Layer 2，会话记忆层：
一个专属的Claude_Memory文件夹。
每次对话产生的灵感、思路、决策，
Claude实时记录在这里，不再「聊完就丢」。
会话结束后，有价值的内容沉淀到Layer 1。
 
Layer 3，大脑目录层：
一个叫MEMORY.md的核心文件，放在Vault根目录。
每次启动Claude Code，它先读这个文件——
瞬间定位：你是谁，数据在哪，用什么风格干活。
 
这套系统跑起来之后，
Claude Code不再是每次都要重新认识的新人，
而是一个越用越懂你、风格永远在线的创作合伙人。
 
视频完整演示搭建流程 + 五个实操案例：
从基础测试到知识提炼，从创作闭环到关联检索，
最后是会话结束归档——让每次对话都沉淀进系统。
