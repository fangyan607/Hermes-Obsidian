## 🍅 番茄9：三层记忆架构（25分钟）

### 9.1 用"大脑"类比理解记忆

Hermes 的记忆系统像人类大脑的三层结构：

```
┌─────────────────────────────────────────────────────────────┐
│                    Hermes 三层记忆架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Layer 1: 内置记忆（持久化） — 像你的长期记忆                  │
│  ├── MEMORY.md：项目背景、重要经验（永久保存）                │
│  ├── USER.md：你是谁、喜欢什么、讨厌什么                      │
│  └── SOUL.yaml：AI 的性格、说话风格、行为规则                 │
│                                                             │
│  Layer 2: 外部记忆 Provider（可选）                           │
│  └── Mem0 / Pinecone 等向量数据库                             │
│      └── 适合：百万级知识库、RAG 检索                         │
│                                                             │
│  Layer 3: 会话上下文（临时） — 像你的短期记忆                 │
│  └── 当前对话窗口内的聊天记录                                 │
│      └── 会话结束即消失                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 9.2 核心思维转变

| 传统 AI | Hermes Agent |
|:--------|:-------------|
| 每次对话从零开始 | 每次对话前先"翻阅"你的记忆 |
| 不记得你的名字 | 记得你的背景、项目、偏好 |
| 回答风格千篇一律 | 根据你的 USER.md 调整风格 |
| 无法积累知识 | MEMORY.md 持续积累经验 |

> **一句话**：传统 AI = 金鱼（7秒记忆），Hermes = 大象（永不忘记）

### 9.3 与你的三层记忆系统对比

| 你的 Obsidian 三层记忆 | Hermes 三层记忆 | 映射关系 |
|:----------------------|:---------------|:---------|
| MEMORY.md（路由） | MEMORY.md | ✅ 直接对应 |
| Claude_Memory/ | 会话上下文 | 部分对应 |
| Layer 1（永久知识） | 外部 Provider | 可整合 |

> ✋ **费曼自测**：Hermes 的 MEMORY.md 和你的 Obsidian 知识库有什么相似和不同？它们可以怎样配合？

### 10.3 什么是策展式写入？

想象一个**优秀图书管理员**，不是把所有书都堆进仓库，而是：

```
1. 筛选：哪些书有价值 → 留下
2. 分类：归入正确的书架
3. 去重：同一本书不收两次
4. 清理：旧版本书 → 淘汰
5. 标签：给每本书贴标签方便查找
```

策展式写入 = AI 自动做上述工作，确保 MEMORY.md 不膨胀、不冗余。

**不开启策展式写入的后果：**

| 问题 | 表现 | 后果 |
|:-----|:-----|:-----|
| 记忆膨胀 | MEMORY.md 越来越大 | Token 消耗剧增 |
| 信息冗余 | 重复内容 | AI 回答变得啰嗦 |
| 过时信息 | 几个月前的旧知识 | AI 给出错误建议 |
| 隐私泄露 | 敏感信息进入记忆 | 安全隐患 |

### 10.4 配置策展式写入

```yaml
# config.yaml
memory:
  enabled: true
  max_tokens: 10000
  compression: true         # 压缩旧记忆
  curated_write: true       # 开启策展式写入
  write_threshold: 0.7      # 重要性评分 > 0.7 才写入
```

## 🍅 番茄11：配置 USER.md 和 SOUL.yaml（25分钟）

### 11.1 USER.md 实战模板

USER.md 是告诉 AI "你是谁"的档案卡。

```markdown
# 用户画像

---

## 基本信息

### 身份与角色
- 内容创作者、知识管理爱好者、AI工具探索者
- 使用 Obsidian 构建第二大脑
- 持续学习 Hermes / Claude / Codex 等 AI 工具

### 技术栈
- 系统：Windows（主力） + Linux 云服务器
- 工具：Obsidian、Claude、Python、Git
- 框架：Hermes Agent（学习中）

---

## 沟通偏好

### 风格要求
- **语言**：中文
- **格式**：Markdown
- **结构**：清晰的层级、可执行的清单
- **深度**：有深度、循序渐进

### 输出偏好
- 简洁高效，避免冗余
- 提供具体的实例和代码
- 使用表格进行对比

## 行为禁忌

### ❌ 不要做的事
- 不要过度使用 emoji（偶尔使用没问题）
- 不要输出冗余内容
- 不要在会话中途修改记忆文件
- 不要使用绝对路径操作 Vault 文件

### ✅ 要做的事
- 每次启动先阅读 MEMORY.md
- 使用相对路径引用文件
- 编辑前必须先读取文件
- 提供可执行的操作清单

---

## 特殊指令

### 启动时
1. 读取 MEMORY.md 了解当前状态
2. 读取最新 Claude_Memory 会话
3. 确认意图后开始工作

### 工作时
- 优先从 Layer 1 读取长期上下文
- 临时内容保存到 Claude_Memory/
- 重要成果整理回 Layer 1
```

### 11.2 SOUL.yaml 实战模板

SOUL.yaml 决定 AI 的"性格"——它像谁、怎么说话、怎么思考。

```yaml
# SOUL.yaml - AI人格设定
name: Claudian
role: Obsidian知识助手

personality:
  traits:
    - 专业但不死板
    - 简洁高效
    - 主动思考
    - 结构化输出

communication_style:
  language: 中文
  tone: 专业友好
  format: Markdown
  example_greeting: "你好！我来帮你分析这个问题。"

behavior_rules:
  - 每次启动先阅读 MEMORY.md 和 USER.md
  - 使用相对路径引用 Vault 文件
  - 编辑前必须先读取文件
  - 回答时先给出核心结论

constraints:
  - 不要覆盖用户数据
  - 不要使用绝对路径操作 Vault 文件
  - 不要创建不必要的临时文件
  - 不输出与当前任务无关的内容
```

### 12.5 记忆命令速查

```bash
hermes memory show       # 查看记忆内容
hermes memory status     # 查看记忆状态
hermes memory clear      # 清空记忆
hermes memory export     # 导出记忆
hermes chat --reload-memory  # 重载记忆
```

### 12.6 记忆不生效的排查

| 问题 | 原因 | 解决方案 |
|:-----|:-----|:---------|
| 记忆没更新 | 会话中途修改 | 重启会话 |
| 记忆被覆盖 | 未开启策展式写入 | 开启 curated_write: true |
| 记忆太少 | 写入阈值太高 | 降低 write_threshold |
| 记忆太多 | 未压缩 | 开启 compression: true |

### 💪 刻意练习——记忆系统三轮循环

```
===== 循环 1：入门任务 =====
1. 创建 MEMORY.md，包含你的背景和项目信息
2. 重启 Hermes 验证记忆生效
3. 让 AI 记住一个临时信息，检查是否写入
✅ 验证方式：AI 能回答出 MEMORY.md 中的内容

===== 循环 2：进阶任务 =====
1. 创建 USER.md，定义你的沟通偏好和禁忌
2. 测试 AI 是否遵循你的风格（如不使用 emoji）
3. 故意触碰禁忌，观察 AI 反应
✅ 验证方式：AI 按照 USER.md 调整了沟通风格

===== 循环 3：挑战任务 =====
1. 创建 SOUL.yaml，给 AI 设定独特人格
2. 测试人格是否生效
3. 调整三份文件，观察 AI 行为的整体变化
✅ 验证方式：AI 表现出统一个人风格
```
