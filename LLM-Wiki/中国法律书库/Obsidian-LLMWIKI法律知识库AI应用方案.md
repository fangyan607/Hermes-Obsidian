# Obsidian LLM-WIKI 法律MD知识库AI检索应用方案

**文档版本**：V1.0
**适配环境**：Obsidian + LLM-WIKI
**资源基础**：2502部法律文件（法律法规2416部 + 法律书籍63本 + 年度案例23册）
**核心用途**：法律条文智能检索、案例检索分析、案件研判辅助、合规审查、法律文书生成、法律问答、法律经济学分析、犯罪心理分析

## 一、库内笔记标准化规整（适配LLM-WIKI索引）

### 1. 统一YAML头部元数据

所有法律笔记头部添加元标签，LLM-WIKI可按分类、技能、标签精准检索匹配。

```yaml
---
title: 法律名称
category: 中国法律/行政法规/司法解释/法律学习书籍/年度案例
skill_tag: 法律条文查询,案例检索,案例分析,案件研判,合规审查,法律文书生成,法律问答,法律经济学分析,犯罪心理分析
level: 入门/进阶/专业
keywords: 核心法律关键词,如:民法典,合同,侵权,物权,婚姻
type: law
---
```

| 字段 | 说明 | 可选值 |
|------|------|--------|
| `title` | 法律文件/书籍名称 | 如"中华人民共和国民法典" |
| `category` | 所属分类目录 | 中国法律/行政法规/司法解释/法律学习书籍/年度案例 |
| `skill_tag` | 关联技能标签（可多选） | 法律条文查询,案例检索,案例分析,案件研判,合规审查,法律文书生成,法律问答,法律经济学分析,犯罪心理分析 |
| `level` | 难度层级 | 入门（普法）/ 进阶（实务）/ 专业（研究） |
| `keywords` | 核心关键词（逗号分隔） | 如:民法典,合同,侵权责任,物权,婚姻家庭 |
| `type` | 书库类型标识 | `law`（固定值） |

### 2. 规范文件夹目录结构

按法律体系逻辑划分目录，方便LLM-WIKI定向扫描检索。

```

### 3. 内容优化适配检索

1. **知识点拆分**：法律文件按编、章、节拆分独立笔记；案例按裁判要旨、争议焦点、裁判观点、法官后语拆分模块
2. **内部链接关联**：法律条文关联对应司法解释、典型案例；案例分析关联对应法律条文；书籍内容关联法律条文
3. **版本时效性标注**：文件名包含版本日期（如`{法律名称}_{版本日期}`），YAML元数据标注生效/修订/废止状态
4. **术语规范统一**：统一法律术语表述，删减冗余内容，保证语义标准一致

## 二、LLM-WIKI检索运行架构

整体流程：Obsidian法律MD笔记 → 框架索引扫描 → 向量缓存入库 → 目录/标签路由筛选 → 调用绑定SKILL技能 → 生成法律条文、案例分析、研判结论应答

### 1. LLM-WIKI基础索引配置

1. 指定**中国法律书库**总文件夹为专属检索数据源，划定扫描范围（5个分类目录）
2. 开启元数据、分类、技能标签联合索引，优先按 `skill_tag` 触发对应功能
3. 调整嵌入匹配阈值，兼顾检索速度与法律条文精准度（建议较高阈值确保法条准确性）
4. 开启笔记溯源功能，回答可跳转查看原始MD笔记原文

### 2. 四类检索匹配规则

| 检索类型 | 匹配策略 | 适用场景 |
|----------|----------|----------|
| **目录定向检索** | 按5个分类文件夹锁定领域 | 快速定位某部法律或某类法规 |
| **标签技能检索** | 通过 `skill_tag` 识别用户意图，自动匹配对应SKILL | 用户触发技能指令时自动路由 |
| **语义智能检索** | 理解自然语言法律问题，跨笔记整合检索结果 | 模糊提问、复杂法律咨询 |
| **层级检索区分** | 按 `level` 标签区分入门/进阶/专业内容输出 | 区分普法问答与专业法律分析 |

## 四、四大实际使用模式

### 模式1：法律条文智能检索

**流程**：用户自然语言提问 → LLM-WIKI匹配目录标签 → 调取中国法律/行政法规/司法解释笔记内容 → 整合条文原文+释义+关联法规+时效性标注 → 输出结果（支持溯源原文）

**典型场景**：
- 查询某部法律的具体条文内容
- 查找某个法律问题的法条依据
- 确认某部法规的现行有效性
- 对比新旧法条差异

**示例**：
```
用户：民间借贷的利息上限是多少？
AI：调用 Skill1 法律条文查询，检索民法典第680条、最高人民法院关于审理民间借贷案件适用法律若干问题的规定，输出：利息上限为LPR的4倍（当前约13.8%），超出部分不予支持，相关条文原文及司法解释要点。
```

### 模式2：案例分析与研判

**流程**：描述案情 → LLM-WIKI识别意图 → 调用Skill2检索相似案例 + Skill3深度分析 + Skill4法律适用研判 → 输出案例参考+裁判预判+风险提示

**典型场景**：
- 查找类似案例参考裁判结果
- 分析某个案例的法律适用
- 预判案件可能的裁判结果
- 评估案件风险点

**示例**：
```
用户：分析一起民间借贷纠纷，甲借乙10万元，约定年利率36%，乙已还款8万元，甲起诉要求还款本息共15万元
AI：调用 Skill3 案例分析 + Skill4 案件研判，提取案情要素，匹配相似案例，分析利率超过法定上限（LPR的4倍），超出部分无效，重新计算合法本息，预判甲的诉讼请求部分支持，输出风险提示和办案建议。
```

### 模式3：合规审查与风险识别

**流程**：用户提交合同/条款 → LLM-WIKI识别合规审查意图 → 调用Skill5合规审查 → 逐条筛查违规条款 → 输出审查意见+修改建议+法律依据

**典型场景**：
- 审查合同条款的合法性
- 识别企业经营中的法律风险
- 筛查劳动合同合规性
- 数据合规审查

**示例**：
```
用户：审查这份劳动合同中的竞业限制条款是否合规
AI：调用 Skill5 合规审查 + Skill1 法律条文查询，检索劳动合同法第23-24条关于竞业限制的规定，筛查条款中竞业范围、期限（不应超过2年）、补偿金标准等要素，标注不合规项，给出修改建议和法律依据。
```

### 模式4：法律文书生成

**流程**：用户描述案件事实 → LLM-WIKI识别文书生成意图 → 调用Skill6法律文书生成 → 调取法律条文作为依据 → 生成规范法律文书

**典型场景**：
- 生成民事起诉状/答辩状
- 生成刑事辩护词
- 生成法律意见书/合规意见书

**示例**：
```
用户：生成一份民间借贷纠纷起诉状，原告张三，被告李四，借款10万元，约定年利率12%，到期未还
AI：调用 Skill6 法律文书生成，依据民事诉讼法、民法典相关规定，生成起诉状，包含当事人信息、诉讼请求（本金+合法利息）、事实与理由、法律依据，并提供证据清单建议。
```

## 五、LLM-WIKI专用提示词模板

### 通用法律知识检索提示词

```
仅依据Obsidian LLM-WIKI内法律MD笔记内容作答，不编造法律条文与案例。
1. 识别问题匹配对应分类与skill标签，调取库内法律知识点
2. 法律条文引用需标注法律名称、条文编号、版本日期
3. 案例引用需注明案例类型、裁判要旨、来源分册
4. 复杂法律问题分层说明，实务操作标注清晰步骤
5. 重要内容关联对应笔记名称，方便溯源查阅
6. 法律法规时效性需明确标注（有效/废止/修订状态）
```

### 法律条文查询提示词

```
启用法律条文查询SKILL，调取中国法律、行政法规、司法解释相关笔记。
根据用户查询内容精准检索法条，
输出条文原文、释义、适用条件、关联法规链接，
同时标注版本日期和时效性状态（现行有效/已修订/已废止）。
```

### 案例分析与研判提示词

```
启用案例分析与案件研判SKILL，调取年度案例库及相关法律笔记。
根据用户描述的案情事实：
1. 抽取案情核心要素（主体、行为、后果、争议焦点）
2. 检索匹配相似案例，输出裁判要旨、争议焦点、裁判观点
3. 分析法律适用，判定适用法律门类、具体法条、司法解释
4. 预判裁判结果（责任划分/赔偿区间/量刑范围）
5. 提示案件风险点和注意事项
```

### 合规审查与文书生成提示词

```
启用合规审查或法律文书生成SKILL。
- 合规审查：逐条筛查违规条款，标注不合规项，给出修改建议和法律依据
- 文书生成：依据法律条文和案件事实，生成规范法律文书
- 所有依据必须引用具体法律名称、条文编号、版本日期
- 文书格式参照法律实务规范要求
```

### 法律问答与普法咨询提示词

```
启用法律问答SKILL，依据全库法律笔记内容作答。
1. 口语化提问用通俗易懂的语言解答
2. 专业问题保持严谨规范的法律表述
3. 涉及具体法条需准确引用条文编号
4. 结合案例解释法律知识点（以案释法）
5. 区分普法科普与专业法律意见的边界
```

### 法律经济学分析提示词

```
启用法律经济学分析SKILL，调取法律经济学相关笔记。
运用经济学理论分析法律制度：
1. 成本-收益分析：评估法律规则的经济效率
2. 激励效应分析：分析法律规则对行为的影响
3. 制度效率比较：比较不同法律规则的效率
4. 适用经典分析框架（汉德公式、科斯定理等）
```

### 犯罪心理分析提示词

```
启用犯罪心理分析SKILL，调取犯罪心理学相关笔记。
根据案件信息进行犯罪心理分析：
1. 作案手法（MO）分析：接近、控制、逃离方式
2. 签名特征识别：心理满足行为模式
3. 犯罪动机推断：利益、报复、性欲、控制等
4. 被害人分析：风险因素、选择原因
5. 心理画像构建：嫌疑人特征推断

## 六、知识库迭代优化方案

### 1. 笔记增量更新
- 新增法律法规文件归入对应分类目录（中国法律/行政法规/司法解释）
- 新增法律书籍归入法律相关目录
