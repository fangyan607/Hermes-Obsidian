# 复习试卷：19-不同领域的AI算法

## Section A：费曼学习法复习文档

### 一、核心概念

| 概念 | 定义 | 一句话记忆 |
|------|------|-----------|
| **Fintech** | Financial + Technology，通过科技手段使金融服务更高效 | 技术让金融更高效 |
| **评分卡模型** | 根据客户属性和行为数据，对信用进行评分的风控手段 | 信用打分卡 |
| **A卡** | 申请评分卡（Application score card），用于贷前审批 | 贷前看申请 |
| **B卡** | 行为评分卡（Behavior score card），用于贷中监控 | 贷中看行为 |
| **C卡** | 催收评分卡（Collection score card），用于贷后催收 | 贷后看催收 |
| **因客定价** | 基于借款人信用、贡献度等因素差别化设定利率 | 看人下菜碟 |
| **GeoHash** | 将地理空间编码为字符串，用于区域划分和选址分析 | 地理编码 |
| **MAE** | Mean Absolute Error，平均绝对误差，越小模型越准 | 误差平均值 |
| **特征工程** | 从原始数据中构造、选择、转换特征的过程 | 造特征=定上限 |

### 二、金融行业七大部门与AI应用

```
个人金融 → 客户画像、精准营销、家庭财务规划
公司金融 → 企业信用评估、供应链金融
金融机构 → 负债/资产/中间业务
渠道运营 → 客户行为分析、网点运营管理
风险管理 → 风险建模、市场风险预测、合规监控
资金交易 → 算法交易、市场趋势预测
数字财务 → 财务预测与决策、智能融资决策
```

### 三、关键数据

| 指标 | 数值 | 含义 |
|------|------|------|
| 新客获取成本 vs 老客户 | 5倍 | 老客户成本远低于新客 |
| 信用评分区间 | 350-950分 | 评分卡常用区间 |
| 优质客户利率 | 基准4.2%（下调0.8%） | 基于信用和贡献度 |
| 普通客户利率 | 6.5%（上浮1.5%） | 基准5%上浮 |
| 二手车评价标准 | MAE < 550 | 挑战赛目标 |

### 四、评分卡计算示例

**公式**：`客户评分 = 基准分 + 年龄评分 + 性别评分 + 婚姻状况评分 + 学历评分 + 贷款申请次数评分`

**示例计算**：27岁男、已婚、本科、贷款1次
`650 + 8(年龄) + 4(性别) + 8(婚姻) + 12(学历) + 13(次数) = 695分`

**范围**：最低630分，最高700分

### 五、制造与快消行业

| 行业 | 应用 | 方法 |
|------|------|------|
| 制造（螺栓） | 扭矩曲线缺陷检测 | 分析平台期长度、曲线抖动次数 |
| 制造（车身） | 表面缺陷检测 | 少样本→机器视觉；多样本→神经网络 |
| 快消 | 供应链补货预测 | 精准预估降低缺货损失 |

### 六、二手车价格预测特征工程框架

```
1. 时间特征：车龄、注册季节、每年行驶公里数
2. 车辆特征：功率、品牌-车型组合、异常值处理
3. 统计特征：品牌均价/中位价/标准差
4. 分类编码：频率编码（避免one-hot维度爆炸）
5. 特征选择：删除SaleID/name/offerType/seller等无用特征
```

**黄金法则**：特征工程决定模型上限，模型调参只是把上限跑出来。

#### 问答题参考答案

**第1题：**
- **A卡（申请评分卡）**：用于贷前阶段，评估新申请客户的信用风险，主要依赖外部征信数据。
- **B卡（行为评分卡）**：用于贷中阶段，对已有客户进行动态监控，利用历史平台行为数据，不重复查询外部数据以节省成本。
- **C卡（催收评分卡）**：用于贷后阶段，评估逾期客户的催收优先级，主要利用催收人员记录的客户历史表现数据。
- **业务价值**：不同阶段的风险特征不同，分阶段评分能更精准地控制风险——贷前防准入、贷中动态监控、贷后差异化催收，实现全生命周期风险管理。

**第2题：**
- **时间特征处理**：转换日期格式，衍生车龄（天/年）、注册季节、每年行驶公里数、车龄分段等。车龄直接影响二手车残值，是重要预测变量。
- **车辆特征**：处理缺失值（中位数填充+缺失标记），构造品牌-车型组合特征，异常值检测与截断。缺失标记让模型捕捉"缺失"隐含信息，品牌-车型组合区分不同细分市场。
- **统计特征**：基于训练集统计品牌均价、中位价、标准差、样本数，计算品牌价格比。反映品牌市场定位和价值区间，只用训练集计算防止数据泄漏。
- **分类特征编码**：采用频率编码（反映类别流行度），避免one-hot维度爆炸。高基数类别（车型、品牌）效果尤佳，且无数据泄漏风险。
- **特征选择**：删除SaleID、name、offerType、seller等无区分度特征，将衍生分类特征转为category类型以适配CatBoost原生处理。
