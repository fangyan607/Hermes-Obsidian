## 🍅 番茄26：知识库RAG检索优化

> ⏱ 25分钟 | 📌 核心技能：RAG原理、分块策略、嵌入优化

### 26.1 RAG是什么？

```
RAG = Retrieval-Augmented Generation
       检索增强生成

传统AI回答：全靠训练数据中的记忆
RAG回答   ：从你的知识库中检索 → 结合检索结果生成 → 引用来源
```

### 26.2 RAG工作流程

```
用户提问 
    ↓
① Embedding：将问题转化为向量
    ↓
② 向量检索：在知识库中找最相似的文档块（Top-K）
    ↓
③ 上下文构建：将检索结果组装为Prompt上下文
    ↓
④ LLM生成：基于上下文生成回答
    ↓
⑤ 来源引用：标注引用的知识库条目
```

### 26.3 分块策略优化

分块（Chunking）是RAG最关键的超参数之一。

```yaml
# Hermes RAG配置优化

knowledge:
  indexing:
    chunk_size: 1000         # 默认1000字符
    
    # 分区策略（推荐）
    chunking_strategy: "hierarchical"
    # 可选: "fixed" | "hierarchical" | "semantic"
    
    # 分层策略配置
    hierarchical:
      max_chunk_size: 2000      # 最大分块大小
      min_chunk_size: 500       # 最小分块大小
      overlap: 200              # 块重叠（保持上下文连贯）
      
    # 语义分块（更智能但更慢）
    semantic:
      enabled: true
      separator: "\n## "        # 按二级标题分块
      second_separator: "\n### "  # 三级标题作为子块
```

**分块策略对比**：

| 策略 | 速度 | 准确度 | 适用场景 |
|:---|:----|:------|:--------|
| 固定大小 | ⭐⭐⭐⭐ | ⭐⭐ | 测试环境 |
| 分层分块 | ⭐⭐⭐ | ⭐⭐⭐⭐ | 生产环境 |
| 语义分块 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 高精度要求 |

**经验法则**：
- 法律/医学文档：小分块（500-800字符）+ 大重叠（200字符）
- 故事/文章：大分块（1500-2000字符）+ 小重叠（100字符）
- 技术文档：中等分块（1000字符）+ 中等重叠（150字符）

### 27.1 Agent架构

```yaml
# Hermes多Agent架构
agents:
  # 知识咨询Agent
  knowledge-assistant:
    description: "通用知识库问答"
    knowledge_base: "main"
    
  # 领域Agent（继承知识咨询 + 专家提示词）
  psychologist:
    parent: "knowledge-assistant"
    knowledge_base: "psychology"
    system_prompt: "心理咨询专家..."
    
  legal-consultant:
    parent: "knowledge-assistant"
    knowledge_base: "law"
    system_prompt: "法律顾问..."
    
  story-writer:
    parent: "knowledge-assistant"
    knowledge_base: "writing/story"
    system_prompt: "故事创作专家..."
```

### 27.2 Agent路由规则优化

```yaml
routes:
  # 精确匹配（高优先级）
  - pattern: "^法律.*|.*[起诉诉讼合同].*"
    agent: "legal-consultant"
    priority: 100
    
  # 语义匹配（基于意图识别）
  - pattern: ".*(?:焦虑|抑郁|失眠|压力|情绪).*"
    agent: "psychologist"
    priority: 95
    
  - pattern: ".*(?:故事|小说|人物|情节|世界观).*"
    agent: "story-writer"
    priority: 90
    
  - pattern: ".*(?:文案|广告|营销|推广|标题).*"
    agent: "copywriter"
    priority: 90
    
  # 默认路由
  - pattern: ".*"
    agent: "knowledge-assistant"
    priority: 1
```

### 27.3 Agent Pipeline（进阶）

```yaml
# 多步骤任务流水线
pipelines:
  - name: "创作发表全流程"
    trigger: ".*创作.*发表.*"
    steps:
      - agent: "story-writer"
        task: "故事构思与框架"
        output: "story_outline"
        
      - agent: "script-writer"
        task: "脚本创作"
        input: "story_outline"
        output: "script_draft"
        
      - agent: "copywriter"
        task: "推广文案"
        input: "script_draft"
        output: "promo_copy"
        
      - agent: "knowledge-assistant"
        task: "最终审核"
        input: ["script_draft", "promo_copy"]
```

### 28.2 配置HTTPS

```bash
# 安装Nginx反向代理
sudo apt install -y nginx

# 安装Certbot（Let's Encrypt SSL证书）
sudo apt install -y certbot python3-certbot-nginx

# 配置Nginx反向代理到Hermes
sudo nano /etc/nginx/sites-available/hermes
```

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名
    
    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

```bash
# 启用站点
sudo ln -s /etc/nginx/sites-available/hermes /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 获取SSL证书
sudo certbot --nginx -d your-domain.com

# 定时自动续期
sudo certbot renew --dry-run
```

```bash
#!/bin/bash
# Hermes健康检查脚本

SERVER_URL="http://localhost:18789"
LOG_FILE="$HOME/logs/health-check.log"

# 检查服务是否在线
if curl -s -o /dev/null -w "%{http_code}" "$SERVER_URL" | grep -q 200; then
    echo "[$(date)] ✅ Hermes运行正常" >> "$LOG_FILE"
else
    echo "[$(date)] ❌ Hermes异常！尝试重启..." >> "$LOG_FILE"
    sudo systemctl restart hermes
fi

# 检查知识库索引状态
hermes knowledge status >> "$LOG_FILE" 2>&1

# 检查磁盘使用
DISK_USAGE=$(df -h / | tail -1 | awk '{print $5}' | sed 's/%//')
if [ "$DISK_USAGE" -gt 85 ]; then
    echo "[$(date)] ⚠️ 磁盘使用率: ${DISK_USAGE}%" >> "$LOG_FILE"
fi
```
