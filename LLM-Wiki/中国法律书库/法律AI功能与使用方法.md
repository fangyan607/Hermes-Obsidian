## 二、功能一：智能法律查询

### 2.1 功能说明

颠覆传统关键词检索，支持**自然语言问答式查询**，精准匹配法律条文，解决"不会搜、搜不准、看不懂"的问题。

### 2.2 查询模式

| 模式 | 说明 | 示例 |
|------|------|------|
| 自然语言提问 | 口语化问题输入 | "民间借贷利息上限是多少" |
| 精准条文检索 | 按法律名称、法条编号检索 | "民法典第680条" |
| 关联法条拓展 | 自动关联配套法规 | 查民法典 → 关联司法解释 |
| 时效性甄别 | 自动标注有效/废止状态 | "刑法最新版本" |

### 2.3 使用方法

**步骤1：提出问题**
```
用户输入：公司可以因员工旷工直接辞退吗？
```

**步骤2：AI检索分析**
```
AI自动：
1. 识别关键词：旷工、辞退、员工
2. 匹配法律库：劳动法、劳动合同法
3. 检索司法解释：劳动争议相关解释
```

**步骤3：输出结果**
```
AI输出：
1. 核心法律依据
   - 《劳动合同法》第39条：严重违反规章制度可解除
   - 《劳动合同法实施条例》相关规定

2. 适用条件
   - 旷工天数标准（通常连续15天或累计30天）
   - 规章制度是否公示
   - 解除程序是否合法

3. 实务注意事项
   - 需要书面通知工会
   - 需要保留考勤证据
   - 经济补偿问题

4. 相关案例参考
   - [关联案例链接]
```

### 2.4 输出内容标准

| 内容项 | 说明 |
|--------|------|
| 核心法律条文原文 | 权威法律依据 |
| 条文释义 | 理解条文含义 |
| 适用条件 | 何种情况下适用 |
| 关联法规 | 配套法规链接 |
| 新旧对比 | 修订变化说明 |
| 实务注意事项 | 实践操作要点 |

## 三、功能二：案例智能分析

### 3.1 功能说明

依托案例库、裁判文书库，实现案例智能匹配、深度拆解、多案对比，为案件办理提供实务参考。

### 3.2 分析模式

| 模式 | 说明 |
|------|------|
| 相似案例匹配 | 输入案情 → 检索相似度最高的案例 |
| 案例深度拆解 | 提取案情、争议焦点、裁判依据、判决结果 |
| 多案例对比 | 分析同案同判、类案不同判原因 |

### 3.3 使用方法

**步骤1：输入案情**
```
用户输入：
案件类型：民间借贷纠纷
借款金额：10万元
约定利率：年利率24%
已还款：5万元
出借人起诉要求：还款本息共15万元
```

**步骤2：AI案例匹配**
```
AI自动：
1. 提取案情要素
2. 在案例库中检索相似案例
3. 按相似度排序展示
```

**步骤3：输出分析报告**
```
AI输出：

【案例深度拆解】

1. 基本案情
   - 借款事实确认
   - 利率约定情况
   - 还款记录

2. 争议焦点
   - 利率是否超过法定上限
   - 已还款项如何抵充
   - 本息计算方式

3. 裁判依据
   - 民法典第680条
   - 最高人民法院关于审理民间借贷案件适用法律若干问题的规定
   - 利率上限：LPR的4倍（当前约13.8%）

4. 裁判观点
   - 超过LPR四倍的部分无效
   - 已还款优先抵充利息

5. 判决结果预判
   - 本金：10万元
   - 合法利息：按LPR四倍计算
   - 超额还款部分可抵充本金

6. 实务启示
   - 借条应明确利率标准
   - 还款凭证需妥善保管
```

### 3.4 案例分析输出模板

| 模块 | 内容 |
|------|------|
| 基本案情 | 案件核心事实、起因、经过、结果 |
| 争议焦点 | 双方核心争议问题、法律分歧点 |
| 裁判依据 | 判决引用的全部法律、司法解释条文 |
| 裁判观点 | 法院核心裁判思路、审判逻辑 |
| 判决结果 | 判罚结论、量刑标准、责任划分 |
| 实务启示 | 同类案件办理要点、风险规避方式 |

## 四、功能三：案件判案辅助

### 4.1 功能说明

这是系统的**核心高阶功能**。针对用户提供的真实案件事实，基于法律法规和案例数据，实现智能化案情研判、法律适用、结果预判。

### 4.2 判案流程

```
┌─────────────────────────────────────────────────────────────┐
│                    案件判案辅助流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入案情 → 要素抽取 → 法律适用 → 裁判预判 → 风险提示       │
│     ↓           ↓           ↓           ↓           ↓      │
│  案情文本    结构化要素    具体法条    结果区间    风险点    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 使用方法

**步骤1：输入案件事实**
```
用户输入：
甲向乙借款10万元，约定年利率36%，借款期限1年。
到期后甲还款8万元，乙起诉要求甲还款本息共15万元。
```

**步骤2：AI案情要素抽取**
```
AI自动抽取：
- 案件主体：甲（借款人）、乙（出借人）
- 法律关系：民间借贷合同关系
- 借款金额：10万元
- 约定利率：年利率36%
- 已还款：8万元
- 争议金额：本息共15万元
```

**步骤3：AI法律适用判定**
```
AI自动判定：
1. 适用法律：民法典、民间借贷司法解释
2. 利率合法性：36%超过LPR四倍（约13.8%），超出部分无效
3. 已还款处理：优先抵充利息，超出部分抵充本金
```

**步骤4：AI裁判结果预判**
```
AI预判：
1. 本金认定：10万元
2. 合法利息：按LPR四倍计算（约13.8%）
3. 已还款处理：
   - 按36%计算利息：不合法
   - 按13.8%计算利息：合法
   - 超出部分抵充本金
4. 预判结果：
   - 乙的诉讼请求部分支持
   - 实际欠款金额需重新计算
```

**步骤5：AI风险提示**
```
AI提示：
1. 证据风险：借条是否明确利率约定
2. 计算风险：还款凭证是否完整
3. 程序风险：诉讼时效是否届满
```

### 4.4 不同案件类型的预判内容

## 五、功能四：合规风险审查

### 5.1 功能说明

支持合同、制度、经营行为等文本内容AI合规审查，自动筛查违规条款、不合规表述、法律风险。

### 5.2 审查模式

| 模式 | 说明 |
|------|------|
| 合同合规审查 | 筛查违规条款、不合规表述 |
| 法律风险提示 | 识别潜在法律风险 |
| 修改建议 | 给出合规修改建议 |
| 合规依据 | 列明合规/违规法律依据 |

### 5.3 使用方法

**步骤1：提交审查文本**
```
用户输入：借款合同主要条款
第一条：借款金额10万元
第二条：借款利率年利率36%
第三条：借款期限1年
第四条：逾期利率为日利率0.1%
第五条：违约金为借款本金的20%
```

**步骤2：AI合规审查**
```
AI审查结果：

【合规审查报告】

1. 利率条款
   ⚠️ 风险提示：年利率36%超过法定上限
   📖 法律依据：民法典第680条，利率不得超过LPR四倍
   ✏️ 修改建议：将利率调整为LPR四倍以内

2. 逾期利率条款
   ⚠️ 风险提示：日利率0.1%（年化36.5%）过高
   📖 法律依据：逾期利率总计不得超过LPR四倍
   ✏️ 修改建议：调整逾期利率标准

3. 违约金条款
   ⚠️ 风险提示：违约金过高可能被调整
   📖 法律依据：违约金不得超过实际损失的30%
   ✏️ 修改建议：降低违约金比例或明确损失计算方式

4. 综合评估
   🔴 高风险：利率条款
   🟡 中风险：逾期利率、违约金条款
   🟢 合规：借款金额、期限条款
```

### 5.4 审查领域

| 领域 | 审查要点 |
|------|----------|
| 合同审查 | 条款合法性、权利义务平衡、违约责任 |
| 劳动合同 | 劳动法合规、社保缴纳、解除条件 |
| 公司章程 | 公司法合规、股东权利、治理结构 |

### 6.3 使用方法

**步骤1：选择文书类型**
```
用户选择：民事起诉状
```

**步骤2：填写基本信息**
```
原告：张三，男，1980年生，住北京市朝阳区
被告：李四，男，1975年生，住北京市海淀区
诉讼请求：
1. 判令被告偿还借款本金10万元
2. 判令被告支付利息（按LPR四倍计算）
3. 诉讼费由被告承担
事实与理由：2024年1月，被告向原告借款10万元，约定年利率12%，借款期限1年。到期后被告未还款。
```

**步骤3：AI生成文书**
```
AI输出：

【民事起诉状】

原告：张三，男，1980年生，住北京市朝阳区
被告：李四，男，1975年生，住北京市海淀区

诉讼请求：
一、判令被告偿还借款本金人民币10万元；
二、判令被告支付借款利息（以10万元为基数，自2024年1月起至实际清偿之日止，按全国银行间同业拆借中心公布的贷款市场报价利率的四倍计算）；
三、判令本案诉讼费用由被告承担。

事实与理由：
2024年1月，被告李四因资金周转需要向原告张三借款人民币10万元，双方约定借款期限为1年，年利率为12%，被告出具借条一张。借款到期后，被告未按约还款。经原告多次催讨，被告拒不履行还款义务。

此致
北京市朝阳区人民法院

具状人：张三
2026年5月27日

【法律依据】
1. 《民法典》第667条：借款合同是借款人向贷款人借款，到期返还借款并支付利息的合同。
```

## 七、功能六：法律智能问答

### 7.1 功能说明

支持自然语言法律问答，提供法律知识科普、实务指导、以案释法等服务。

### 7.2 问答类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 法律概念 | 法律名词解释 | "什么是不当得利" |
| 法律制度 | 制度原理说明 | "诉讼时效制度" |
| 实务指导 | 办事流程指导 | "如何起诉离婚" |
| 以案释法 | 案例法律解读 | "案例分析" |

### 7.3 使用方法

**用户提问**：
```
什么是善意取得？需要满足什么条件？
```

**AI回答**：
```
【善意取得】

一、概念
善意取得，是指无权处分人将其占有的动产或登记在其名下的不动产转让给第三人时，若第三人受让该物时出于善意，则依法取得该物所有权的一项法律制度。

二、法律依据
《民法典》第311条：无处分权人将不动产或者动产转让给受让人的，所有权人有权追回；除法律另有规定外，符合下列情形的，受让人取得该不动产或者动产的所有权：
（一）受让人受让该不动产或者动产时是善意；
（二）以合理的价格转让；
（三）转让的不动产或者动产依照法律规定应当登记的已经登记，不需要登记的已经交付给受让人。

三、构成要件
1. 受让人善意：不知道转让人无处分权
2. 合理价格：以市场价格或接近市场价购买
3. 完成公示：已登记或已交付
4. 转让人无处分权：转让人非所有权人或未经授权

四、实务案例
甲将电脑借给乙使用，乙将电脑卖给不知情的丙，丙以市场价购买并已收到电脑。此时丙善意取得电脑所有权，甲无权要求丙返还。

五、风险提示
1. 购买二手物品应核实卖家是否有处分权
2. 保留交易凭证、支付记录
3. 及时办理过户登记（不动产）
```

## 八、使用技巧

### 8.1 提问技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| 明确问题类型 | 确定是查询法条还是分析案例 | "查询民间借贷利率规定" |
| 提供完整信息 | 案件分析需提供完整事实 | 包含主体、行为、后果、争议 |
| 分步提问 | 复杂问题拆分为多个小问题 | 先问法律依据，再问适用条件 |
| 追问关联 | 获取更多信息 | "还有哪些相关法规" |

### 8.2 常见问题

**Q1：查询结果是否权威？**
A：所有输出内容100%源自本地资料库，无外网信息，确保权威合规。

**Q2：时效性如何保证？**
A：系统自动标注法条的有效/废止/修订状态，建议优先使用最新版本。

**Q3：判案预判是否准确？**
A：预判基于同类生效案例的裁判尺度，仅供参考，具体以法院判决为准。

**Q4：如何处理多个版本的法律？**
A：系统优先展示最新版本，同时保留历史版本供参考。
