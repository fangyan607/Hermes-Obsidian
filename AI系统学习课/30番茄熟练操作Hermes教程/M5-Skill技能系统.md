## 🍅 番茄17：理解 Skill 系统（25分钟）

### 17.1 用大白话理解 Skill

**Skill 是什么？**

想象你招了一个全能助理，但他不会做你行业里的专业工作。于是你：

1. **写操作手册** → 这就是创建 Skill
2. **教他照着做** → 这就是安装 Skill
3. **他学会了以后自动做** → 这就是 Skill 自动生成

**核心思维转变：**

```
传统 AI：每次都要详细描述任务
Hermes + Skill：一句话触发，AI 自动按技能流程执行
```

### 17.2 Skill 的类型

| 类型 | 来源 | 特点 | 示例 |
|:-----|:-----|:-----|:-----|
| **内置 Skill** | 安装时自带 | 开箱即用 | 代码生成、文档处理 |
| **社区 Skill** | agentskills.io 下载 | 别人写好的技能 | 深度研究、网页抓取 |
| **自定义 Skill** | 自己创建 | 完全定制 | 日报生成、知识库查询 |
| **自动生成 Skill** | 执行任务时 AI 自动创建 | 动态学习 | 复杂任务标准化 |

### 17.3 Skill 的工作机制

```
┌─────────────────────────────────────────────────────────────┐
│              Skill 的工作流程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户输入任务描述                                              │
│      ↓                                                      │
│  Hermes 匹配已有 Skill？                                     │
│      ├── 匹配成功 → 按 Skill 流程执行                        │
│      └── 匹配失败 → 普通对话处理                              │
│                          ↓ (如果 auto_generate=true)          │
│                      AI 自动创建新 Skill                     │
│                           ↓                                  │
│                      存入 skills/ 下次可直接用               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 🍅 番茄18：安装与触发 Skill（25分钟）

### 18.1 查看已有 Skill

```bash
# 列出所有 Skill
hermes skill list

# 查看 Skill 详情
hermes skill show <skill-name>
```

### 18.2 安装社区 Skill

从社区下载 Skill：

```bash
# 搜索可用 Skill
hermes skill search <关键词>

# 安装 Skill
hermes skill install <skill-name>
```

你也可以从 agentskills.io 浏览和安装 Skill。

### 18.3 手动触发 Skill

```bash
# 方法1：在对话中通过描述触发
hermes chat
# → "请帮我做深度研究，分析 AI Agent 的发展趋势"
# (Hermes 自动匹配"deep-research" Skill)

# 方法2：直接指定 Skill
hermes chat --skill <skill-name>
```

### 18.4 对话中触发示例

```
You: 请帮我做一次深度研究，分析 2026 年 AI Agent 的发展趋势。

Hermes: 好的，我来使用"深度研究"技能执行这个任务。
[开始执行研究流程...]
1. 搜索相关信息
2. 分析多个来源
3. 汇总关键发现
4. 输出结构化报告
```

### 18.5 Skill 触发不成功的排查

| 问题 | 原因 | 解决方案 |
|:-----|:-----|:---------|
| 技能未触发 | 描述不精确 | 使用技能名称中的关键词 |
| 技能找不到 | 未安装 | `hermes skill install` 先安装 |
| 匹配错技能 | 关键词冲突 | 调整描述或删除冲突技能 |
| 技能执行报错 | 依赖缺失 | 查看日志，补充依赖 |

## 🍅 番茄19：Skill 自动生成（25分钟）

### 19.1 什么是自动生成？

当 `auto_generate: true` 时，Hermes 在执行复杂任务后会**自动将过程抽取为 Skill**，以后类似任务只需一句话触发。

**这就是 Hermes 的"肌肉记忆"——用越多，越聪明。**

### 19.2 配置自动生成

```yaml
# config.yaml
skills:
  enabled: true
  auto_generate: true      # 开启自动生成
  path: ~/.hermes/skills   # 技能存储路径
```

### 19.3 触发自动生成示例

```
You: 请帮我写一份关于本周 AI 行业动态的日报。
      格式要求：1）重要新闻 2）深度分析 3）我的观点
      信息来源：搜索最近3天的 AI 相关新闻

Hermes: [执行任务... 完成后自动创建"日报生成" Skill]

下次你说"帮我生成今天的日报"时，就会自动使用这个 Skill。
```

### 19.4 查看自动生成的 Skill

```bash
# 查看自定义 Skill（包含自动生成的）
ls ~/.hermes/skills/custom/

# 查看所有 Skill
hermes skill list
```

### 19.5 Skill 文件结构

自动生成的 Skill 本质是一个 Markdown 文件：

```markdown
# 技能名称: 日报生成

## 描述
自动生成 AI 行业动态日报

## 触发词
- 日报
- 今日简报
- daily report

## 执行流程
1. 搜索最近3天的AI相关新闻
2. 筛选重要信息
3. 按格式整理
4. 输出结构化报告

## 参数
- 时间范围: 最近N天（默认3天）
- 输出格式: markdown
```

## 🍅 番茄20：Skill 实操练习（25分钟）

### 20.1 三级练习

```
===== 循环 1：入门任务 =====
1. 运行 `hermes skill list` 查看已有 Skill
2. 从社区安装一个 Skill（如 "web-search"）
3. 在对话中触发该 Skill
✅ 验证方式：Skill 成功安装并执行

===== 循环 2：进阶任务 =====
1. 完成一个复杂任务（如"帮我研究XX话题并写报告"）
2. 检查是否触发了 Skill 自动生成
3. 查看自动生成的 Skill 文件
✅ 验证方式：custom/ 目录下有新 Skill 文件

===== 循环 3：挑战任务 =====
1. 手动创建一个自定义 Skill
2. Skill 内容：自动整理 Obsidian 笔记
3. 在对话中触发这个 Skill
4. 验证执行效果
✅ 验证方式：新 Skill 能被正确触发并执行
```
