## 番茄钟1：三层记忆架构（25分钟）

### 1.1 用大白话理解记忆系统

**Hermes 的记忆系统是什么？**

想象你的大脑有三层记忆：
1. **长期记忆**（MEMORY.md）- 像一本永不丢失的日记
2. **用户画像**（USER.md）- 像一份个人档案卡
3. **会话上下文** - 像当前对话的短期记忆

**核心思维转变：**

```
传统AI：每次对话都是"失忆"状态
Hermes：每次对话前先"翻阅"你的日记和档案
```

## 番茄钟2：记忆写入机制（25分钟）

### 2.1 记忆写入的两种方式

**方式一：手动写入**

直接编辑 MEMORY.md 文件，下次会话生效。

**方式二：自动写入**

AI 在对话过程中自动更新记忆。

### 2.2 策展式写入（Curated Write）

**什么是策展式写入？**

想象一个**图书管理员**，不是把所有书都堆进仓库，而是：
1. 筛选有价值的内容
2. 分类整理
3. 删除过时信息
4. 保持仓库整洁

**为什么需要策展式写入？**

| 问题 | 后果 | 策展式写入解决 |
|:-----|:-----|:---------------|
| 记忆膨胀 | Token消耗大，响应慢 | 自动压缩旧记忆 |
| 信息冗余 | 重复信息干扰 | 去重合并 |
| 过时信息 | 错误记忆 | 自动清理 |
| 隐私泄露 | 敏感信息进入记忆 | 过滤敏感信息 |

### 2.3 配置策展式写入

```yaml
# config.yaml
memory:
  enabled: true
  max_tokens: 10000        # 记忆窗口大小
  compression: true        # 压缩旧记忆
  curated_write: true      # 开启策展式写入
  write_threshold: 0.7     # 写入阈值（重要性评分>0.7才写入）
```
