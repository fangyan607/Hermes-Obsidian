### 三、问答题答案要点

**1. DeepSeek-R1与DeepSeek-V3的区别及应用场景：**

**DeepSeek-V3：**
- 定位：通用大语言模型
- 特点：MoE架构，平衡性能与效率，支持多任务
- 适用场景：日常对话、内容创作、代码辅助、翻译等通用任务

**DeepSeek-R1：**
- 定位：推理增强模型
- 特点：强化学习训练，深度思维链，推理透明
- 适用场景：复杂推理、数学证明、逻辑分析、代码调试等需要深度思考的任务

**选择建议：**
- 简单任务选V3，效率更高
- 复杂推理任务选R1，质量更好

**2. 思维链（Chain of Thought）技巧：**

**原理：**
- 让模型模仿人类"一步步思考"的过程
- 将复杂问题分解为简单步骤
- 每步推理都显式呈现，便于验证

**使用方法：**
1. 直接指令："让我们一步步思考这个问题"
2. Few-shot + CoT：提供带推理过程的示例
3. 分步提问：引导模型按步骤回答

**适用场景：**
- 数学计算题
- 逻辑推理题
- 复杂决策分析
- 代码调试与优化
- 问题诊断

### 四、应用题参考答案

**提示词设计示例：**

```
# 角色设定
你是一位专业的电商数据分析师，擅长用户反馈分析和情感洞察。

# 背景信息
我是一家电商公司的运营人员，收集了100条用户对某产品的评论数据。
我需要分析这些评论，了解用户真实感受，找出产品问题，提出改进建议。

# 任务要求
请对以下用户评论数据进行分析：

[评论数据]
{{在此粘贴100条评论}}

# 分析步骤
请按以下步骤完成分析：

## 第一步：情感分析
- 将每条评论分类为：正面/负面/中性
- 统计各类别的数量和占比

## 第二步：问题提取
- 从负面和中性评论中提取用户提到的主要问题
- 按出现频率排序
- 归类到：产品质量、物流服务、价格、客服、其他

## 第三步：改进建议
- 基于发现的问题，提出具体的改进建议
- 每条建议需包含：问题描述、改进方向、预期效果

# 输出格式
请使用以下格式输出：

## 一、情感分析统计
| 类型 | 数量 | 占比 |
|------|------|------|
| 正面 | XX条 | XX% |
| 负面 | XX条 | XX% |
| 中性 | XX条 | XX% |

## 二、主要问题（按频率排序）
1. [问题类别] - [具体问题描述]（出现XX次）
...

## 三、改进建议
### 优先级高
1. **问题**：xxx
   **建议**：xxx
   **预期效果**：xxx
...

## 四、总结
一句话总结用户满意度和核心改进方向。
```

**运用的提示词技巧：**

| 技巧 | 具体应用 |
|------|----------|
| 角色设定 | "你是一位专业的电商数据分析师" |
| 背景上下文 | 说明任务背景和数据来源 |
| 任务分解 | 三步走：情感分析→问题提取→改进建议 |
| 格式约束 | 指定表格和分点格式 |
| 分隔符 | 使用#和##分隔内容块 |
| 可操作性 | 每步都有明确的输出要求 |
