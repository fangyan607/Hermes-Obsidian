### 1.3 代码质量分析

```bash
codex -q -m gpt-5.5 "
对项目进行深度代码质量分析，识别以下问题：

## 代码异味
- 过长函数（>50行）
- 过多参数（>5个）
- 重复代码
- 上帝类/上帝对象
- 全局变量滥用

## 架构问题
- 循环依赖
- 模块耦合度
- 缺少抽象层
- 数据流混乱

## 技术债务
- 遗留的 TODO/FIXME
- 已弃用的 API 使用
- 过时的依赖版本
- 缺少类型定义

## 安全风险
- 硬编码密钥
- SQL 注入风险
- XSS 漏洞
- 不安全的认证

对每个问题标注严重等级：🔴 严重 / 🟡 中等 / 🟢 轻微
"
```

## 番茄钟4：安全重构策略（25分钟）

### 4.1 Codex 重构的安全原则

```
原则一：小步提交
  每次只改一个逻辑单元 → 容易回退

原则二：先测试后重构
  重构前确保有足够测试 → 重构后验证不回归

原则三：保持接口兼容
  先加新接口，再弃用旧接口 → 不影响调用方

原则四：可回退
  每次重构前 commit → 出问题直接 reset
```

### 4.2 Git 驱动的安全重构工作流

```bash
# Step 1: 在重构前保存当前状态
git checkout -b refactor/god-service
git commit -m "chore: checkpoint before refactoring GodService"

# Step 2: 让 Codex 分析并生成方案
codex -q "分析 GodService.ts，给出拆分方案" > plan.md

# Step 3: 逐步执行重构（每次只改一个文件）
codex -a auto-edit "创建 AuthService.ts，从 GodService 提取认证相关功能"

# Step 4: 验证
npm test

# Step 5: 提交
git add -A && git commit -m "refactor: extract AuthService from GodService"

# Step 6: 重复 Step 3-5 直到完成

# Step 7: 合并
git checkout main
git merge refactor/god-service
```

### 4.3 并行重构（多 Agent）

```bash
# 终端 1：重构 Module A
cd ../project-agent1
codex -a auto-edit "重构 Module A，提取核心逻辑到独立服务"

# 终端 2：重构 Module B  
cd ../project-agent2
codex -a auto-edit "重构 Module B，移除与 Module A 的循环依赖"

# 并行执行，各自在分支上工作
```

```
1. 用 Codex 分析项目中 3 个不同的文件，识别代码异味
2. 记录每个文件的问题数量和严重等级
3. 生成一份排序后的重构优先级列表
验证方式：分析报告中至少包含 3 种不同类型的代码问题

===== 循环 2：制定并执行重构 =====
1. 从优先级列表中选一个 P0 问题
2. 用 Codex 制定重构方案（包含回退方案）
3. 在 Git 分支上执行重构，每步提交一次
验证方式：重构后所有测试通过，生成变更摘要

===== 循环 3：量化重构效果 =====
1. 对比重构前后的代码质量指标（行数、圈复杂度、重复率等）
2. 让 Codex 生成重构总结报告
3. 记录本次重构的耗时和效果
验证方式：至少 2 个质量指标有可量化的改善
```
