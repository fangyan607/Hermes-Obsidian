## 番茄钟1：DNS 工作原理

### 1.1 用大白话理解 DNS

**DNS 是什么？**

想象互联网是一个**巨大的电话簿**：
- 电话簿记录：姓名 → 电话号码
- DNS 记录：域名 → IP 地址

**没有 DNS：**
- 你要访问网站，需要记住 172.67.72.239
- 人类记不住这么多数字

**有了 DNS：**
- 你只需要记住 google.com
- DNS 帮你翻译成 IP 地址

### 1.2 DNS 解析流程

```
用户输入 www.example.com
        ↓
    本地 DNS 缓存
    有记录？──→ 是 → 直接返回 IP
        │
        否
        ↓
    递归 DNS 服务器（ISP 提供）
        ↓
    根域名服务器（.）
        │ 返回 .com 的服务器地址
        ↓
    顶级域名服务器（.com）
        │ 返回 example.com 的服务器地址
        ↓
    权威域名服务器（Cloudflare）
        │ 返回 www.example.com 的 IP
        ↓
    返回给用户，同时缓存
```

### 1.3 Cloudflare DNS 的优势

| 特性 | 说明 |
|:-----|:-----|
| **速度快** | 全球 Anycast，平均查询 <10ms |
| **免费** | 无限域名，不限查询次数 |
| **安全** | DNSSEC 支持，防止 DNS 劫持 |
| **智能** | 负载均衡，故障转移 |

## 番茄钟2：DNS 记录类型

### 2.1 常用 DNS 记录类型

| 记录类型 | 用途 | 示例 |
|:---------|:-----|:-----|
| **A** | 域名 → IPv4 地址 | `example.com → 192.0.2.1` |
| **AAAA** | 域名 → IPv6 地址 | `example.com → 2001:db8::1` |
| **CNAME** | 域名 → 另一个域名 | `www → example.com` |
| **MX** | 邮件服务器 | `example.com → mail.example.com` |
| **TXT** | 文本记录（验证用） | SPF、DKIM、验证记录 |
| **NS** | 域名服务器 | `example.com → bob.ns.cloudflare.com` |
| **SRV** | 服务记录 | VoIP、游戏服务器 |

### 2.2 各记录类型详解

**A 记录（最常用）**

```
名称：@（代表根域名）
类型：A
内容：192.0.2.1
代理状态：已代理（橙色云）

效果：example.com → 192.0.2.1
```

**CNAME 记录（别名）**

```
名称：www
类型：CNAME
内容：example.com
代理状态：已代理

效果：www.example.com → example.com
```

**MX 记录（邮件）**

```
名称：@
类型：MX
内容：mail.example.com
优先级：10

效果：xxx@example.com 的邮件发往 mail.example.com
```

### 2.3 TTL（生存时间）

**TTL 决定 DNS 记录缓存多久：**

| TTL 值 | 适用场景 |
|:-------|:---------|
| 3600秒（1小时） | 稳定的记录，减少查询 |
| 300秒（5分钟） | 需要快速更新的记录 |
| 自动 | 让 Cloudflare 自动选择 |

## 番茄钟3：Cloudflare DNS 配置

### 3.1 进入 DNS 管理

```
Cloudflare Dashboard
    → 选择域名
    → 左侧菜单 DNS
    → Records
```

### 3.2 添加 A 记录

**场景：将域名指向服务器 IP**

```
点击 "Add Record"

类型：A
名称：@（或留空，代表根域名）
IPv4地址：你的服务器IP，如 192.0.2.1
代理状态：已代理（橙色云）推荐
TTL：自动

点击保存
```

### 3.3 添加 CNAME 记录

**场景：www 子域名指向主域名**

```
点击 "Add Record"

类型：CNAME
名称：www
目标：example.com（或 @）
代理状态：已代理
TTL：自动

点击保存
```

### 3.4 批量导入记录

**导入格式（BIND）：**

```
example.com.    IN  A       192.0.2.1
www.example.com. IN CNAME   example.com.
mail.example.com. IN A      192.0.2.2
example.com.    IN  MX  10  mail.example.com.
```

> ✋ **费曼自测**：添加一条 A 记录指向你的服务器 IP。

## 番茄钟4：代理模式详解

### 4.1 代理开启 vs 关闭

| 对比项 | 代理开启（橙色云） | 代理关闭（灰色云） |
|:-------|:-------------------|:-------------------|
| **IP 地址** | 显示 Cloudflare IP | 显示真实 IP |
| **CDN 加速** | ✅ 有 | ❌ 无 |
| **安全防护** | ✅ 有 | ❌ 无 |
| **缓存** | ✅ 有 | ❌ 无 |
| **SSL** | ✅ 自动配置 | 需要手动配置 |
| **适用场景** | 网站服务 | 仅需 DNS 解析 |

### 4.2 何时关闭代理

**需要关闭代理的场景：**

| 场景 | 原因 |
|:-----|:-----|
| 邮件服务器 | MX 记录不支持代理 |
| FTP 服务 | 需要真实 IP |
| 游戏/语音服务器 | 需要 UDP 直连 |
| 域名验证 | 某些验证需要真实 IP |

### 4.3 仅 DNS 模式配置

```
在 DNS 记录编辑页面：

代理状态：点击云朵图标
    橙色 → 灰色 = 关闭代理

适用于：
- MX 记录（邮件）
- TXT 记录（验证）
- SRV 记录（服务）
```

## 番茄钟6：验证与排查

### 6.1 验证命令

**验证 DNS 解析：**

```bash
# 查询 A 记录
dig example.com A

# 查询 CNAME
dig www.example.com CNAME

# 查询所有记录
dig example.com ANY

# 指定 DNS 服务器查询
dig @8.8.8.8 example.com A
```

**验证代理是否生效：**

```bash
# 查询返回的 IP
dig example.com A +short

# 如果返回 Cloudflare IP（104.x.x.x 或 172.x.x.x）
# 说明代理已生效
```

### 6.2 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|:-----|:---------|:---------|
| **域名无法访问** | NS 未生效 | 检查 NS 是否修改正确 |
| **显示真实 IP** | 代理未开启 | 点击云朵开启代理 |
| **解析错误** | 记录配置错误 | 检查记录类型和内容 |
| **www 无法访问** | CNAME 未配置 | 添加 www 的 CNAME |

### 6.3 DNS 传播检查

**在线工具：**
- https://dnschecker.org/
- https://www.whatsmydns.net/

**检查步骤：**
1. 输入域名
2. 选择记录类型
3. 查看全球解析情况

> ✋ **费曼自测**：使用 dig 命令验证你的 DNS 配置是否生效。
