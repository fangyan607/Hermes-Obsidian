## Section A：费曼学习法复习文档

### 一、本节课的核心问题

> **如何用集成学习（Boosting类模型）解决分类和回归预测问题？**

---

### 二、核心概念地图

```
预测任务
  ├── 分类问题（员工离职预测 / 男女声音识别）
  └── 回归问题（二手车价格预测）

常用模型
  ├── 线性分类：LR、SVM
  ├── 树模型：GBDT、XGBoost、LightGBM、CatBoost、NGBoost
  └── 集成学习：Blending（70%训练基模型，30%训练元模型）

机器学习神器（Boosting系列）
  ├── XGBoost  → 二阶泰勒展开 + 正则化项 + 贪心分裂 + 近似直方图算法
  ├── LightGBM → XGBoost + Histogram + GOSS + EFB（更快、更省内存）
  └── CatBoost → 高效处理分类特征 + 对称决策树 + 过拟合程度小
```

---

### 三、关键思想提炼

#### 1. 集成学习三大流派

| 方法 | 训练方式 | 核心作用 | 典型代表 |
|------|---------|---------|---------|
| **Bagging** | 并行训练多个基学习器 | 减少 **variance** | 随机森林 |
| **Boosting** | 串行训练，每个新模型依赖于前一个 | 减少 **bias** | AdaBoost、GBDT、XGBoost |
| **Stacking** | 不同模型用全部数据训练，再用Meta Classifier组合 | 综合多种模型优势 | Blending是其简化版本 |

#### 2. GBDT → XGBoost → LightGBM 演进逻辑

```
GBDT（梯度提升决策树）
  └── XGBoost 的改进：
        ├── 目标函数加入正则化项（叶子数T + L2范数w），防止过拟合
        ├── 对损失函数做二阶泰勒展开（用一阶导g和二阶导h），收敛更快
        ├── 贪心算法 + 近似直方图算法加速分裂点寻找
        └── 支持并行计算（按特征列并行计算增益）
  
  └── LightGBM 的优化：
        ├── Histogram算法：连续特征离散化成k个bin，候选分裂点从n-1降到k-1
        ├── GOSS算法：保留梯度大的样本，采样梯度小的样本，减少计算量
        └── EFB算法：将互斥稀疏特征绑定，减少特征数量
```

#### 3. XGBoost 目标函数推导（精髓）

```
Obj(Θ) = L(Θ) + Ω(Θ)
        损失函数  正则化项

Ω(f) = γT + (1/2)λ∑w_j²
      叶子数   L2正则

经过二阶泰勒展开后，每个叶子节点的最优权重：
w_j* = -G_j / (H_j + λ)

对应的结构分数（打分函数）：
Obj = -1/2 ∑(G_j²/(H_j+λ)) + γT

分数越小 → 树结构越好
Gain = 左Obj + 右Obj - 分裂前Obj - γ（新增叶子代价）
Gain < 0 → 不分裂
```

#### 4. 如何防止模型过拟合

| 优化层面 | 具体方法 |
|---------|---------|
| **模型层面** | 正则化（L1/L2）、降低max_depth(5-8)、增大min_samples_leaf(10-20)、Dropout(0.2-0.5) |
| **早停机制** | 监控验证集loss，patience=10-20个epoch |
| **集成方法** | Blending（70%训练基模型，30%训练元模型） |

---

### 四、模型选型速查

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 作为baseline快速出结果 | LR | 实现简单、速度快、可解释性强 |
| 样本数 < 特征数 | SVM + 线性核 | 不用选择非线性核 |
| 样本数 >= 特征数 | SVM + 非线性核 | 映射到高维空间效果更好 |
| Kaggle竞赛、通用场景 | XGBoost / LightGBM | 精度高、速度快、效果好 |
| 分类特征多 | CatBoost | 原生支持类别特征，不需要OneHot编码 |
| LR欠拟合 | 增加feature | tree-ensemble则增加树的数量 |

### 问答题参考答案

**1. XGBoost目标函数推导：**

XGBoost的目标函数由两部分组成：**损失函数**（拟合数据） + **正则化项**（惩罚复杂模型）。

```
Obj(Θ) = L(Θ) + Ω(Θ)

正则化项：Ω(f) = γT + (1/2)λ∑w_j²
           T = 叶子节点数量，γ控制叶子数复杂度
           w_j = 叶子分数，λ控制L2正则强度

第t轮迭代时的目标函数：
Obj(t) = ∑L(y_i, ŷ_i^(t-1) + f_t(x_i)) + Ω(f_t)

对L做二阶泰勒展开：
Obj(t) ≈ ∑[g_i * f_t(x_i) + (1/2) * h_i * f_t(x_i)²] + Ω(f_t)

其中 g_i = ∂L/∂ŷ^(t-1)（一阶梯度）
     h_i = ∂²L/∂(ŷ^(t-1))²（二阶梯度）
```

定义每个叶子节点上的梯度和：
```
G_j = ∑g_i,   H_j = ∑h_i   （对叶子j上所有样本求和）

Obj = -1/2 ∑[G_j²/(H_j+λ)] + γT
```

这个 Obj 称为**结构分数（打分函数）**——分数越小，代表在指定树结构下损失最小化得越好，即树的**结构越好**。分裂时的增益计算：
```
Gain = (左子树GL²/(HL+λ) + 右子树GR²/(HR+λ) - (GL+GR)²/(HL+HR+λ)) / 2 - γ
```

**2. 三种模型对比与选型建议：**

| 维度 | XGBoost | LightGBM | CatBoost |
|------|---------|----------|----------|
| 提出时间 | 2014（陈天奇） | 2017（微软） | 2017（Yandex） |
| 分裂算法 | 预排序 + 近似直方图 | Histogram + GOSS + EFB | 对称决策树 |
| 分类特征 | 需OneHot编码 | 支持 | **原生支持，最强** |
| 训练速度 | 较慢 | **最快（1/10时间）** | 中等 |
| 内存消耗 | 较高 | **最低（1/6内存）** | 中等 |
| 过拟合控制 | 正则化+早停 | 正则化+早停+采样 | **过拟合程度最小** |
| 并行能力 | 支持（按特征列） | 支持 | 支持GPU |

- **场景A（Kaggle竞赛，数值特征为主）：推荐 LightGBM**。训练速度最快，内存消耗最小，精度与XGBoost相当。在处理大规模数据时效率优势明显，是Kaggle使用最多的模型之一。
- **场景B（大量类别特征，特征稀疏）：推荐 CatBoost**。原生支持分类特征，不需LabelEncoding或OneHot预处理，对分类数据处理效果远优于XGBoost和LightGBM。过拟合程度最小，测试集准确度最高。
- **场景C（快速出baseline，强调可解释性）：推荐 LR（逻辑回归）**。实现简单，计算量小，速度极快，方便观测样本概率分数。可以作为预测的Baseline模型，之后再尝试更复杂的树模型提升效果。
