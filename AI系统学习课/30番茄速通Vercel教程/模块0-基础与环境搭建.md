## 🍅1 Vercel 账号注册与平台概览（25分钟）

### 费曼输入

Vercel 是一个云平台即服务(PaaS)，专门用于**前端应用和 Serverless 后端的部署托管**。与传统的服务器管理不同，你只需要推送代码，Vercel 自动处理构建、部署、CDN 分发和 HTTPS。

**核心优势**：
- **零配置部署**：自动检测框架（React/Vue/Next.js/Svelte 等）
- **全球 CDN**：部署到 100+ 边缘节点
- **Serverless Functions**：在后端写 API，无需管理服务器
- **Fluid Compute**：按实际 CPU 使用付费，无请求时不扣费
- **AI 云基础设施**：AI Gateway、Sandbox、Workflow SDK

### 刻意练习

1. 打开 [vercel.com](https://vercel.com)，点击 **Sign Up**
2. 使用 GitHub/GitLab/Bitbucket 账号登录（推荐 GitHub，便于后续导入仓库）
3. 完成新手引导（创建第一个项目或跳过）
4. 浏览 Dashboard 各功能区：
   - **Overview**：项目总览和最近部署
   - **Projects**：所有项目列表
   - **Teams**：团队管理（多人协作时使用）
   - **Settings**：账号设置和账单
5. 理解 Hobby Plan（免费版）的限制：
   - 每月 100h 构建时长
   - 每月 100GB 带宽
   - Serverless Functions 每月 100 万次调用
   - Sandbox 45 分钟超时

- [ ] 理解 Dashboard 各功能区
- [ ] 了解 Hobby Plan 的免费额度限制

### 📖 费曼三句话

1. Vercel 是一个"只管写代码，部署交给我"的云平台
2. 它自动检测你用的是什么框架，自动配置构建和 CDN

## 🍅2 Vercel CLI 安装与登录（25分钟）

### 费曼输入

Vercel CLI 是连接本地开发环境和 Vercel 云平台的命令行工具。通过它，你可以在终端完成部署、管理环境变量、查看日志等所有操作。

**核心命令**：
- `vercel` — 部署到预览环境
- `vercel --prod` — 部署到生产环境
- `vercel dev` — 本地运行（加载 vercel.json 配置和 Serverless Functions）
- `vercel link` — 本地目录关联 Vercel 项目

### 刻意练习

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 验证安装
vercel --version  # 应显示 v48+ 版本

# 2. 登录
vercel login

# 根据提示选择登录方式（推荐 GitHub）

# 3. 验证登录
vercel whoami

# 4. 创建一个测试项目（用任意前端项目或从头创建）
mkdir vercel-test
cd vercel-test
echo '<h1>Hello Vercel!</h1>' > index.html

# 5. 部署预览
vercel

# 6. 部署到生产
vercel --prod
```

### ✅ 完成标准

- [ ] CLI 安装成功，`vercel --version` 正常输出
- [ ] `vercel login` 完成认证
- [ ] 成功部署一个 HTML 页面到预览和生产环境

### 📖 费曼三句话

1. Vercel CLI 是你在终端里控制 Vercel 的"遥控器"
2. `vercel` = 预览部署（测试用），`vercel --prod` = 正式上线
3. `vercel dev` 可以在本地模拟线上环境，包括 API 函数

## 🍅3 首个项目部署（从 Git 仓库导入）（25分钟）

### 费曼输入

除了用 CLI 部署，更常见的方式是将 GitHub 仓库连接到 Vercel。这样每次推送代码到仓库，Vercel 会自动重新部署（CI/CD）。

**自动检测的框架**（部分）：Next.js、React(Vite/CRA)、Vue、Svelte、Astro、Nuxt、Gatsby、Hexo、Hugo

### 刻意练习

```bash
# 1. 创建一个 GitHub 仓库
# 在 GitHub 上创建新仓库 vercel-first-project

# 2. 本地初始化 React Vite 项目
npm create vite@latest vercel-first-project -- --template react
cd vercel-first-project
npm install
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的用户名/vercel-first-project.git
git push -u origin main

# 3. 在 Vercel 中导入
# 访问 https://vercel.com/new
# → Import Git Repository → 选择 vercel-first-project
# → 框架自动检测为 Vite → 点击 Deploy

# 4. 验证自动部署
# 部署完成后获得一个 .vercel.app 域名
# 每次 git push 都会触发自动重新部署

# 5. (可选) 连接自定义域名
# Vercel 项目 → Settings → Domains → 添加自定义域名
```

## 🍅4 vercel.json 配置详解（25分钟）

### 费曼输入

`vercel.json` 是 Vercel 项目的配置文件，放在项目根目录。它控制路由规则、构建命令、HTTP 头、重定向、Cron 任务等所有行为。

所有配置项见：[官方文档](https://vercel.com/docs/project-configuration)

### 刻意练习

```json
// 在项目根目录创建 vercel.json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "devCommand": "npm run dev",
  "framework": "vite",
  "cleanUrls": true,
  "trailingSlash": false,

  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ],

  "redirects": [
    { "source": "/old-page", "destination": "/new-page", "permanent": true }
  ],

  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    }
  ],

  "functions": {
    "api/**/*.ts": {
      "memory": 256,
      "maxDuration": 30
    }
  }
}
```

**关键是这三种路由控制**：

| 类型 | 作用 | 用例 |
|:----|:-----|:-----|
| `rewrites` | 内部转发（URL 不变） | SPA 路由 / API 代理 |
| `redirects` | 客户端重定向（URL 变化） | 链接迁移 / SEO |
| `headers` | 自定义 HTTP 响应头 | 缓存控制 / 安全策略 |

### ✅ 完成标准

- [ ] 创建 `vercel.json` 并理解每个字段含义
- [ ] 掌握 rewrites/redirects/headers 的区别
- [ ] 用 `vercel dev` 本地测试配置效果
- [ ] 部署后验证配置生效

### 📖 费曼三句话

1. vercel.json 是项目部署的"遥控器说明书"，告诉 Vercel 如何构建和路由
2. rewrites（内部转发）和 redirects（客户端跳转）是整个配置文件的核心
3. 每次修改 vercel.json 后重新部署即可生效

## 🍅5 环境变量与域名管理（25分钟）

### 费曼输入

环境变量是连接应用和外部服务（API 密钥、数据库 URL、配置参数）的安全方式。Vercel 支持按环境（Production/Preview/Development）分别设置环境变量。

### 刻意练习

```bash
# 1. 通过 CLI 添加环境变量
vercel env add VITE_API_URL

# 按提示选择环境（Production/Preview/Development）
# 输入值: https://api.example.com

# 2. 拉取环境变量到本地
vercel env pull .env.local

# 3. 在代码中使用（Vite 项目）
const apiUrl = import.meta.env.VITE_API_URL;

# 4. 批量添加多个变量
vercel env add DATABASE_URL production
vercel env add SUPABASE_KEY production

# 5. 通过 Vercel Dashboard 添加
# 项目 → Settings → Environment Variables → 添加

# 6. 添加自定义域名
# 项目 → Settings → Domains → 输入你的域名
# 按照提示在 DNS 服务商中添加 CNAME 记录
```

**环境变量作用域**：

| 环境 | 用途 | 在何时使用 |
|:-----|:-----|:----------|
| Production | 正式环境 | `vercel --prod` |
| Preview | 预览/分支部署 | `vercel`（非 main 分支） |
| Development | 本地开发 | `vercel dev` |

### ✅ 完成标准

- [ ] 通过 CLI 和 Dashboard 分别添加环境变量
- [ ] 理解三个环境（Production/Preview/Development）的区别
- [ ] `vercel env pull` 成功拉取环境变量到本地
- [ ] 能添加并配置自定义域名

### 📖 费曼三句话

1. 环境变量 = 应用的"配置文件"，把密钥和参数写在代码外面
2. Vercel 区分 Production/Preview/Development 三个环境，变量可以不同
3. 自定义域名需要去 DNS 服务商添加解析记录
