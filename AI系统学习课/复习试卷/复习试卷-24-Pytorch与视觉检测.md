## Section A：费曼学习法复习文档

### 一、PyTorch 核心概念

#### 1. 张量（Tensor）
- Tensor = 替代 NumPy 的多维数组，核心区别在于 Tensor 可以放到 GPU 上加速计算
- 创建方式：`torch.Tensor()`、`torch.zeros()`、`torch.from_numpy()`
- 支持丰富运算：幂运算、行列求和、对角矩阵、矩阵运算等

#### 2. 自动求导（Autograd）
- 设置 `requires_grad=True` 后，PyTorch 跟踪运算并构建计算图
- 调用 `.backward()` 自动计算梯度
- 示例：`y = x**2` → `y.backward()` → 梯度 `dy/dx = 2x`

#### 3. 动态计算图 vs 静态图
- **PyTorch 动态图**：每次前向传播实时构建新图，可逐行调试，适合 RNN/GAN 等复杂结构
- **TensorFlow 1.x 静态图**：先构建图再执行，调试不直观

#### 4. 常用功能函数
- `torch.nn` 和 `torch.nn.functional` 功能相同
- 核心函数：`conv2d`、`max_pool2d`、`relu`、`dropout`
- `nn.Module` 提供预定义层，`torch.optim` 提供优化器（SGD、Adam）

#### 5. 分布式训练
- **DataParallel（单机多卡）**：`nn.DataParallel(model, device_ids=[0,1])`，简单但主 GPU 负载不均
- **DistributedDataParallel（多机多卡）**：每个 GPU 独立计算梯度再同步，更高效率
- **PyTorch Lightning 简化**：设置 `accelerator="gpu"` 和 `devices=N` 即可

### 二、图像识别技术

#### 1. 传统图像识别流程
```
数据获取 → 预处理（滤波/平滑/增强/边缘提取）→ 特征提取 → 决策分类 → 输出
```
- 常见特征：幅度特征、统计特征、几何特征、SIFT 特征

#### 2. 人脑识别 vs CNN
- 人脑通过分级、多层网络模型识别，逐层减少数据量，保留有用信息
- CNN 模仿人脑神经系统：假定过滤器图 → 逐行卷积 → 得到特征图（Feature Map）
- 训练过程：随机初始化过滤器 → 前向传播 → 误差函数计算梯度 → 反向传播 → SGD 更新过滤器

#### 3. 深度学习视觉检测
- **传统检测不足**：物体变化时规则需重新设计，不同批次也带来变化
- **深度学习优势**：处理不易量化的变量（照明、颜色变化、曲面、视野）
- **深度学习不足**：数据量要求大 → 解决方案：数据增强、无监督学习、自动标注

#### 4. 卷积神经网络（CNN）
- 从生物学概念演化：视觉皮层存在 S 细胞（感受野内对边缘刺激响应）和 C 细胞（更大感受野，空间定位）
- CNN = 让权重在不同位置共享的神经网络
- 卷积核（kernel/filter/feature detector）滑动扫描，参数共享
- Conv2d 参数：`in_channels`、`out_channels`、`kernel_size`、`stride`、`padding`
- **CNN 层级可视化**：
  - Layer 1-2：颜色、边缘等低层特征
  - Layer 3：纹理特征
  - Layer 4：有区别性的特征（如狗头）
  - Layer 5：完整的辨别性关键特征

### 三、缺陷检测方案

#### 方案三要素
1. **硬件**：相机、镜头、光源
2. **数据**：数据湖打通（工业云、PLC 控制）
3. **算法**：传统（Halcon、OpenCV）vs 深度学习（YOLO、Faster RCNN）
4. **调试**：位置距离、角度、光线强度

#### 硬件选择要点
- **黑白 vs 彩色相机**：推荐黑白相机（算法基于灰度值处理），同分辨率下精度更高；基于颜色检测时用彩色
- **工业相机 vs 手机**：工业相机传感器更大（邮票大小），接受更多光线，可实现高速拍照+低光照成像
- **CCD vs CMOS**：CCD 灵敏度/分辨率高、噪声控制好；CMOS 低成本/低耗电
- **相机接口**：USB 即插即用但距离短（3米）；GigE 传输距离长（100米）
- **光源类型**：环形光（圆形物体）、条形光（矩形物体、角度要求高）、背光（轮廓尺寸测量）、线光（线阵相机）
- **光照角度**：直射光（背遮挡处形成黑色剪影）、反射光（镜面反射+漫反射）

#### 制定缺陷检测的考量
1. 数据量——尤其是缺陷数据量
2. 人工判异方法——写下规则，量化指标
3. 类似公开图片——数据不足时做迁移学习（如新冠 CT 仅 349 张缺陷样本，用其他肺部 CT 模型迁移）

### 四、YOLO 目标检测

#### YOLO 核心特点
- 速度快，达到实时要求，泛化能力强
- 将检测变为回归问题：输入整张图像 → 一个神经网络 → 直接输出 bounding box 和类别概率
- End-to-End 优化
- 标准版 YOLO 每秒 45 帧，Fast YOLO 每秒 155 帧

#### YOLOv1 原理
- **图像分割**：将图片分成 S×S 个 grid，物体中心落在哪个 grid 就由该 grid 检测
- **每个 grid**：预测 B 个 bounding box，每个 box 有 (x, y, w, h, confidence) 共 5 个量
- **输出张量**：S × S × (5×B + C)
  - 例：PASCAL VOC（20 类），S=7，B=2 → 7×7×30
  - 30 维含义：前 20 为类别概率，后 2 为边界框置信度，最后 8 为边界框 (x,y,w,h)
- **每个 grid 只能预测 1 个物体** → 7×7 grid 最多 49 个物体
- **置信度公式**：反映框含物体的可能性和预测准确率

#### 损失函数
```
误差 = 定位误差 + 包含目标的框置信度误差 + 不包含目标的框置信度误差 + 分类误差
```
- 预测框宽高用平方根之差，兼顾大框和小框误差
- YOLOv1 简单粗暴全部采用 sum-squared error loss

#### YOLOv1 不足
1. 输出层为全连接层，只支持与训练图像相同分辨率
2. 每个格子只能预测 1 个物体，小物体场景受限
3. 大/小物体 IOU 误差对 loss 贡献近似，小物体定位准确性受影响

#### YOLOv3 改进
- **多尺度预测**：13×13、26×26、52×52
- 先验框（anchors）基于数据集用 K-means 聚类计算
- 每个单元格预测 3×(4+1+B) 个值

#### 非极大值抑制（NMS）
1. 选 scores 最大的 box_best，保留
2. 计算 box_best 与其余 box 的 IOU
3. IOU > 阈值（如 0.5）则舍弃
4. 重复直到所有框处理完毕

#### IOU（重叠度）
- 定义两个 bounding box 的重叠度
- 用于评估定位精度

#### YOLOv12 使用
- **加载模型**：`model = YOLO('./yolov12n.pt')`
- **预测**：`results = model(image_path)` → `results[0].show()`
- **验证**：`model.val(data='coco.yaml')` → `metrics.box.map`
- **训练参数**：`epochs`、`batch`、`imgsz`、`scale`、`mosaic`、`mixup`、`copy_paste`、`patience`

### 五、钢铁缺陷检测实战

#### 项目数据
- 6 种缺陷：氧化铁皮压入(RS)、斑块(Pa)、开裂(Cr)、点蚀(PS)、夹杂(In)、划痕(Sc)
- 图像分辨率 200×200 像素
- Train：1400 张（xml 标注），Test：400 张

#### 检测流程
1. **数据标注**（labelImg）
2. **数据集切分**（9:1 比例）
3. **格式转换**（xml → txt）
4. **配置参数**（rbc.data、rbc.names）
5. **训练**（train.py）
6. **评估**（test.py — 输出 Seen/Nt/P/R/Ap/F1）
7. **预测**（detect.py → submission.csv）

#### 评估指标
- **mAP**：各类别 AP 的平均值
- **AP**：PR 曲线下面积
- **Precision** = TP/(TP+FP)
- **Recall** = TP/(TP+FN)
- **TP**：IoU>0.5 的检测框数
- **FP**：IoU≤0.5 的检测框或多余框
- **FN**：未检测到的 GT 数量

### 六、YOLO 发展演进总结

| 演进方向 | 详情 |
|---------|------|
| 回归→Anchor 机制 | v1 简单网格回归 → v2 引入 Anchor Boxes |
| 浅层→深层残差网络 | Darknet-19 → Darknet-53 → CSPDarknet53 → 融合 Transformer |
| 多尺度与注意力融合 | v3 多尺度预测、v4 Bag of Freebies/Specials、v9~v12 注意力模块 |
| 工业化易用性 | v5/v6/v7/v8 工程化，快速上手部署 |
| 精度与实时性并重 | 高 FPS 基础上追求更高精度 |

#### 问答题要点

**第 1 题要点：**
- YOLOv1 将图片分割为 S×S 个 grid，物体中心落在哪个 grid 就由该 grid 检测
- 每个 grid 预测 B 个 bounding box，每个 box 有 (x, y, w, h, confidence)
- 输出张量 S×S×(5×B+C)，例 7×7×30（前 20 类 + 2 置信度 + 8 定位）
- 优点：简化了检测流程，实现 end-to-end 优化
- 缺点：每个格子只能预测 1 个物体，当物体占画面比例较小时（如畜群、鸟群）会漏检

**第 2 题要点：**
- **硬件选型**：黑白相机（精度高、灰度值处理）vs 彩色相机（颜色特征）；工业相机传感器更大，适合高速低光照；CCD（高分辨率、低噪声）vs CMOS（低成本、低耗电）；光源类型（环形光、条形光、背光、线光）和角度（直射光、反射光）的选择
- **数据获取**：数据量（尤其是缺陷数据量）；数据湖打通（工业云、PLC）；数据不足时用迁移学习
- **算法选择**：传统算法（Halcon、OpenCV）适合易量化特征；深度学习（YOLO、Faster RCNN）适合不易量化的变量；算法+硬件+调试联动
