1. **知识点拆分**：按章节/主题拆分笔记内容，适配LLM-WIKI嵌入检索窗口大小
2. **内部链接关联**：同类知识点建立双向WikiLink，AI检索可联动调取关联笔记

## 二、LLM-WIKI检索运行架构

整体流程：Obsidian MD笔记 -> 框架索引扫描 -> 向量缓存入库 -> 目录/标签路由筛选 -> 调用绑定SKILL技能 -> 生成创新/认知/算法应答

### 1. LLM-WIKI基础索引配置

1. 指定**创造性思维知识库**总文件夹为专属检索数据源，划定扫描范围
2. 开启元数据、分类、技能标签联合索引，优先按 `skill_tag` 触发对应功能
3. 调整嵌入匹配阈值，兼顾检索速度与精准度
4. 开启笔记溯源功能，回答可跳转查看原始MD笔记原文

### 2. 四类检索匹配规则

| 类型 | 说明 |
|------|------|
| **目录定向检索** | 按创造力理论/认知科学/AI算法等类目快速锁定相关笔记 |
| **标签技能检索** | 识别 skill_tag，自动匹配对应能力模块 |
| **语义智能检索** | 理解问题语义，跨笔记整合知识点 |
| **层级检索区分** | 按 level 标签区分入门科普与专业算法内容输出 |

### 3. 多技能联动匹配

对于跨域复杂问题，LLM-WIKI自动识别多维度需求，同时激活多个SKILL模块，实现知识融合：

| 复杂问题 | 联动技能 |
|----------|----------|
| "如何用AI辅助创意设计？" | Skill3(AI算法) + Skill1(创造力) + Skill5(设计思维) |
| "大脑如何产生创新想法？" | Skill2(认知科学) + Skill1(创造力) |
| "如何可视化复杂算法？" | Skill3(AI算法) + Skill4(可视化思维) |

## 四、三大实际使用模式

### 模式1：知识问答检索

**流程**：提问 -> LLM-WIKI匹配目录/标签 -> 调取笔记内容 -> 整合回答（支持溯源）

**适用场景**：
- 查询创造力理论概念（如"什么是创造力曲线？"）
- 了解认知科学原理（如"大脑的记忆机制是什么？"）
- 学习算法知识（如"CNN和RNN有什么区别？"）

**示例**：
```
用户：创新者的五大基因是什么？
-> 匹配category:创造力理论 + skill_tag:Skill1
-> 调取《创新者的基因》笔记
-> 输出：五大基因（联想、质疑、观察、实验、社交）+ 各基因详解 + 训练方法
```

### 模式2：创新方法指导

**流程**：输入创新需求 -> 匹配创造力/设计思维SKILL -> 输出创新方法与工具推荐

**适用场景**：
- 产品创新方案设计
- 商业模式创新探索
- 团队创新工作坊设计
- 个人创造力提升计划

**示例**：
```
用户：我们团队想设计一个创新的用户激励体系，有什么方法？
-> 联动Skill1(创造力激发) + Skill5(设计思维)
-> 调取多本创新方法笔记
-> 输出：用户激励体系创新设计流程 + 创意生成工具推荐 + 原型验证方案
```

### 模式3：AI算法学习

**流程**：提出算法问题 -> 匹配AI算法SKILL -> 按level分层调取笔记 -> 输出学习指导

**适用场景**：
- 基础算法学习（level:入门）
- 机器学习进阶（level:进阶）
- 深度学习研究（level:专业）

## 七、完整落地部署步骤

### 第一步：目录与索引搭建

1. 确认 `知识库/创造性思维/` 目录已包含全部16本MD文件
2. 创建 `.书库索引/` 目录，放入 README.md、书籍元数据索引.md、技能模块索引.md
3. 创建 `schema/` 目录，放入 CLAUDE.md、kb.yaml
4. 创建 Log.md 操作日志

### 第二步：YAML元数据批量补全

为16本MD笔记逐一添加YAML frontmatter：

```yaml
---
title: 书名
category: [创造力理论/认知科学/AI与算法思维/可视化思维/交叉学科]
skill_tag: [对应Skill编号]
level: [入门/进阶/专业]
keywords: [3-8个核心关键词]
type: creativity
---
```

### 第三步：LLM-WIKI索引配置

1. 在LLM-WIKI中选定 `知识库/创造性思维/` 为检索数据源
2. 开启元数据 + 分类 + 技能标签联合索引
3. 设置嵌入匹配阈值，完成初始化索引构建
4. 开启笔记溯源功能

### 第四步：SKILL技能绑定

将6项创造性思维SKILL技能与对应分类/标签绑定：
| 技能 | 绑定标签 | 绑定分类 |
|------|----------|----------|
| Skill1 创造力激发 | skill_tag: Skill1 | 创造力理论 |
| Skill2 认知科学 | skill_tag: Skill2 | 认知科学 |
| Skill3 AI算法思维 | skill_tag: Skill3 | AI与算法思维 |
| Skill4 可视化思维 | skill_tag: Skill4 | 可视化思维 |
| Skill5 设计思维 | skill_tag: Skill5 | 交叉学科 |
| Skill6 系统思考 | skill_tag: Skill6 | 交叉学科 |

### 第五步：提示词导入配置

将第五部分的提示词模板导入LLM-WIKI，设定检索应答规范：
- 通用检索提示词作为默认配置
- 各SKILL专用提示词绑定对应技能模块

### 第六步：功能测试与调优

测试以下核心场景，微调匹配精度：

1. **知识问答**：测试创造力理论、认知科学、AI算法三类问题的检索准确度
2. **创新咨询**：测试创新方法指导场景的完整性与实用性
3. **算法学习**：测试入门/进阶/专业三级内容的分层输出

### 第七步：日常维护

- 新增书籍时：添加YAML -> 放入目录 -> 更新索引 -> 刷新缓存
- 定期检查：元数据完整性、技能绑定关系、检索效果
- 季度迭代：根据使用反馈优化技能定义与提示词模板
