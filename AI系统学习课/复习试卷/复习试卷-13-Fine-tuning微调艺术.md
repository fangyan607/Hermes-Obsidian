## Section A：费曼学习法复习文档

### 一、三大技术选型：提示工程 vs RAG vs 微调

| 问题类型 | 解决方案 | 说明 |
|---------|---------|------|
| 没问清楚 | 提示工程 | 优化 prompt，让模型理解意图 |
| 缺乏背景知识 | RAG | 检索外部知识库补充上下文 |
| 能力不足 | 微调 | 修改模型权重，提升特定任务能力 |

### 二、两类微调者

| 角色 | 目标 | 规模 | 技术路线 |
|------|------|------|---------|
| 基座模型公司（OpenAI、Meta、DeepSeek 等） | 通用语言能力 | 100B+，万亿 token，千卡月级 | 继续预训练 + 指令微调 + RLHF/PPO/DPO |
| 应用开发者（企业、创业公司、个人） | 垂直领域/私域知识 | 6B-70B，1k-1M 条数据，数小时-数天 | LoRA/QLoRA 为主，交付几十 MB 增量权重 |

### 三、高效微调方法一览

| 方法 | 可训练参数比例 | 核心思想 | 常用场景 |
|------|--------------|---------|---------|
| Prompt Tuning | <0.01% | 在输入层添加可训练的"软提示"向量 | >10B 大模型，资源极度受限 |
| P-Tuning v1/v2 | <0.1%~1% | 将软提示插入每一层输入（不仅是输入层） | NLU 任务，中小模型效果好 |
| Prefix Tuning | ~0.1%~3% | 在每层头部添加虚拟"前缀 token"，用 MLP 生成 | NLG 任务（对话、摘要、翻译） |
| **LoRA** | ~1%~10% | 低秩更新：用两个小矩阵乘积近似权重更新 | **最通用**，适合 LLM 和扩散模型 |
| QLoRA | ~1%~10% | LoRA + 4-bit 量化，内存更省 | 单卡微调超大模型（如 65B） |

### 四、LoRA 数学原理

**核心假设**：模型微调时的权重更新 ΔW 具有**内在的低秩特性**。

**数学体现**：对 ΔW 做 SVD 分解，奇异值衰减极快，绝大部分接近零，只有前几个奇异值很大 ⇒ 有效信息集中在少量方向上。

**前向传播公式**：

```
h = Wx + BAx
```

- W：原始预训练权重，**冻结**不动
- A：降维矩阵（高斯分布初始化）
- B：升维矩阵（初始化为全 0）
- BA = ΔW，即为低秩更新矩阵

**训练/推理**：
- 训练时只训练 A 和 B
- 推理时将 BA 合并回 W：`h = (W + BA)x`，**不引入额外推理延迟**
- 多任务切换只需换 BA：`W + B1A1 → W + B2A2`

**秩的选取**：一般任务 rank=1,2,4,8 即可；任务较大可选更大 rank。

### 五、矩阵分解与推荐系统

**核心思想**：将评分矩阵 R 分解为用户矩阵 X 和物品矩阵 Y，用 `x_u · y_i` 内积预测评分。

**目标函数**：

```
min Σ (r_{ui} - x_u^T y_i)^2 + λ(||x_u||^2 + ||y_i||^2)
```

**工程求解方法**：
1. **ALS（交替最小二乘法）**：固定 Y 优化 X，固定 X 优化 Y，交替迭代至收敛
2. **SGD（随机梯度下降）**：每次随机选一个样本更新参数

**隐含特征个数 k**：k 越大，分类越细，计算量越大。

### 六、SVD 奇异值分解

**公式**：`A = P · Λ · Q^T`
- P：左奇异矩阵（User 矩阵）
- Q：右奇异矩阵（Item 矩阵）
- Λ：对角线为奇异值 λ₁, λ₂, ..., λ_k

**图像压缩原理**：保留前 k 个最大奇异值，其余置零。k=50 时仅需保留约 8% 的元素，即可还原 99% 的图像信息。

### 七、微调数据准备

| 原则 | 说明 |
|------|------|
| 一致性 | 所有数据遵循相同模板格式 |
| 准确性 | 答案必须正确，garbage in garbage out |
| 多样性 | 指令和输入覆盖各种情况，提高泛化力 |

**数据量参考**（7B 模型）：
- 简单任务：1000-5000 条
- 复杂任务：5000-50000+ 条
- **黄金起点**：1000-6000 条高质量数据即可产生好效果

### 八、硬件需求与显存估算

**总显存估算公式**：

```
总显存 ≈ 模型权重 + 优化器状态 + 梯度 + 前向传播激活值
```

- 模型权重（FP16）：参数量 × 2 字节
- 优化器状态（AdamW）：L × 4 字节 × 2（动量 + 方差）
- 梯度：L × 2 字节
- 激活值：≈ 模型权重的 20%-50%

**案例**：7B 模型，LoRA 可训练参数 1%
- 权重：14 GB
- 优化器：~0.56 GB
- 梯度：~0.14 GB
- 激活值（30%）：~4.2 GB
- **总计：~19 GB** ⇒ 一张 24GB 显卡（如 RTX 4090）可流畅微调

**硬件建议**：
- 7B/8B：推荐 24GB 显存（3090/4090）
- 13B/14B：推荐 32GB+，或使用 QLoRA
- 70B：必须 QLoRA + 多卡

### 九、微调后模型评估

**数据集三划分**：
- 训练集：更新权重
- 验证集：监控训练、调超参、选 checkpoint
- 测试集：最终一次性评估，**全程封存**

**五个测试维度**：

| 维度 | 内容 |
|------|------|
| 任务主指标 | 分类：准确率/F1；生成：BLEU/ROUGE；问答：EM/F1 |
| 通用能力保持 | 常识推理、基础代码、通用对话 ⇒ 防灾难性遗忘 |
| 泛化能力 | 换句式、换词汇、加干扰 ⇒ 检验是真学会还是死记硬背 |
| 人工评估 | 流畅度、相关性、有用性 |
| 输出质量分析 | 正面案例 + 失败案例对比分析 |

**成功标准**：主指标大幅提升 + 通用能力无显著退化。

#### 问答题要点

**1. LoRA 核心思想：**
- 不直接修改原始权重 W，而是在旁边增加低秩旁路 BA
- W：原始预训练权重，冻结不动
- A：降维矩阵（高斯初始化），B：升维矩阵（全 0 初始化）
- BA = ΔW，是对 W 的低秩更新
- 训练时：只训练 A 和 B
- 推理时：将 BA 合并回 W，`h = (W + BA)x`，无额外推理延迟

**2. 微调 7B 模型：**
- (a) 一致性（统一模板格式）、准确性（答案正确）、多样性（覆盖各种情况）
- (b) 5000-50000+ 条
- (c) 总显存 ≈ 模型权重 + 优化器状态 + 梯度 + 激活值
  - 权重 14 GB + 优化器 0.56 GB + 梯度 0.14 GB + 激活值 ~4.2 GB ≈ 19 GB < 24 GB ⇒ 可用
