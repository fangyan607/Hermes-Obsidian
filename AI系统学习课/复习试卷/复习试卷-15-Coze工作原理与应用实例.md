## Section A：费曼学习法复习文档

### 1. 核心概念：Agent vs Copilot

| 维度 | Copilot | Agent |
|------|---------|-------|
| **自主性** | 辅助工具，依赖用户输入和反馈来提供建议和改进 | 较高自主性，能独立完成任务，不需要持续的用户干预 |
| **目标导向** | 辅助用户，提高任务完成的效率和质量 | 被设计为实现特定目标，可根据环境变化调整策略 |
| **交互方式** | 主要与用户交互，通过用户输入和反馈提供实时建议 | 可与其他Agent或系统进行复杂交互，常用于多Agent系统中 |
| **应用场景** | 编程辅助、文档写作、教育等需要实时建议和反馈的领域 | 广泛应用于自动化、监控、推荐系统等各种领域 |

### 2. Coze平台概况

Coze（扣子）是一个AI应用开发平台，用于快速搭建基于大模型的各类Bot，可发布到社交平台、通讯软件或网站。核心能力：

- **插件**：内置60+款插件（资讯阅读、旅游出行、效率办公、图片理解），支持自定义插件
- **知识库**：支持上传TXT、PDF、DOCX、Excel、CSV，也可基于URL获取在线网页内容
- **记忆能力**：持久化记忆，提供方便AI交互的数据库记忆能力
- **工作流**：灵活的工作流设计，用于处理逻辑复杂、稳定性要求高的任务

### 3. 搭建AI Bot的流程

**Step1**：点击创建Bot，填写相关信息。

**Step2**：进入Bot编排页面，包含三部分：
- **人设与回复逻辑**：描述Bot的身份和任务，可使用模板格式
- **中间技能面板**：配置各种扩展能力（插件、知识库、工作流等）
- **右侧预览与调试面板**：实时调试Bot

**Step3**：编写提示词。对于复杂Bot，推荐使用结构化Markdown格式，包含：
- `# Character` — Bot人设
- `## Skills` — Bot的功能（可包含多个Skill）
- `## Constraints` — Bot的约束规则

### 4. 插件使用与创建

**使用插件**：在技能面板 → 添加插件中可选择60多款已有插件。

**创建自定义插件**：每个插件可添加多个工具（工具 = 独立的API服务，即endpoint），同一插件内的工具必须使用相同域名。

创建步骤：
1. 在团队空间选择"插件"→"创建插件"
2. 填写插件信息（基于已有服务创建，配置URL、Header、Service、Region、Version）
3. 添加工具：配置工具名称、描述、路径、请求方法
4. 配置输入参数
5. 配置输出参数（支持自动解析返回参数）
6. 调试与校验
7. 发布插件

### 5. 工作流使用

工作流由多个**节点**组成，节点是基本单元，用于完成复杂任务处理。

#### CASE 1：搜索新闻工作流（getNews_tasks）
- 节点顺序：开始 → getToutiaoNews → 结束
- 开始节点新增 `user_input` 参数（String类型）
- getToutiaoNews节点引用开始节点的 `user_input`
- 结束节点引用getToutiaoNews的 `news`

#### CASE 2：意图识别工作流（weather_news）
- 使用**大模型节点**将用户输入分为3类：1（天气）、2（新闻）、3（其他）
- 使用**选择器节点**进行分支判断
- 天气分支：通过第二个大模型提取地理位置经纬度，传给天气插件
- 新闻分支：传给 getToutiaoNews 获取新闻
- 其他：直接返回

#### CASE 3：抖音文案提取&二创
- 节点顺序：用户输入 → 提取视频信息（插件） → 音频转文字（插件） → 文案风格优化（大模型） → 输出
- 中间增加输出节点显示处理状态

#### CASE 4：LLM联网搜索
- 节点顺序：用户输入 → 提取关键字（大模型） → Bing搜索（插件） → 文章总结（大模型） → 输出
- 中间增加输出节点显示"提取到关键字"

#### CASE 5：古诗词Agent（文生图）
- 节点顺序：用户输入古诗 → 联想意象（大模型） → 翻译英文（大模型） → 文生图（ByteArtist插件） → 输出图片URL

### 6. RAG知识库

**RAG（检索增强生成）**：解决大模型胡乱编造的问题，两个关键阶段：
- **检索阶段**：使用编码模型基于问题检索相关文档
- **生成阶段**：使用检索到的上下文作为条件生成文本

知识库支持上传外部数据（TXT、PDF、DOCX、Excel、CSV），自动分段和编码。

**应用场景**：虚拟人物生成（语料补充）、产品资料问答（客服场景）、专业领域知识（垂直场景）

#### CASE：飞连产品知识库
数据来源包括：管理员手册（线上文档）、客户端手册（Word）、故障排查手册（Word）、常见问题（Excel表格）、产品介绍（官网）、价格信息（线上文档）

**自定义切分文档**：对于长文档，可手动添加 `###` 分段标识符，配合2000长度分段

**在线数据采集**：安装Coze Scraper浏览器插件，在页面上标注要提取的内容

**表格数据**：创建知识库时选择"表格格式"，上传Excel文件如"大模型定价.xlsx"，对模型字段进行索引

**Bot搭建**：设置人设提示词 → 添加知识库 → 配置搜索策略（混合搜索）

#### 问答题参考答案

**第1题参考答案：**

创建weather_news工作流的完整步骤：

1. **创建工作流**：在页面顶部进入工作流页面，单击创建工作流。名称设置为 `weather_news`，描述为"识别用户意图并获取相关信息，只支持2种意图：获取天气、获取新闻"。

2. **添加节点**：依次添加大模型节点、选择器节点、插件节点。

3. **开始节点配置**：新增 `input` 参数，选择String类型。

4. **大模型节点配置**：单次模式，输入引用开始节点的 `input`。提示词：分析用户输入文本，与天气有关返回1，与新闻有关返回2，否则返回3。

5. **第一个选择器节点**：
   - 分支1：判断条件设置为"大模型-output 包含输入 1"，连接至第二个LLM节点（用于提取地理位置经纬度），再连接天气插件 `weather`（设置lat和lng参数引用大模型_1的输出），最后连到结束节点。
   - 分支2：否则区域连接至第二个选择器节点。

6. **第二个选择器节点**：
   - 判断条件设置为"大模型-output 包含输入 2"，连接至 `getToutiaoNews` 插件节点（搜索新闻），然后连到结束节点。
   - 否则区域直接连到结束节点。

7. **结束节点配置**：新增参数 `output` 引用大模型输出（返回意图），新增参数 `newsItems` 引用getToutiaoNews的news（返回新闻内容）。

8. **试运行验证**：测试"上海天气"（走天气分支）、"AI新闻"（走新闻分支）、"1+1=？"（直接返回）。

**Coze平台的Agent实际应用举例：**

- **AI新闻Agent**：利用Coze的插件（getToutiaoNews）和工作流，自动搜索AI相关新闻并筛选排序，无需用户持续干预，体现了Agent的自主性和目标导向。
- **抖音文案提取&二创Agent**：用户只需提供抖音视频链接，Agent自动完成视频信息提取、音频转文字、文案优化等多个步骤，体现了Agent的复杂任务处理能力。
- **飞连产品知识库Agent**：通过RAG知识库技术，Agent能够基于产品文档自动回答用户问题，体现了Agent在客服场景中的自动化应用。
- **古诗词Agent**：用户输入古诗，Agent自动完成意象联想、中英文翻译、文生图生成，展示了Agent在多步骤、多工具协同中的目标导向特性。
