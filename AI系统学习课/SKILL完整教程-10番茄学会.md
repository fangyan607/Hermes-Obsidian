## 番茄钟1：什么是SKILL？为什么需要它？

### 1.1 用大白话理解 SKILL

**SKILL 是什么？**

想象你有一个新员工入职，你需要给他一份**入职培训手册**：
- 告诉他公司是做什么的
- 告诉他工作流程是什么
- 告诉他遇到问题怎么办
- 告诉他哪些事不能做

**SKILL 就是给 AI 的"入职培训手册"**：
- 让 AI 在特定领域变得更专业
- 让 AI 按照你的期望方式工作
- 让 AI 记住重要的规则和流程

**核心思维转变：**

```
没有 SKILL：AI 是"通才"，什么都会但不够专业
有了 SKILL：AI 变成"专才"，在特定领域深度专业
```

### 1.2 SKILL 能做什么？

| 能力 | 没有SKILL | 有SKILL |
|:-----|:----------|:--------|
| **专业知识** | 通用知识，可能过时 | 领域专家，持续更新 |
| **工作流程** | 每次都要解释 | 自动按流程执行 |
| **工具集成** | 需要手动调用 | 自动识别和调用 |
| **记忆能力** | 会话内记忆 | 跨会话持久记忆 |
| **行为约束** | 可能做不该做的事 | 严格遵守规则 |

### 1.3 SKILL 的典型应用场景

```

## 番茄钟2：SKILL的核心原理与架构

### 2.1 SKILL 的三层结构

```
┌─────────────────────────────────────────────────────────────┐
│                    SKILL 三层结构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Layer 1: 元数据层（YAML Frontmatter）                       │
│  ├── name: 技能名称                                         │
│  ├── description: 技能描述（决定何时触发）                   │
│  └── license: 许可证                                        │
│           ↓                                                 │
│  Layer 2: 指令层（SKILL.md 正文）                            │
│  ├── 核心理念                                               │
│  ├── 执行流程                                               │
│  ├── 行为规则                                               │
│  └── 输出格式                                               │
│           ↓                                                 │
│  Layer 3: 资源层（可选，Bundled Resources）                  │
│  ├── scripts/: 可执行脚本                                   │
│  ├── references/: 参考文档                                   │
│  └── assets/: 静态资源                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 渐进式加载机制

**SKILL 的加载顺序：**

| 层级 | 内容 | 何时加载 | Token消耗 |
|:-----|:-----|:---------|:----------|
| 元数据 | name + description | 始终加载 | ~100 tokens |
| 指令层 | SKILL.md 正文 | 技能触发时 | <5K tokens |
| 资源层 | scripts/references | 按需加载 | 不限 |

**这种设计的意义：**
- 元数据始终在上下文中，AI 知道有哪些技能可用
- 只在需要时才加载完整指令，节省 Token
- 脚本可以无限复杂，不影响主上下文

### 2.3 技能触发机制

**触发方式：**

```
用户输入 → AI 分析意图 → 匹配 description → 触发 SKILL
```

**触发词设计原则：**

| 原则 | 说明 | 示例 |
|:-----|:-----|:-----|
| **具体性** | 描述要具体到场景 | "生成教程" vs "生成技术教程" |
| **多样性** | 覆盖多种表达方式 | "写教程"、"帮我写个教程"、"教程生成" |
| **区分性** | 避免与其他技能冲突 | 不同技能的触发词要有区分度 |

### 2.4 SKILL 目录结构

```
skill-name/
├── SKILL.md              # 必需：主文件
├── scripts/              # 可选：可执行脚本
│   ├── setup.sh
│   └── process.py
├── references/           # 可选：参考文档
│   ├── guide.md
│   └── examples.md
└── assets/               # 可选：静态资源
    ├── template.json
    └── config.yaml
```

## 番茄钟3：SKILL vs MCP 对比

### 3.1 用大白话理解区别

**SKILL 是什么？**
- 像**员工手册**：告诉 AI 怎么工作
- 侧重：知识、流程、规则

**MCP（Model Context Protocol）是什么？**
- 像**工具箱**：给 AI 提供外部工具
- 侧重：连接、调用、数据

### 3.2 详细对比

| 维度 | SKILL | MCP |
|:-----|:------|:----|
| **定义** | AI 的技能包 | AI 的外部连接协议 |
| **作用** | 赋予 AI 专业能力 | 让 AI 访问外部资源 |
| **内容** | 知识、流程、规则 | API、数据库、工具 |
| **触发** | 自动匹配 description | 需要配置和调用 |
| **开发** | 写 Markdown 文件 | 写代码/配置 |
| **复杂度** | 低（纯文本） | 中高（需要编程） |
| **适用场景** | 专业领域知识、工作流程 | 外部系统集成、数据访问 |

### 3.3 协同使用场景

```
┌─────────────────────────────────────────────────────────────┐
│                    SKILL + MCP 协同                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  场景：让 AI 管理 GitHub 项目                                │
│                                                             │
│  SKILL 提供：                                                │
│  ├── Git 工作流程规范                                        │
│  ├── 提交信息格式规则                                        │
│  ├── 代码审查标准                                            │
│  └── 项目管理方法论                                          │
│                                                             │
│  MCP 提供：                                                  │
│  ├── GitHub API 连接                                         │
│  ├── 读写仓库权限                                            │
│  ├── Issue/PR 操作能力                                       │
│  └── Webhook 事件订阅                                        │
│                                                             │
│  协同效果：                                                  │
│  SKILL 告诉 AI "怎么做" + MCP 提供"做的能力"                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 选择指南

| 你的需求 | 推荐方案 |
|:---------|:---------|
| 让 AI 按特定方法论工作 | SKILL |
| 让 AI 访问数据库/API | MCP |
| 让 AI 既懂方法又能操作 | SKILL + MCP |
| 快速创建，不需要编程 | SKILL |
| 需要复杂系统集成 | MCP |

> ✋ **费曼自测**：你的项目需要 SKILL 还是 MCP？还是两者都需要？

## 番茄钟4：SKILL.md 文件结构详解

### 4.1 完整文件结构

```markdown
---
name: skill-name
description: "技能描述，决定何时触发此技能"
license: MIT
---

# 技能标题

## 语言
[使用的语言]

## 何时激活
[触发条件说明]

## 核心理念
[一句话核心理念]

## 执行流程

### Phase 1: [阶段名称]
[具体步骤]

### Phase 2: [阶段名称]
[具体步骤]

## 行为规则
[必须遵守的规则]

## 不做的事
[明确禁止的行为]

## 自检清单
- [ ] 检查项1
- [ ] 检查项2
```

### 4.2 YAML Frontmatter 详解

**必需字段：**

```yaml
---
name: my-skill                    # 技能名称（小写、连字符）
description: "技能描述"            # 决定何时触发
license: MIT                      # 许可证
---
```

**可选字段：**

```yaml
---
name: my-skill
description: "技能描述"
license: MIT
version: 1.0.0                    # 版本号
author: Your Name                 # 作者
tags: [tag1, tag2]                # 标签
triggers:                         # 触发词
  - "触发词1"
  - "触发词2"
---
```

### 4.3 description 写法技巧

**好的 description 示例：**

```yaml
# ✅ 具体、明确、包含触发场景
description: "将任意技术主题转化为2个番茄钟的结构化学习教程。触发词：番茄费曼、学习教程、教程生成。"

# ✅ 说明功能和适用场景
description: "GitHub项目自动化助手。自动创建Issue、PR审查、代码提交。适用于：Git工作流、项目管理。"
```

**不好的 description 示例：**

```yaml
# ❌ 太模糊，容易误触发
description: "帮助用户做事"

# ❌ 没有说明触发场景
description: "一个强大的工具"
```

### 4.4 正文结构最佳实践

**执行流程的写法：**

```markdown
## 执行流程

### Phase 1: 准备阶段

**目标**：确认用户需求

**步骤**：
1. 询问用户具体需求
2. 确认输入材料
3. 明确输出格式

**输出**：需求确认清单

---

### Phase 2: 执行阶段

**目标**：完成核心任务

**步骤**：
1. [具体步骤1]
2. [具体步骤2]

**检查点**：执行前确认XXX
```

## 番茄钟5：编写你的第一个SKILL

### 5.1 创建 SKILL 目录

```bash

mkdir -p .claude/skills/my-first-skill
```

### 5.2 编写 SKILL.md

```markdown
---
name: note-summarizer
description: "笔记摘要生成器。将长篇笔记提炼为结构化摘要。触发词：摘要、总结、提炼笔记。"
license: MIT
---

# 笔记摘要生成器

## 语言
使用用户输入的语言。

## 何时激活
- 用户说"帮我总结这篇笔记"
- 用户说"提炼一下这个内容"
- 用户提供长文本要求摘要

## 核心理念
**结构化摘要 = 核心观点 + 关键要点 + 行动建议**

## 执行流程

### Phase 1: 分析输入

**步骤**：
1. 读取用户提供的笔记内容
2. 识别笔记类型（会议记录/读书笔记/技术文档/其他）
3. 确定摘要长度需求

### Phase 2: 提取核心

**步骤**：
1. 找出核心观点（一句话总结）
2. 提取3-5个关键要点
3. 识别可执行的行动项

### Phase 3: 生成摘要

**输出格式**：
```markdown
# [笔记标题] 摘要

> 📌 核心观点：[一句话]

## 关键要点
1. [要点1]
2. [要点2]
3. [要点3]

## 行动建议
- [ ] [行动1]
- [ ] [行动2]

## 原文链接
[[]]
```

## 行为规则
- 保持原文的核心意思，不添加个人观点
- 摘要长度控制在原文的10-20%
- 使用清晰的 Markdown 格式

## 不做的事
- 不改变原文的观点立场
- 不添加原文没有的信息
- 不跳过重要的细节

## 自检清单
- [ ] 核心观点准确吗？
- [ ] 关键要点完整吗？
- [ ] 格式清晰易读吗？
```

### 5.3 验证 SKILL

```bash

# 测试触发（在对话中）

### 5.4 常见问题排查

| 问题 | 原因 | 解决方案 |
|:-----|:-----|:---------|
| SKILL不触发 | description 不匹配 | 调整触发词 |
| 执行流程混乱 | Phase 划分不清 | 重新梳理流程 |
| 输出不符合预期 | 输出格式定义不清 | 明确输出模板 |

## 番茄钟7：复杂SKILL设计模式

### 7.1 多阶段流程设计

**适用于：复杂任务需要多步骤完成**

```markdown
## 执行流程

### Phase 0: 入口分流
判断用户输入属于哪种场景：
| 场景 | 处理方式 |
|------|---------|
| 场景A | → Phase 1A |
| 场景B | → Phase 1B |

### Phase 1A: [场景A处理]
[具体步骤]

### Phase 1B: [场景B处理]
[具体步骤]

### Phase 2: 统一处理

## 番茄钟9：最佳实践与常见问题

### 9.1 SKILL 编写最佳实践

**规则具体化：**

```markdown
# ❌ 模糊的规则
行为规则：做事要认真

# ✅ 具体的规则
行为规则：
1. 每次输出前检查格式是否正确
2. 确保所有链接都是有效的
3. 提供可执行的行动清单
```

**边界清晰化：**

```markdown
# ❌ 不清晰的边界
不做的事：不做坏事

# ✅ 清晰的边界
不做的事：
- 不编造用户没有提供的信息
- 不在用户明确拒绝后继续推荐
- 不存储敏感信息到外部服务
```

**检查点嵌入：**

```markdown
### Phase 2: 执行

**检查点**：执行前确认
- [ ] 输入数据完整？
- [ ] 参数格式正确？

**失败处理**：如果检查失败，返回 Phase 1 补充信息。
```

### 9.2 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|:-----|:-----|:---------|
| **触发不准确** | description 太模糊 | 添加具体触发词 |
| **执行偏离** | 流程描述不清晰 | 细化 Phase 步骤 |
| **输出不一致** | 输出格式定义不清 | 提供明确模板 |
| **Token超限** | SKILL.md 太长 | 压缩正文，移到 references/ |
| **调用失败** | 资源路径错误 | 使用相对路径 |

### 9.3 SKILL 质量评分表

| 维度 | 1分 | 3分 | 5分 |
|:-----|:----|:----|:-----|
| **触发准确性** | 经常误触发 | 基本准确 | 精准触发 |
| **流程清晰度** | 步骤混乱 | 基本清晰 | 逻辑严密 |
| **输出质量** | 格式混乱 | 基本可用 | 专业标准 |
| **边界定义** | 模糊不清 | 有基本边界 | 清晰完整 |
| **可维护性** | 难以修改 | 可以修改 | 结构清晰易改 |

**总分 20-25 分：优秀 SKILL**

> ✋ **费曼自测**：对照最佳实践，检查你的 SKILL 有哪些可以改进的地方。
