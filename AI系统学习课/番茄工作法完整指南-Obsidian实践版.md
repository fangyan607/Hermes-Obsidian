## 第一部分：番茄工作法理论基础

### 1.1 什么是番茄工作法？

番茄工作法（Pomodoro Technique）由 Francesco Cirillo 于1980年代末发明，是一种简单而高效的时间管理方法。

**核心原理：**
```
工作25分钟 → 休息5分钟 → 循环
每4个番茄后 → 长休息15-30分钟
```

### 1.2 为什么有效？

| 机制 | 说明 |
|------|------|
| **时间盒** | 固定时长创造紧迫感，对抗拖延 |
| **单任务** | 25分钟专注一件事，减少切换成本 |
| **节奏感** | 工作-休息交替，维持大脑最佳状态 |
| **可量化** | 用番茄数衡量工作量，清晰直观 |
| **心流触发** | 短时目标更容易进入专注状态 |

### 1.3 番茄工作法的五个阶段

```
1. 计划 → 选择要完成的任务
2. 设定 → 启动25分钟计时器
3. 专注 → 全神贯注工作直到计时器响
4. 记录 → 完成后标记，记录中断情况
5. 回顾 → 每日/每周总结，优化流程
```

### 2.1 当前配置分析

你的插件当前配置：

| 设置 | 当前值 | 建议值 | 说明 |
|------|--------|--------|------|
| `workLen` | 25分钟 | ✅ 保持 | 标准番茄时长 |
| `breakLen` | 5分钟 | ✅ 保持 | 标准休息时长 |
| `autostart` | 关闭 | ✅ 保持 | 手动控制更有意识 |
| `useStatusBarTimer` | 开启 | ✅ 保持 | 状态栏显示时间 |
| `notificationSound` | 开启 | ✅ 保持 | 提醒结束 |
| `showTaskProgress` | 开启 | ✅ 保持 | 显示进度 |
| `enableTaskTracking` | 关闭 | **建议开启** | 追踪任务番茄数 |
| `logFile` | NONE | **建议设置** | 记录到日记 |

### 2.2 推荐配置优化

打开 **设置 → 社区插件 → Pomodoro Timer**，建议修改：

```
✅ enableTaskTracking: 开启
   → 追踪每个任务的番茄数

✅ logFile: 设置为日记文件
   → 自动记录番茄完成情况

✅ logFormat: VERBOSE
   → 详细记录时间戳
```

### 2.3 基本操作

| 操作 | 方法 |
|------|------|
| **启动番茄钟** | 点击状态栏番茄图标，或命令面板 `Pomodoro: Start` |
| **暂停** | 点击状态栏计时器 |
| **跳过休息** | 命令面板 `Pomodoro: Skip Break` |
| **重置** | 命令面板 `Pomodoro: Reset` |
| **查看统计** | 命令面板 `Pomodoro: Show Stats` |

### 3.2 每日番茄工作流程

#### 🌅 早晨启动（5分钟）

```
1. 打开今日日记
2. 回顾"明日计划"承接任务
3. 填写番茄计划表：
   - 列出3-5个任务
   - 为每个任务估算番茄数
   - 总和不超过8个（一天实际工作时间约4小时）
4. 点击状态栏番茄图标，开始第一个番茄
```

#### 🍅 番茄执行中

```
专注规则：
✅ 手机静音/放远
✅ 关闭无关网页/应用
✅ 只做当前任务
✅ 有想法先记下来，不切换

中断处理：
- 内部中断（自己分心）→ 记录，继续专注
- 外部中断（他人打扰）→ 协商推迟或记录
```

#### 🌙 晚间复盘（10分钟）

```
1. 统计今日完成番茄数
2. 分析中断原因
3. 更新"番茄完成"属性
4. 填写反思
5. 规划明日任务
```

## 第四部分：进阶技巧

### 4.1 番茄数估算方法

**经验法则：**
```
1个番茄 = 25分钟专注时间
实际1小时 ≈ 2个番茄（含休息）

估算公式：
任务预计时间 ÷ 25分钟 = 番茄数
建议 +1 作为缓冲
```

**示例：**
| 任务类型 | 预计时间 | 番茄数 |
|----------|----------|--------|
| 回复邮件 | 15分钟 | 1 |
| 写文档 | 1小时 | 3 |
| 学习课程 | 2小时 | 5 |
| 深度工作 | 3小时 | 7 |

### 4.2 不同场景的番茄策略

#### 📚 学习场景
```
策略：番茄+主动回忆
- 每个番茄结束：用1分钟回忆学到的内容
- 每4个番茄：做一次知识梳理
```

#### 💻 编程场景
```
策略：番茄+调试窗口
- 番茄开始：明确要解决的具体问题
- 番茄结束：记录当前进度和下一步
- 遇到阻塞：利用休息时间放松思考
```

#### ✍️ 写作场景
```
策略：番茄+大纲
- 先用1个番茄做大纲
- 每个段落1-2个番茄
- 最后1个番茄用于修改
```

### 4.3 团队协作中的番茄工作法

```
团队番茄规则：
1. 番茄时间不安排会议
2. 紧急事项用"番茄中断"标记
3. 每日站会同步番茄完成情况
4. 周会分析团队番茄数据
```

## 第五部分：常见问题与解决方案

### Q1: 总是被打断怎么办？

**解决方案：**
```
1. 物理隔离：戴耳机、挂"专注中"标识
2. 时间隔离：告知同事你的番茄时间
3. 工具隔离：关闭通知、使用专注模式
4. 记录中断：分析中断来源，系统性解决
```

### Q2: 25分钟太短/太长？

**解决方案：**
```
根据任务类型调整：
- 写作/学习：25分钟（标准）
- 编程/设计：45-50分钟（深度工作）
- 行政/沟通：15分钟（短番茄）

但建议：保持25分钟标准，用多个番茄完成长任务
```

### Q3: 如何处理"差一点就完成了"？

**解决方案：**
```
严格原则：番茄时间到，必须停止

原因：
- 保护休息节奏
- 避免"再坚持一下"的过度疲劳
- 让未完成成为下一个番茄的动力
```

### Q4: 一天应该完成多少番茄？

**参考标准：**
```
理想情况：8个番茄 = 约4小时专注工作

实际建议：
- 新手：从4-5个开始
- 进阶：6-8个
- 高手：8-10个（包含深度工作）

注意：质量 > 数量
```
