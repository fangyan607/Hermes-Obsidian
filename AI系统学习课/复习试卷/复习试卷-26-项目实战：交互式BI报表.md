## Section A：费曼学习法复习文档

### A.1 核心概念

| 概念 | 定义 | 类比 |
|------|------|------|
| **ChatBI** | 通过 Text-to-SQL 将自然语言查询自动转换为可执行 SQL 语句，让非技术用户直接与数据库交互 | 像一个"翻译官"，把"上季度销量最高的产品"这种大白话翻译成数据库能懂的 SQL |
| **Tushare** | 免费、开源的 Python 财经数据接口库，主要用于获取国内金融市场数据 | 金融数据的"水龙头"，拧开就能拿到股票、指数、基金等数据 |
| **ARIMA** | 自回归积分滑动平均模型，经典时间序列预测方法，结合趋势(AR)、差分(I)和噪声(MA)建模 | 用过去股价的"惯性 + 波动 + 随机扰动"来猜未来走势 |
| **布林带 (Bollinger Bands)** | 由中轨(20日SMA)、上轨(中轨+2σ)、下轨(中轨-2σ)组成的技术分析工具 | 给股价画一条"安全通道"，碰到上轨可能太贵了，碰到下轨可能太便宜了 |
| **Prophet** | Facebook 开发的时序分析工具，通过突变点检测分段建模，每段趋势斜率可独立调整 | 一个能自动发现"拐点"的预测器，比如营销活动后销量突然飙升的那个时间点 |
| **SPC (统计过程控制)** | 使用控制图监控生产过程，及时发现系统性因素导致的潜在质量问题 | 面包店的"品控仪表盘"——每天抽样试吃，发现味道不对劲立刻调整烤箱 |

### A.2 项目架构全景

```
用户自然语言查询
     │
     ▼
┌──────────────────────┐
│   大模型 (Qwen3)      │  ← 理解意图、生成SQL、总结回复
│   (qwen-agent 框架)   │
└────────┬─────────────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
┌────────┐ ┌────────┐
│ ExcSQL │ │  Tavily │  ← MCP 工具
│ Tool   │ │   MCP   │
└───┬────┘ └───┬────┘
    │          │
    ▼          ▼
┌────────┐ ┌────────┐
│  MySQL  │ │  新闻   │
│ 数据库  │ │  搜索   │
└────────┘ └────────┘
    │
    ├── ARIMA 趋势预测 (5,1,5)
    ├── Boll带 异常检测 (20日+2σ)
    └── Prophet 周期性分析 (trend/weekly/yearly)
```

### A.3 关键技术点

**ChatBI 核心流程：**
1. **语义解析**：大模型理解用户意图，识别实体（表、字段）和操作（排序、聚合）
2. **SQL生成**：将语义结构转化为合规 SQL，需解决 schema linking（匹配用户术语与实际表名/字段名）
3. **交互优化**：支持模糊查询（如"卖得不好的产品"需定义阈值）和纠错

**数据采集流程：**
- Tushare 获取历史价格 → 保存 `.xlsx` → 按 `trade_date` 排序 → 生成建表 SQL → 导入 MySQL
- 注意：`trade_date` 建议用 `varchar(10)` 而非 `DATE` 类型，避免解析错误

**可视化优化：**
- 数据量大时，自动由柱状图切换为折线图
- 横坐标过密时选取部分点位显示
- 结果只有一行时，不展示统计信息和可视化图表
- 对比分析时，提取前5行+后5行让 AI 看到完整信息

**FAQ知识库（faq.txt）：**
- 通过 qwen-agent 的 files 能力作为上下文补充
- 涨跌幅计算：`(最后一天价格 - 第一天价格) / 第一天价格 * 100%`
- 查询时需按 `trade_date` 升序排序

**大模型 + 专业模型分工：**
- 大模型：与用户交互，理解问题并回答
- 专业模型：提供建模（ARIMA预测、Boll异常检测、Prophet周期性分析）

### A.4 判异准则速记

SPC 控制图 8 条判异准则：

| 编号 | 口诀 | 描述 |
|------|------|------|
| 1 | **1点界外** | 1个点子落在A区以外 |
| 2 | **9点单边** | 连续9点落在中心线同一侧 |
| 3 | **6连串** | 连续6点递增或递减 |
| 4 | **14升降** | 连续14点相邻点子总是上下交替 |

| 7 | **15C** | 连续15点落在中心线同两侧C区之内 |

#### 问答题要点

**第1题：**
- **大模型（Qwen3）**：与用户交互，理解自然语言问题，生成 SQL 查询，调用工具，对查询结果进行总结回复用户
- **专业模型**：提供具体的数学建模和分析
  - **ARIMA(5,1,5)**：对未来N天股价进行趋势预测，适合非平稳时间序列
  - **布林带(20日+2σ)**：检测超买超卖点，辅助风险控制——超过上轨可能超买，跌破下轨可能超卖
  - **Prophet**：分析 trend（趋势）、weekly（周周期）、yearly（年周期）规律，通过突变点检测自适应趋势变化
- **协作方式**：大模型决定"什么时候调用什么专业模型"，专业模型返回结果给大模型，大模型负责向用户解释

**第2题：**
- **步骤**：用户输入问题 → Qwen3 语义解析 → 生成 SQL（查询两支股票2024年第一天和最后一天的价格）→ ExcSQLTool 执行 SQL → df 展示前5行 → AI 计算涨跌幅 → 回复用户
- **发现的问题**：
  1. df 只展示了前10行，导致 AI 只看到1月2日和1月8日的数据，分析范围错误
  2. 涨跌幅计算结果不正确
- **faq.txt 解决方案**：
  - 在 faq.txt 中添加 Q&A 示例，明确涨跌幅计算公式
  - 强调 SQL 查询时需按 `trade_date` 升序排序，以便看到不同股票在同一天的价格对比
  - 通过 qwen-agent 的 files 能力，将 faq.txt 作为知识库上下文补充给大模型
  - 同时在 ExcSQLTool 中修改为提取前5行+后5行，让 AI 看到完整的日期范围
