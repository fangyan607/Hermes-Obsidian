# Obsidian LLM-WIKI 中医MD知识库AI检索应用方案
文档版本：V1.0
适配环境：Obsidian + LLM-WIKI
资源基础：全套中医MD笔记，涵盖中医基础、辨证论治、中药方剂、经络针灸、体质辨识、病症诊疗、养生调理等全类目
核心用途：智能辨证诊断、方药配伍查询、体质调理、经络实操、中医知识检索、康养方案制定

---

## 一、库内笔记标准化规整（适配LLM-WIKI索引）
### 1. 统一YAML头部元数据
所有中医笔记头部添加元标签，LLM-WIKI可按分类、技能、标签精准检索匹配
```yaml
---
title: 笔记名称
category: 中医基础/辨证论治/中药方剂/经络针灸/体质养生/内科杂病/骨伤妇科
skill_tag: 辨证诊断,方药查询,体质调理,经络取穴,食疗康养
level: 入门/进阶/临床实操
keywords: 阴阳五行,气虚,风寒感冒,当归,足三里
type: tcm
---
```

### 2. 规范文件夹目录结构
按诊疗逻辑划分目录，方便LLM-WIKI定向扫描检索
```
LLM-WIKI/
├─中医知识库/
   ├─01中医基础理论
   ├─02中医四诊望闻问切
   ├─03辨证论治体系
   ├─04中药材性味功效
   ├─05经典方剂配伍
   ├─06经络穴位针灸
   ├─07九种体质辨识
   ├─08各科病症诊疗
   ├─09食疗药膳调理
   └─10日常养生康养
```

### 3. 内容优化适配检索
1. 知识点拆分：单片段控制合适篇幅，拆分病机、症状、药方、取穴、调理方案独立模块
2. 内部链接关联：病症关联对应方剂、穴位、体质笔记，检索可联动调取
3. 内容规整统一：统一中医术语，删减冗余重复内容，保证语义标准一致

## 二、LLM-WIKI检索运行架构
整体流程：Obsidian中医MD笔记 → 框架索引扫描 → 向量缓存入库 → 目录/标签路由筛选 → 调用绑定SKILL技能 → 生成辨证、方药、调理应答

### 1. LLM-WIKI基础索引配置
1. 指定**中医知识库**总文件夹为专属检索数据源，划定扫描范围
2. 开启元数据、分类、技能标签联合索引，优先按skill_tag触发对应功能
3. 调整嵌入匹配阈值，兼顾检索速度与辨证精准度
4. 开启笔记溯源功能，回答可跳转查看原始MD笔记原文

### 2. 四类检索匹配规则
- 目录定向检索：按病症、方药、经络类目快速锁定相关笔记
- 标签技能检索：识别技能标签，自动匹配对应诊疗能力模块
- 语义智能检索：理解身体症状、不适表现，跨笔记整合辨证依据
- 层级检索区分：按难度标签，区分科普养生与临床辨证内容输出

## 三、中医专属SKILL技能模块封装
技能绑定对应目录与标签，输入指令即可自动调取库内资料作答

### Skill1 中医基础理论检索
触发指令：五行阴阳、脏腑学说、气血津液、病因病机
关联目录：01中医基础理论
能力：术语释义、中医原理讲解、病因机理分析

### Skill2 四诊辨证诊断
触发指令：症状判断、舌苔脉象、寒热虚实、病症辨证
关联目录：02四诊望闻问切、03辨证论治体系
能力：依据症状辨析证型，判断身体失衡问题

### Skill3 中药材查询应用
触发指令：药材功效、性味归经、禁忌搭配、药材用法
关联目录：04中药材性味功效
能力：药材信息解读、适用症状、配伍宜忌说明

### Skill4 经典方剂调取配伍
触发指令：药方查询、方剂功效、加减配伍、主治病症
关联目录：05经典方剂配伍
能力：匹配对应证型方剂，讲解组方思路与使用场景

### Skill5 经络穴位实操应用
触发指令：穴位取穴、针灸按摩、经络疏通、疼痛调理
关联目录：06经络穴位针灸
能力：定位穴位、说明功效、给出按摩/艾灸实操方法

### Skill6 九种体质辨识调理
触发指令：体质判断、气虚阳虚、湿热痰湿、体质改善
关联目录：07九种体质辨识
能力：判定个人体质，输出专属养护方向

### Skill7 各科常见病症诊疗
触发指令：感冒咳嗽、脾胃不适、失眠腰腿不适、妇科小问题
关联目录：08各科病症诊疗
能力：辨证病因，给出合规中医调理思路

### Skill8 食疗药膳日常康养
触发指令：食补方子、四季养生、饮食禁忌、作息养护
关联目录：09食疗药膳调理、10日常养生康养
能力：搭配食疗方案，制定四季与生活养生建议

## 四、三大实际使用模式
### 模式1 中医知识问答检索
提问理论、药材、方剂、经络问题 → LLM-WIKI匹配目录标签 → 调取笔记内容整合回答，支持溯源原文

### 模式2 症状辨证调理
描述身体不适、舌苔睡眠饮食情况 → 自动辨证分型 → 输出证型判断+方药/穴位/饮食综合调理方案

### 模式3 体质康养定制
根据身形、怕冷怕热、日常状态判定体质 → 调取对应养护笔记，生成长期养生调整方案

## 五、LLM-WIKI专用提示词模板
### 通用中医知识检索提示词
```
仅依据Obsidian LLM-WIKI内中医MD笔记内容作答，不编造域外偏方与不实理论。
1. 识别问题匹配对应分类与skill标签，调取库内原有知识点
2. 辨证、方药内容严谨规范，养生内容通俗易懂
3. 复杂内容分层说明，实操类标注清晰步骤
4. 重要内容可关联对应笔记名称，方便溯源查阅
```

### 症状辨证诊断提示词
```
启用中医辨证诊断SKILL，调取四诊、辨证论治相关笔记。
根据用户描述的身体症状、脉象舌苔、作息饮食综合辨析证型，
客观分析病机成因，给出合规中医判断与调理方向，不做医疗确诊定论。
```

### 方药穴位调理提示词
```
启用方药穴位调理SKILL，检索中药材、方剂、经络穴位笔记。
结合对应证型匹配合适药材、方剂、按摩艾灸穴位，
说明用法用量基础参考、使用注意事项与搭配禁忌。
```

### 体质养生定制提示词
```
启用体质康养SKILL，依据九种体质笔记内容。
综合用户身体特征判定体质类型，针对性给出饮食、运动、作息、四季养护方案。
```

## 六、知识库迭代优化方案
1. 笔记增量更新：新增中医资料归入对应目录，框架自动刷新索引收录
2. 技能细分拓展：拆分儿科、风湿、亚健康等细分技能标签，缩小检索范围
3. 症状案例沉淀：从笔记提取典型病案，单独归档纳入检索库，提升辨证参考性
4. 定期索引维护：笔记修改增减后，手动刷新LLM-WIKI向量缓存，保证数据同步
5. 会话记忆联动：开启上下文记忆，连续问诊可衔接过往症状，连贯辨证调理

## 七、完整落地部署步骤
1. 按照设定目录，将全部中医MD笔记分类移入对应文件夹
2. 批量为所有笔记补全YAML元数据标签
3. 在LLM-WIKI中选定中医知识库文件夹，完成初始化索引构建
4. 将8项中医SKILL技能与目录、标签一一绑定关联
5. 导入专用提示词模板，设定检索应答规范
6. 测试问诊、查药、取穴、体质判定各项功能，微调匹配精度
7. 日常维护更新笔记，定期刷新索引即可正常使用
