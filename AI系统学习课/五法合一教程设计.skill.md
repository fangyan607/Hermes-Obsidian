# 🎯 五法合一教程设计 Skill

> 将任意技术主题转化为融合五种学习方法的进阶教程：番茄工作法×费曼学习法×金字塔原理×刻意练习×系统思考
>
> 设计原则：五法不是简单叠加，而是在时间结构、理解验证、知识组织、技能提升、全局框架五个维度上形成**增强回路**

---

## 适用场景

- 将现有教程/课程按五法体系重新改写
- 从零设计一个完整的技能学习路径
- 将碎片知识整合为结构化的"刻意练习"序列
- 为复杂技术主题设计阶梯式学习方案

---

## 核心方法论

### 五法的角色分工

| 方法 | 解决什么问题 | 在教程中的角色 | 关联GBrain实体 |
|:-----|:------------|:--------------|:--------------|
| 🍅 **番茄工作法** | 时间结构与注意力管理 | 每25分钟为一个单元剂量，精准控制认知负荷 | [[GBrain/学习方法/时间管理]] |
| 🧠 **费曼学习法** | 理解的真实性检验 | 每个概念后用三句话复述，逼学习者真正理解 | [[GBrain/学习方法/刻意练习]] |
| 🔺 **金字塔原理** | 知识的结构化组织 | 结论先行，层层支撑，形成可检索的思维框架 | [[GBrain/学习方法/金字塔原理]] |
| 🎯 **刻意练习** | 技能的系统性提升 | 三大循环（模仿→变式→创造），突破舒适区 | [[GBrain/学习方法/刻意练习]] |
| 🔄 **系统思考** | 全局理解与关联能力 | 反馈回路、杠杆点、层次关系、知识全景图 | [[GBrain/创造性思维/系统思考]] |

### 四条核心增强原则（每条都是两个方法的化学反应）

| 原则 | 公式 | 效果 | 说明 |
|:-----|:-----|:-----|:------|
| **模块化专业技能** | 刻意练习 + 系统思考 | 每个知识块都是可拆装的专业技能 | 刻意练习的专注训练 + 系统思考的全局框架 → 不孤立的知识点 |
| **清晰结构呈现** | 金字塔原理 + 设计思维 | 输出既有深度又有可理解性 | 金字塔的纵向逻辑 + 设计思维的用户中心视角 → 结构清晰+易于接受 |
| **心流创作状态** | 深度工作 + 创造力 | 在专注中产生"啊哈时刻" | 深度工作的无干扰专注 + 创造力的发散联想 → 高质量创意产出 |
| **有纪律的自由空间** | 时间管理 + 番茄工作法 | 为重要的事留出专属时间 | 时间管理的优先级筛选 + 番茄工作法的结构化执行 → 不是填满每分鐘 |

## 工作流程

### Phase 1：源材料分析（1-2🍅）

1. **理解内容**：通读源材料，提取 3-5 个最核心概念
2. **识别知识类型**：哪些是入门级？哪些是进阶级？
3. **映射GBrain知识**：关联学习方法域和创造性思维域的实体，作为理论支撑
4. **诊断问题**：原材料的结构问题、遗漏的关键连接、缺失的练习环节

> ✋ **费曼自测**：我能否用三句话向一个外行说清"这个教程到底教什么"？

### Phase 2：金字塔结构设计（1🍅）

1. **定义最终结论**：学完这门课后，学习者能做到什么？
2. **搭建金字塔**：结论 → 3-5 个主论据 → 每个论据的子论据（MECE原则）
3. **SCQA序言**：用情境-冲突-问题-答案引出教程的核心命题
4. **检查纵向逻辑**：每一层是否是对上一层的直接支撑？

**输出**：金字塔结构大纲（结论层→论据层→子论据层）

### Phase 3：番茄单元拆分（1🍅）

1. **确定总番茄数**：按内容的广度和深度估算
2. **拆分节奏**：
   - 00:00-03:00 → 🔺 金字塔开场（结论先行）
   - 03:00-18:00 → 🔬 核心概念与论据（嵌入费曼自测）
   - 18:00-20:00 → 🔄 系统连接（反馈回路、知识全景图定位）
   - 20:00-22:00 → 🧠 费曼三句话复述
   - 22:00-25:00 → 🎯 刻意练习（模仿/变式/创造）
   - 25:00-30:00 → ☕ 休息
3. **设计练习层级**：
   - 模仿阶段：跟着做（前1/3教程）
   - 变式阶段：改参数换场景（中间1/3）
   - 创造阶段：自主设计（后1/3）

### Phase 4：系统思考嵌入（0.5🍅）

1. **为每个章节设计"系统连接图"**：这个知识点在全局中的位置
2. **标注反馈回路**：A如何影响B，B又如何反过来影响A
3. **识别杠杆点**：哪些概念一旦理解就能产生乘数效应？
4. **制作知识全景图**：教程结束时，学习者能看到完整的知识版图

### Phase 5：输出与归档（0.5🍅）

1. **生成教程文件**：按金字塔结构组织，每章结论先行
2. **创建洞见记录**：将设计过程中的关键决策归档到 Claude_Memory
3. **更新MEMORY.md**：在 Layer 3 记录版本变更
4. **关联GBrain**：在教程中嵌入指向相关GBrain实体的wikilink

## 设计原则

### 每段内容自检三问

写每一段时，问自己：

1. **金字塔检验**：这段内容是对上层结论的直接支撑吗？去掉它，论证链条会断吗？
2. **费曼检验**：这段内容学习者能用大白话复述吗？如果不能，重写。
3. **系统检验**：这段内容和其他知识点的连接是什么？学习者知道它"属于哪里"吗？

### 必须避免的

| 陷阱 | 后果 | 解决方案 |
|:-----|:-----|:---------|
| 只讲"是什么"不讲"为什么" | 学习者无法迁移知识 | 每个概念后加"为什么重要"段落 |
| 没有刻意练习的纯理论 | 学了就忘 | 每番茄结尾嵌入三级练习 |
| 各章节孤立无连接 | 知识碎片化 | 每章末尾加"系统连接"段落 |
| 没有明确结论的平铺直叙 | 学习者抓不住重点 | 每个章节以"🔺 结论先行"开头 |

### 必须包含的

| 元素 | 位置 | 作用 |
|:-----|:-----|:-----|
| 🔺 结论先行 | 每个章节开头 | 建立心智框架 |
| ✋ 费曼自测 | 每个核心概念之后 | 验证理解 |
| 🔄 系统连接 | 每章后半段 | 形成知识网络 |
| 🎯 刻意练习 | 每章结尾 | 技能固化 |
| 📊 自检清单 | 教程末尾 | 可量化的掌握度 |
