## 番茄钟1：什么是 Cloudflare？

### 1.1 用大白话理解 Cloudflare

**Cloudflare 是什么？**

想象你的网站是一座**商店**，用户是**顾客**：

| 没有 Cloudflare | 有 Cloudflare |
|:----------------|:--------------|
| 顾客直接走进商店 | 顾客先经过一个**超级保安亭** |
| 商店自己处理所有请求 | 保安亭帮忙处理很多事情 |
| 商店暴露真实地址 | 商店地址被隐藏保护 |
| 遇到攻击商店直接受影响 | 保安亭帮你挡住攻击 |

**Cloudflare 就是那个"超级保安亭"**，它在用户和你的服务器之间，提供：
- 🛡️ 安全防护（挡住攻击）
- ⚡ 加速访问（就近服务）
- 🔒 隐藏真实地址（保护服务器）

**核心思维转变：**

```
传统模式：用户 ──────────→ 你的服务器
                  直连

Cloudflare模式：用户 → Cloudflare边缘节点 → 你的服务器
                      安全+加速+隐藏
```

### 1.2 Cloudflare 解决什么问题？

| 问题 | Cloudflare 解决方案 |
|:-----|:--------------------|
| **网站太慢** | CDN 加速，就近访问 |
| **担心被攻击** | WAF + DDoS 防护 |
| **没有 HTTPS** | 免费 SSL 证书 |
| **服务器暴露** | 隐藏真实 IP |
| **域名解析慢** | 全球最快的 DNS |
| **需要无服务器** | Workers 边缘计算 |
| **静态网站托管** | Pages 免费托管 |

## 番茄钟3：架构原理 - 边缘网络

### 3.1 什么是边缘计算？

**传统架构 vs 边缘架构：**

```
传统架构：
用户（北京）──→ 服务器（美国）
         延迟 ~200ms

边缘架构：
用户（北京）──→ Cloudflare边缘节点（北京）──→ 源站（美国）
         延迟 ~20ms + 缓存命中 = 0ms 到源站
```

### 3.2 Cloudflare 全球网络

**网络规模：**

| 指标 | 数据 |
|:-----|:-----|
| 数据中心数量 | 310+ |
| 覆盖国家 | 120+ |
| 处理流量 | 20%+ 全球互联网 |
| 平均延迟 | <50ms |

### 3.3 边缘节点的作用

```
┌─────────────────────────────────────────────────────────────┐
│                    边缘节点工作原理                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户请求                                                    │
│      ↓                                                      │
│  最近的边缘节点（北京）                                       │
│      ├── 检查缓存：有 → 直接返回                             │
│      ├── 检查安全：攻击 → 拦截                               │
│      ├── 检查规则：需要转发 → 源站                           │
│      └── 处理 Workers：执行边缘函数                          │
│      ↓                                                      │
│  返回响应给用户                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 Anycast 网络

**什么是 Anycast？**

同一个 IP 地址在全球多个节点广播，用户自动连接到最近的节点。

```
Cloudflare IP: 104.16.0.1

北京用户 → 北京节点（104.16.0.1）
纽约用户 → 纽约节点（104.16.0.1）
东京用户 → 东京节点（104.16.0.1）

同一个 IP，自动路由到最近的节点
```

## 番茄钟4：核心工作流程

### 4.1 请求处理流程

```
用户输入 www.example.com
        ↓
    DNS 查询
        ↓
    返回 Cloudflare IP
        ↓
    用户请求到达边缘节点
        ↓
    ┌─────────────────────┐
    │  Cloudflare 处理    │
    │  ├── 安全检查       │
    │  ├── 缓存检查       │
    │  ├── WAF 规则       │
    │  └── Workers 执行   │
    └─────────────────────┘
        ↓
    缓存命中？──→ 是 → 直接返回
        │
        否
        ↓
    转发到源站
        ↓
    返回响应 + 缓存
        ↓
    用户收到响应
```

### 4.2 各层级的职责

| 层级 | 职责 | 产品 |
|:-----|:-----|:-----|
| **DNS 层** | 域名解析，返回边缘节点 IP | DNS |
| **安全层** | DDoS 防护、WAF、Bot 检测 | Security |
| **缓存层** | 静态资源缓存、加速访问 | CDN |
| **计算层** | 边缘函数执行 | Workers |
| **源站层** | 动态内容生成 | 你的服务器 |

### 4.3 代理模式 vs 仅 DNS

| 模式 | 说明 | 功能 |
|:-----|:-----|:-----|
| **代理模式（橙色云）** | 流量经过 Cloudflare | 安全+加速+缓存 |
| **仅 DNS（灰色云）** | 只做 DNS 解析 | 仅解析，无其他功能 |

**图标含义：**
- 🟠 橙色云朵 = 代理开启（推荐）
- ⚪ 灰色云朵 = 仅 DNS

## 番茄钟5：账号注册与控制台

### 5.1 注册 Cloudflare 账号

**注册步骤：**

1. 访问 https://dash.cloudflare.com/sign-up
2. 输入邮箱和密码
3. 验证邮箱
4. 选择免费计划（Free）

**账号安全建议：**

| 建议 | 说明 |
|:-----|:-----|
| 使用强密码 | 12位以上，包含大小写数字符号 |
| 启用 2FA | 两步验证，提高安全性 |
| 记录 API Token | 用于 CI/CD 自动化 |

### 5.2 控制台概览

**登录后界面：**

```
Cloudflare Dashboard
├── 概览（Overview）
│   └── 账号信息、用量统计
├── 网站（Websites）
│   └── 管理域名
├── 分析（Analytics）
│   └── 流量、安全、性能
├── Workers & Pages
│   └── 无服务器函数和静态网站
├── R2
│   └── 对象存储
├── Zero Trust
│   └── 零信任网络访问
└── 设置（Settings）
    └── 账号、API、团队
```

### 5.3 API Token 创建

**创建步骤：**

1. 进入 My Profile → API Tokens
2. 点击 "Create Token"
3. 选择模板或自定义权限
4. 复制并保存 Token

**常用 Token 权限：**

| 用途 | 需要权限 |
|:-----|:---------|
| DNS 编辑 | Zone.DNS Edit |
| Workers 部署 | Account.Workers Scripts Edit |
| Pages 部署 | Account.Cloudflare Pages Edit |

> ✋ **费曼自测**：创建一个具有 DNS 编辑权限的 API Token。

### 6.2 常见域名注册商 NS 修改
