## Section A：费曼学习法复习文档

### 一、核心概念

**VLM (Vision-Language Model)**：视觉语言模型，能够同时理解图像/视频内容和文本信息，实现跨模态的理解与推理。典型应用包括：从医疗图像中抽取结构化病历信息、车辆损伤评估、危险驾驶行为识别等。

**多模态理解**：模型同时处理来自不同模态（视觉、文本、音频、语音等）的数据，融合各模态信息进行综合理解与推理。

### 二、VLM在保险行业的典型应用

| 场景 | 输入 | 输出 | 目的 |
|------|------|------|------|
| 中文病历抽取 | 医疗报告图片 | 姓名、性别、年龄、CT号、临床诊断、影像学表现等结构化信息 | 健康保险理赔评估 |
| 日文病历抽取 | 诊断书图片 | 疾病名称、入院时间、请假时间 | 跨国保险理赔 |
| 法文病历抽取 | 法文医疗报告(RAPPORT MEDICAL) | 患者信息、住院日期、病史、生命体征、体格检查等 | 跨国保险理赔 |
| 德文病历抽取 | 德文医疗证明(Ärztliche Bescheinigung) | 诊断结果(cALL白血病)、治疗方案、总治疗时间 | 跨国保险理赔评估 |
| 韩文病历抽取 | 韩国诊断书图片 | 患者信息、疾病名称、受伤原因、受伤部位、治疗期间 | 健康保险理赔 |
| 车辆里程表识别 | 仪表盘图片 | 总里程、当前速度、时间、温度、挡位 | 评估车辆使用情况与保险风险 |

### 三、Qwen-VL微调

Qwen-VL的微调数据格式采用结构化JSON，包含`messages`（对话历史）和`images`（图片路径）两个主要字段。

- `messages`字段包含`role`（user/assistant）和`content`（文本内容，图片用`<image>`占位）
- `images`字段列出对话涉及的图片路径列表

示例结构：
```json
{
  "messages": [
    {"content": "<image>Who are they?", "role": "user"},
    {"content": "They're Kane and Gretzka from Bayern Munich.", "role": "assistant"}
  ],
  "images": ["mllm_demo_data/1.jpg", "mllm_demo_data/1.jpg"]
}
```

### 四、InternVideo2视频理解模型

**设计哲学**：渐进式训练方法，将视频编码器规模扩展到60亿参数。

**三个训练阶段**：

| 阶段 | 训练任务 | 目标 |
|------|----------|------|
| Stage 1 | 掩蔽视频建模（通过未掩蔽重建） | 捕获视频的时空结构 |
| Stage 2 | 跨模态对比学习（与其他模态语义对齐） | 对齐视频与音频、文本等模态 |
| Stage 3 | 下一个token预测 | 增强开放式对话能力 |

**数据规模**：预训练使用5千万视频-音频-语音-文本对(InternVid2)和3亿图像-文本对。

### 五、InternVideo2模型表现

**动作识别**：在Kinetics-400上InternVideo2-6B使用16帧即达92.1% Top-1 Accuracy，超越此前SOTA。在MiT数据集上达51.2%（超CoCa-g 2.2%），在SSv2上达77.5%（超MVD的77.3%），在ActivityNet达95.9%，HACS达97.0%。

**视频问答基准测试（InternVideo2.5）**：多个短视频与长视频基准中性能领先，尤其在长视频理解（MLVU、LongVideoBench、LVBench）上大幅超越此前开源模型。

**对比其他模型**：
- 时间动作识别：Gemini Pro和InternVideo2-Chat能准确描述动作，GPT-4V会产生幻觉
- 混淆动作识别：视频中误导性动作，Gemini Pro给出错误答案，GPT-4V识别了误导但未给正确答案，InternVideo2-Chat给出正确答案
- 视频对象时间识别：识别人按时间顺序取字母，Gemini Pro识别4个但顺序完全相反，GPT-4V只识别3个且结果错误，InternVideo2-Chat错误最少且顺序正确
- 事件计数：GPT-4V和InternVideo2-Chat均正确捕捉动作时间
- 视觉语言导航：GPT-4V和InternVideo2-Chat理解指令并做出正确决策，Gemini Pro容易产生幻觉

### 六、MinerU文档解析工具

**定位**：专注于高效解析PDF、网页和电子书，转换为Markdown或JSON格式，由上海人工智能实验室OpenDataLab团队开发。

**核心技术栈**：
- 布局检测：基于LayoutLMv3微调
- 公式识别：YOLOv8检测 + UniMERNet模型转换LaTeX
- OCR增强：PaddleOCR（支持84种语言）

**使用方式**：
1. **在线使用**：mineru.net
2. **客户端**：mineru.net/client
3. **API**：申请Token → 提交PDF获取task_id → 通过task_id获取解析结果（含full.md、full.json等）
4. **私有化部署**：`pip install magic-pdf[full]` → 下载models → `magic-pdf -p xxx.pdf -o ./output`

#### 问答题要点

**1. InternVideo2的三个训练阶段：**
- **Stage 1（掩蔽视频建模）**：对视频帧进行掩蔽，通过未掩蔽部分重建掩蔽区域，捕获视频的时空结构信息，使编码器理解视频内部的空间布局和时间连续性。
- **Stage 2（跨模态对比学习）**：将视频表示与音频、文本等其他模态进行语义对齐，使用对比学习拉近同一视频在不同模态下的表示，使编码器具备跨模态理解能力。
- **Stage 3（下一个token预测）**：接入LLM进行语言建模训练，以视频特征为条件预测下一个文本token，增强模型的开放式对话和推理能力，使其能基于视频内容进行自然语言问答。

**2. 车险理赔自动化系统关键场景：**
- **里程表识别**：输入仪表盘图片，输出总里程、当前速度等信息，评估车辆使用强度
- **车辆承保分析**：输入多角度车辆照片，输出品牌型号、车辆状况、保险建议，支持承保决策
- **危险驾驶行为识别**：输入车内摄像头照片或行车记录仪视频，输出是否存在使用手机、超速等危险行为
- **车辆损失评估**：输入事故现场照片，输出损坏部位、损坏程度、维修建议，辅助定损
- **车辆事故要素提取**：输入事故照片，输出JSON格式（道路状况、天气、受损部位、损害程度、损害分类等），标准化事故记录
- **车辆一致性校验**：输入事故前后车辆照片对比，输出是否为同一辆车，用于防欺诈检测
