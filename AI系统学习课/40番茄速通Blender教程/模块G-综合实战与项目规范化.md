## \U0001F34536 规范化制作流程（25分钟）

### 费曼输入（概念解释）

**为什么需要规范化？**

规范化不是"限制自由"，而是"保护未来的自己"：

| 场景 | 没有规范 | 有规范 |
|:----|:--------|:------|
| 一个月后找文件 | "这个 v23_final_true_final.blend 是啥？" | "城堡_场景_v03.blend —— 清晰明确" |
| 团队协作 | "这个材质叫什么？我找不到" | "Table_Wood —— 看一眼就知道" |
| 客户要修改 | "我忘了用的哪些贴图..." | "_assets/textures/ 目录里分类存放" |
| 导出给程序 | "这个模型面数多少？比例对吗？" | "LOD 分级命名，按规范导出" |

**文件管理规范（推荐结构）：**

```
project_root/
├── project_v01.blend              # 主文件（按版本命名）
├── _assets/                       # 外部资产
│   ├── textures/                  # 贴图（按类型分类）
│   │   ├── Table_Diffuse_2K.png
│   │   ├── Table_Normal_2K.png
│   │   └── Table_Roughness_2K.png
│   ├── hdri/                      # 环境贴图
│   │   └── sunset_park_4K.hdr
│   ├── models/                    # 引用模型
│   │   └── chair_reference.fbx
│   └── references/                # 参考图
│       └── concept_art_01.jpg
├── _exports/                      # 导出文件夹
│   ├── fbx/                       # FBX 格式
│   ├── obj/                       # OBJ 格式
│   ├── glb/                       # GLB 格式
│   └── alembic/                   # 缓存格式
├── _renders/                      # 渲染输出
│   ├── preview/                   # 预览小图
│   └── final/                     # 最终渲染
│       └── project_final_4K.png
├── _scripts/                      # 自定义 Python 脚本
│   └── batch_export.py
└── _blend_backups/                # 文件备份（版本号递增）
    └── project_v01_backup.blend
```

> **为什么文件夹名前加下划线 `_`？** 这样在文件管理器中它们会排在前面，与项目主文件区分开。

**命名规范：**

| 类型 | 格式 | 示例 |
|:----|:----|:----|
| 项目文件 | `项目名_类型_版本.blend` | `城堡_场景_v03.blend` |
| 物体 | `类型_名称` | `Light_Key`、`Mesh_Table`、`Curve_Cable` |
| 材质 | `物体名_材质名` | `Table_Wood`、`Screen_Glass` |
| 贴图 | `物体名_类型_分辨率` | `Table_Diffuse_2K`、`Table_Normal_2K` |
| 备份 | `项目名_版本_backup.blend` | `城堡_v03_backup.blend` |

**版本管理策略：**

```
大版本（里程碑式保存）
  v01 → v02 → v03 ...
  每次完成一个重要阶段后保存

小版本（增量保存）
  _v03a → _v03b → _v03c
  同一个大版本内的多次迭代

自动备份
  Blender 文件菜单 → Save Version
  自动创建带时间戳的备份文件

Git + Blender
  需要配合 Git LFS（Large File Storage）
  .blend 是二进制文件，普通 Git 无法做 diff
```

### 刻意练习（具体操作步骤 + 代码/命令）

**练习 1：创建项目文件夹结构**

```bash

mkdir -p my_project/_assets/references

mkdir -p my_project/_renders/preview
mkdir -p my_project/_renders/final
mkdir -p my_project/_scripts
mkdir -p my_project/_blend_backups
```

**练习 2：命名规范实践**

在 Blender 中创建一个简单的场景，练习规范命名：
1. 添加一个立方体 → 命名为 `Mesh_Box`
2. 添加一个柱体 → 命名为 `Mesh_Column`
3. 添加一个相机 → 命名为 `Camera_Main`
4. 添加一个灯光 → 命名为 `Light_Key`
5. 创建材质 → 命名为 `Box_Metal`、`Column_Stone`
6. 保存文件 → 命名为 `练习_场景_v01.blend`

**练习 3：版本管理练习**

1. 保存文件为 `练习_场景_v01.blend`
2. 做三次修改，每次用 Save Version 保存
3. 再保存为 `练习_场景_v02.blend`
4. 检查备份目录，确认所有版本都在

## \U0001F34537 综合实战Project A：场景全流程（上）（25分钟）

### 费曼输入（概念解释）

**项目目标：** 创建一个"科幻控制台"场景，从零到最终渲染输出。

**为什么是"科幻控制台"？**
- 涵盖硬表面建模（控制台主体）
- 包含发光材质（屏幕/按钮）
- 有机械细节（按钮、开关、线缆）
- 适合科技感灯光氛围

**整体流程概览：**

```
第一阶段（🍅37前半）：参考与布局
第二阶段（🍅37后半）：建 模
第三阶段（🍅38上半）：材质与UV
第四阶段（🍅38下半）：灯光与渲染
第五阶段（🍅38收尾）：后期合成
```

**第一阶段：参考与布局**

任何复杂场景都是从"方块布局"开始的。核心原则：
1. 先搭积木再细化——不要一开始就扣细节
2. 参考图是"翻译"不是"抄袭"——把 2D 参考变成 3D 方案
3. 相机定构图 = 画框子——框定了创作范围

**第二阶段：建模**

硬表面建模技术栈（回顾模块 A）：

| 技术 | 用途 | 快捷键/修改器 |
|:----|:----|:------------|
| 细分修改器 | 圆滑表面 | Subdivision Surface |
| 布尔运算 | 挖槽/打孔 | Boolean Modifier |
| 倒角修改器 | 边缘圆角 | Bevel Modifier |
| 阵列修改器 | 重复复制 | Array Modifier |
| 曲线 + Bevel | 管状路径 | Curve + Geometry Bevel |

### 刻意练习（具体操作步骤 + 代码/命令）

**第一阶段：参考与布局**

```
1. 创建新 .blend 文件，按规范命名 "科幻控制台_v01.blend"
2. 创建资产文件夹结构（按🍅36的规范）
3. 收集参考图：
   - 打开浏览器，搜索 "sci-fi console concept"
   - 或 "科幻控制台 设计"
   - Pinterest / ArtStation / Google Images
   - 下载 3-5 张参考图到 _assets/references/
4. 搭建设计布局：
   - Shift+A → Mesh → Cube 作为控制台主体（缩放 4x1x1.5）
   - Shift+A → Mesh → Cylinder 作为屏幕面板（旋转 90°，放主体上方）
   - Shift+A → Mesh → Plane 作为地面
   - 用基本几何体确定所有物体的位置和比例
5. 添加相机：
   - 数字键盘 0 进入相机视图
   - N 键打开 Sidebar → View 调整相机
   - 焦距设置 35mm（标准透视）
   - 调整角度：前侧方 45°，略微俯视
```

**第二阶段：建模（硬表面）**

```
# 控制台主体
1. 选择布局用的 Cube → Tab 进入编辑模式
2. Ctrl+R 添加环切线，定义面板分区
3. 添加修改器：
   - Bevel（宽度 0.02m，段数 2）
   - Subdivision Surface（级别 2）
   - Solidify（厚度 0.05m）
4. 布尔挖槽：
   - Shift+A 添加 Cube，缩放到凹槽大小
   - 给主体添加 Boolean Modifier（差集）
   - 指向刚才创建的 Cube
   - 隐藏 Cube（H 键）

# 屏幕面板
1. 选择布局用的 Cylinder
2. Tab 进入编辑模式
3. 选中正面环面 → I 键内插面
4. E 键挤出内凹（-0.05m）
5. 添加 Bevel 和 Subdivision 修改器

# 按钮和开关
1. Shift+A → Cylinder，缩放到按钮大小
2. 添加 Array 修改器（Count: 6，沿 X 轴偏移 0.3m）
3. 调整单个按钮：顶面倒角 + 底面环切

# 线缆管道
1. Shift+A → Curve → Bezier Curve
2. 编辑曲线控制点，弯曲成管道路径
3. 在 Curve 属性 → Geometry → Bevel（Depth: 0.05m）
4. 调整曲线形状直到满意
```

**检查拓扑（重要！）：**

```
1. Tab 进入编辑模式
2. 选中所有面（A 键）
3. 顶部菜单 Select → Select All by Trait → Non Manifold
4. 修复发现的任何非流形几何
5. 开启 X-Ray（Alt+Z）检查内部面
```

## \U0001F34538 综合实战Project A：场景全流程（下）（25分钟）

### 费曼输入（概念解释）

**第三阶段：材质与UV**

材质的核心是 Principled BSDF 节点。对于科幻控制台场景，我们需要三种主要材质：

| 材质 | 关键参数 | 视觉效果 |
|:----|:--------|:--------|
| 金属面板 | 金属度 0.8，粗糙度 0.3 | 拉丝金属质感 |
| 屏幕发光 | Emission + 自发光颜色 | 科技感蓝色光晕 |
| 玻璃按钮 | 透射 1.0，粗糙度 0.05 | 半透明发光按钮 |

**第四阶段：灯光与渲染**

科幻场景的灯光逻辑不同日景——不是模拟太阳，而是创造氛围：

```
传统日景：太阳为主 → 强调真实感
科幻场景：人造光为主 → 强调氛围感
```

核心灯光方案：
- HDRI 环境光——选择暗色调科技感
- 三点布光——突出主体
- 屏幕自发光——作为场景的氛围补光

**第五阶段：后期合成**

合成的核心是"让好变成更好"。Blender 的 Compositor 提供了节点化后期处理：

```
渲染输出 → 调色 → 辉光 → 色差 → 暗角 → 最终输出
```

### 刻意练习（具体操作步骤 + 代码/命令）

**第三阶段：材质与UV**

```
# 1. 金属面板材质（控制台主体）
1. 选中控制台主体
2. 新建材质，命名为 "Console_Metal"
3. 设置 Principled BSDF 参数：
   - Base Color: #445566（灰蓝色金属）
   - Metallic: 0.8
   - Roughness: 0.3
   - Normal: 添加 Noise Texture 做细微表面细节

# 2. 屏幕发光材质
1. 选中屏幕面板
2. 新建材质，命名为 "Screen_Glow"
3. 设置：
   - Base Color: #00D4FF（科技蓝）
   - Emission Strength: 5-10
   - 可选：添加渐变纹理做屏幕内容效果

# 3. 按钮材质
1. 选中所有按钮
2. 新建材质，命名为 "Button_Glass"
3. 设置：
   - Base Color: 不同按钮设不同颜色
   - Metallic: 0.0
   - Roughness: 0.05
   - Transmission: 1.0（全透射）
   - IOR: 1.45
   - Emission: 微弱自发光 + 颜色

# 4. UV展开
1. 选中控制台主体，Tab 编辑模式
2. A 全选面 → U → Smart UV Project
3. 切换到 UV Editing 工作区
4. 调整 UV 岛位置，最大化利用纹理空间
5. 如有贴图需求：
   - 在 Image Editor 新建纹理
   - 使用 Texture Paint 绘制细节
```

**第四阶段：灯光与渲染**

```
# 1. HDRI 环境光
1. 切换到 Shading 工作区
2. World 节点：添加 Environment Texture
3. 选择暗色调 HDRI（或在 Poly Haven 搜索 "studio" 或 "night"）
4. Strength: 0.5（调暗，突出人工光）

# 2. 三点布光
1. 主光（Key Light）：
   - Shift+A → Light → Area Light
   - 位置：相机右侧上方 45°
   - Strength: 800W
   - Color: 暖白 #FFEECC
2. 补光（Fill Light）：
   - Shift+A → Light → Area Light
   - 位置：相机左侧 30°
   - Strength: 300W
   - Color: 冷白 #CCEEFF
3. 轮廓光（Rim Light）：
   - Shift+A → Light → Area Light
   - 位置：控制台背后上方
   - Strength: 500W
   - Color: #4488FF（科技蓝）

# 3. 屏幕氛围光
（屏幕自发光材质会自动照亮周围，无需额外光源）

# 4. 体积雾（可选增强氛围）
1. 切换到 Shading 工作区 → World 节点
2. Add → Shader → Volume Scatter
3. 连接 Color（深蓝）和 Density（0.01）
4. 混合原 Background

# 5. 渲染设置
1. 切换到 Render Properties
2. Render Engine: Cycles
3. Samples: 1024
4. Denoising: 开启（OptiX）
5. Output Properties:
   - Resolution: 3840×2160 (4K)
   - Output Format: PNG
   - Color: sRGB
6. F12 渲染 → 保存到 _renders/final/
```

**第五阶段：后期合成**

```
# 在 Compositor 中调色
1. 切换到 Compositing 工作区
2. 勾选 Use Nodes
3. 已有 Render Layers → Composite 节点链

# 添加辉光（Glare）
1. Shift+A → Filter → Glare
2. 连接到 Render Layers 输出和 Composite 输入之间
3. Glare Type: Fog Glow
4. Threshold: 0.8（只有亮部发光）
5. Mix: 0.3（混合度）

# 添加色差
1. Shift+A → Distort → Lens Distortion
2. Distortion: 0.0
3. Dispersion: 0.02（轻微色散）
4. 连接到输出链

# 调色
1. Shift+A → Color → RGB Curves
2. 轻微 S 曲线增加对比度
3. 蓝调偏色营造科技感

# 最终渲染
1. 调整所有参数到满意
2. F12 渲染最终版本
3. Ctrl+S 保存合成设置为 Compositor 预设
4. 输出到 _renders/final/

# 对比检查
- 关闭合成节点 → 渲染一次
- 开启合成节点 → 再渲染一次
- 对比有/无合成的效果差异
```

### ✅ 完成标准

- [ ] 完成了金属面板材质（金属度 0.8，粗糙度 0.3）
- [ ] 完成了屏幕发光材质（Emission + 自发光）
- [ ] 完成了玻璃按钮材质（透射 + 发光）

- [ ] 完成了后期合成（辉光/色差/调色）

- [ ] 保存了最终渲染图

### \U0001F4D6 费曼三句话

1. 场景全流程 = 布局 → 建模 → 材质 → UV → 灯光 → 渲染 → 后期
2. 合成是"让好变成更好"的最后一步——调色 + 辉光 + 暗角能大幅提升画面质量
3. 完成一个完整场景比学完所有功能重要 100 倍

## \U0001F34539 商业交付标准与输出（25分钟）

### 费曼输入（概念解释）

**不同用途的输出规格：**

| 用途 | 格式 | 分辨率 | 色彩空间 | 备注 |
|:----|:----|:------:|:--------:|:----|
| 电商产品图 | PNG / TIF | 2K-4K | sRGB | 白底 + 多角度 |
| 影视资产 | EXR | 4K+ | ACES / Rec.709 | 带通道 / 分层渲染 |
| 游戏资产 | FBX / GLB | 按项目规范 | sRGB | 低多边形 + LOD |
| 3D 打印 | STL / OBJ | 按精度要求 | N/A | 封闭流 + 水密 |
| 建筑可视化 | PNG / EXR | 4K+ | sRGB | 带环境 + 人物 |
| 动画短片 | MP4 / EXR 序列 | 2K-4K | Rec.709 | H.264 / ProRes |

**色彩空间为什么重要？**

```
同一个红色在不同色彩空间下完全不同：

sRGB（标准显示）  →  最常用，网页/电商/社交媒体
Rec.709（高清电视）→  广播/影视标准
ACES（电影级）    →  高端影视，色域最广
RAW（线性）       →  需要后期调色的原始素材

选错色彩空间 = 颜色偏色，大忌！
```

**练习 1：配置不同用途的输出预设**

```
# 电商产品图预设
1. 打开科幻控制台场景
2. Output Properties:
   - Resolution: 1920x1920（正方形电商图）
   - Format: PNG
   - Color: sRGB
3. Render Properties:
   - Cycles, 512 samples
   - Denoising ON
4. F12 渲染测试
5. Save As: "科幻控制台_电商图_v01.png"

# 游戏资产导出预设
1. 选择控制台主体
2. File → Export → FBX
3. 设置：
   - Scale: 0.01（厘米转米）
   - Apply Scalings: FBX Units Scale
   - Mesh Smoothing: Normals Only
4. 选择导出目录 _exports/fbx/
5. 导出为 "Console_Main.fbx"
```

**练习 2：制作作品集展示图**

```
1. 线框图截图：
   - Overlay 开启 Wireframe（Shift+Z）
   - 截图保存到 _renders/preview/
2. 素模渲染：
   - 删除所有材质灯光，仅保留默认材质
   - HDRI 设为纯白
   - 渲染保存
3. 多角度渲染：
   - 旋转相机到 4 个不同角度
   - 每个角度渲染一版
4. Turntable 动画（可选）：
   - 给空物体添加 Z 轴旋转关键帧（0°→360°，240帧）
   - 绑定相机到空物体
   - 输出为 MP4
```
