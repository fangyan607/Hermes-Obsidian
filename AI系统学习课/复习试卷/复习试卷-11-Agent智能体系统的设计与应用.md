## Section A：费曼学习法复习文档

### 一、核心概念速览

| 概念 | 一句话理解 |
|------|-----------|
| AI Agent（智能体） | 给它一个目标，它能自主理解、规划、执行复杂任务的系统 |
| 规划（Planning） | 把大任务拆成小步骤，就像做菜先列菜单再洗切炒 |
| 记忆（Memory） | 短期记忆=对话上下文中的即时回想；长期记忆=存入外部数据库随时取用 |
| 工具使用（Tool use） | Agent调用外部API做事，相当于人用计算器、查词典 |
| Generative Agents | 斯坦福搞的《西部世界》小镇，25个AI像真人一样生活、社交、记忆 |
| 反应式架构（Reactive） | 凭直觉快速反应，像膝跳反射，不思考长远 |
| 深思熟虑架构（Deliberative） | 先在脑子里推演几步再行动，像下棋高手 |
| 混合架构（Hybrid） | 结合直觉+深思，紧急时快速反应，常规时仔细规划 |
| 记忆流（Memory Stream） | Generative Agents的核心模块，完整记录Agent经验的数据库 |
| LangChain | 开源框架，用"链"的方式搭LLM应用，适合线性任务 |
| LangGraph | LangChain的升级版，用"图"做复杂工作流，支持循环和条件判断 |
| Coze / Dify | 无代码/低代码AI Bot平台，拖拽搭建 |

### 二、三种架构对比

| 维度 | 反应式（Reactive） | 深思熟虑（Deliberative） | 混合式（Hybrid） |
|------|-------------------|------------------------|----------------|
| 决策方式 | 基于当前环境即时决策 | 基于内部模型推理规划 | 动态切换两种模式 |
| 速度 | 快（毫秒级） | 慢（需多步推理） | 灵活 |
| 适应性 | 差，无法应对新场景 | 强，能处理复杂多步任务 | 兼具两者 |
| 典型例子 | 扫地机器人避障 | 物流调度、投资决策 | 自动驾驶 |
| 缺点 | 短视，可能陷入局部循环 | 计算成本高 | 系统设计复杂 |

### 三、三个CASE的核心差异

| 项目 | 私募基金问答助手 | 智能投研助手 | 投顾AI助手 |
|------|----------------|-------------|-----------|
| 架构 | 反应式 | 深思熟虑（LangGraph） | 混合式 |
| 核心流程 | LLM选工具→执行→反馈→循环 | 感知→建模→推理→决策→报告 | 评估→反应式/深思熟虑二选一 |
| 关键技术 | LangChain + Qwen-Turbo | StateGraph + Pydantic模型 | WeaverAdvisorState + 三层架构 |
| 适用场景 | 即时问答 | 深度研究报告生成 | 通用财富管理咨询 |

### 四、Generative Agents 核心机制（费曼讲解版）

想象你建了一个叫 Smallville 的小镇，住了25个AI。每个AI都有自己的职业、家人、朋友——比如 John Lin 是药店老板，老婆是大学教授，儿子学音乐。

**它们怎么活？**
1. **记忆流**：每个AI有一个日记本（记忆流），记录自己做了什么、看到了什么。比如 Isabella Rodriguez 正在写日记、查看邮件，全都被记下来。
2. **社交传播**：Sam 告诉 Tom 他在竞选镇长 → Tom 转头告诉 John → 慢慢整个镇子都在讨论。信息像八卦一样在Agent之间传播。
3. **关系记忆**：Sam 第一天不认识 Latoya，在公园偶遇后记住了"她在做摄影项目"。第二天再见到就能问："Latoya，你的项目进展如何？"——这靠长期记忆和检索。

**核心技术栈**：长期记忆（数据库） + 短期记忆（Attention检索最相关内容） + 外部工具（让GPT调用更多服务）

### 五、构建Agent的三个核心思想

1. **别对所有任务都用Agent**：简单任务用工作流更高效。Agent适合复杂、模糊、高价值的任务（如代码生成）
2. **保持简洁**：聚焦三个核心组件——环境、工具集、系统提示。先跑通再优化
3. **像Agent一样思考**：Agent只有10-20K Token的上下文窗口。模拟它的视角来发现设计缺陷

#### 问答题要点

**21题要点：**
- **反应式**：基于当前环境即时决策，速度快但缺乏适应性。如私募基金问答助手，用LangChain + Qwen-Turbo实现LLM选工具→执行→反馈循环
- **深思熟虑**：基于内部模型推理规划，能处理多步复杂任务，但成本高。如智能投研助手用LangGraph StateGraph实现感知→建模→推理→决策→报告五阶段
- **混合式**：结合两者，底层快速响应、顶层战略规划、中层协调切换。如投顾AI助手根据查询类型（紧急/信息型/分析型）动态选择处理模式

**22题要点：**
- **记忆流（Memory Stream）**：Generative Agents的核心模块，完整记录Agent所有经验。所有内容用自然语言记录，LLM可推理
- **长期记忆**：存数据库，解决GPT token上限问题
- **短期记忆**：基于Attention机制检索长期记忆中最相关内容喂给GPT
- **社交传播**：示例——Sam告诉Tom竞选镇长→Tom告诉John→整个镇子讨论。信息像八卦一样在Agent之间自然传播
- **关系记忆**：Sam和Latoya在公园偶遇，Sam记住她在做摄影项目，再次见面能自然问候"你的项目进展如何？"
