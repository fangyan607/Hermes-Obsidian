# Obsidian LLM-WIKI 学习方法库AI检索应用方案

**版本**：V1.0
**适配环境**：Obsidian + LLM-WIKI
**存量资源**：15本学习方法MD笔记
**核心用途**：学习方法知识检索、个性化学习计划制定、学习问题诊断、技能训练方案生成

## 一、库内笔记标准化规整

### 1. 统一YAML头部元数据

所有学习方法笔记头部添加元标签，LLM-WIKI可按分类、技能、难度精准检索匹配。

```yaml
---
title: 书名
category: 时间管理/学习方法/思维方式/阅读技巧/右脑开发
skill_tag: 时间管理,刻意练习,深度工作,清单系统,高维思考,金字塔原理
level: 入门/进阶/综合
keywords: 关键词1,关键词2,关键词3
type: learning
---
```

**各字段说明**：

| 字段 | 说明 | 可选值 |
|------|------|--------|
| `category` | 领域分类 | 时间管理、学习方法、思维方式、阅读技巧、右脑开发 |
| `skill_tag` | 技能标签（逗号分隔） | 时间管理,刻意练习,深度工作,清单系统,高维思考,金字塔原理 |
| `level` | 难度等级 | 入门（概念科普）、进阶（方法实操）、综合（多技能融合） |
| `type` | 库类型标识 | 固定为 `learning` |

### 3. 内容优化适配检索

1. **知识点概念归一**：各书同一概念（如"心流""专注""番茄钟"）统一术语，便于跨书语义关联
2. **内部链接关联**：同一Skill下的书籍互链 [[创建时间]] ←→ [[番茄工作法]] ←→ [[极简时间]]
3. **元数据转化**：将现有 `**Title:**` `**Authors:**` 格式逐步迁移为 YAML 标准 frontmatter
4. **核心方法提炼**：每本书提取 3-5 个核心方法/模型，单独建立概念笔记

## 二、LLM-WIKI检索运行架构

整体流程：Obsidian学习方法MD笔记 → 框架索引扫描 → 向量缓存入库 → 目录/标签路由筛选 → 调用绑定SKILL技能 → 生成学习方案应答

### 1. LLM-WIKI基础索引配置

1. 指定 `知识库/学习方法/` 为专属检索数据源，划定扫描范围
2. 开启元数据、分类、技能标签联合索引，优先按 `skill_tag` 触发对应功能
3. 调整嵌入匹配阈值，兼顾检索速度与方法精准度
4. 开启笔记溯源功能，回答可跳转查看原始MD笔记原文

### 2. 四类检索匹配规则

| 类型 | 说明 |
|------|------|
| 目录路由检索 | 按 `知识库/学习方法/` 文件夹锁定学习方法领域 |
| 标签技能检索 | 通过 `skill_tag`（如"时间管理"）触发对应SKILL技能模块 |
| 语义全文检索 | 理解用户学习场景描述，跨15本书整合方法论 |
| 层级区分检索 | 按 `level` 区分入门科普与进阶实操内容输出 |

### 3. 技能-书籍数据映射

| 技能ID | 技能名称 | 绑定书籍数 | 触发关键词 |
|--------|----------|-----------|-----------|
| Skill1 | 时间管理 | 5 | 时间管理、日程规划、创造时间、番茄钟 |
| Skill2 | 刻意练习 | 3 | 刻意练习、技能学习、费曼学习法 |
| Skill3 | 深度工作 | 5 | 深度工作、专注力、心流、影像阅读、速读 |
| Skill4 | 清单系统 | 1 | 任务清单、待办事项、GTD |
| Skill5 | 高维思考 | 6 | 高维思考、右脑思维、模式识别、发现问题 |
| Skill6 | 金字塔原理 | 2 | 金字塔原理、结构化思维、逻辑表达 |

## 四、三种实际使用模式

### 模式1：学习方法知识检索

**场景**：用户提问学习方法论概念、技巧、模型

**流程**：
```
用户提问 → LLM-WIKI匹配目录/标签 → 调取对应书籍笔记内容 → 整合回答
```

**示例问题**：
- "什么是番茄工作法？如何执行？" → 调取 [[番茄工作法]]
- "金字塔原理的核心原则是什么？" → 调取 [[金字塔原理大全集]]
- "如何制定高效的学习计划？" → 跨书整合 [[创造时间]] + [[高效清单工作法]] + [[刻意学习]]
- "费曼学习法的具体步骤是什么？" → 调取 [[费曼经典：一个好奇者的探险人生]]

---

### 模式2：个性化学习计划制定

**场景**：用户提供学习目标、时间预算、当前水平，系统制定定制化学习方案

**流程**：
```
用户输入目标+时间+水平 → 高维思考(Skill5)分析需求 → 时间管理(Skill1)规划窗口
→ 清单系统(Skill4)拆解任务 → 刻意练习(Skill2)设计训练 → 金字塔原理(Skill6)输出计划
```

**示例输入**：
> 我是一名软件开发者，想学习AI Agent开发，每天有2小时学习时间，零基础。

**系统输出**：
1. **需求分析**（Skill5 高维思考）：拆解AI Agent的核心技能树
2. **时间规划**（Skill1 时间管理）：14周渐进学习时间表
3. **任务清单**（Skill4 清单系统）：每周具体任务拆解
4. **练习方案**（Skill2 刻意练习）：每个知识点的刻意练习方法
5. **输出框架**（Skill6 金字塔原理）：学习笔记结构化模板

---

### 模式3：学习问题诊断

**场景**：用户描述学习困难（拖延、低效、无法专注、学了就忘），系统诊断并给出改进方案

```
用户描述困难 → 语义理解匹配问题类型 → 跨技能诊断 → 输出诊断报告+改善方案
```

**常见学习问题与对应技能**：

| 问题类型 | 典型描述 | 诊断技能 | 推荐书籍 |
|----------|----------|----------|----------|

| 方法效率问题 | "学得很慢""记不住" | Skill2 刻意练习 | [[刻意学习]]、[[费曼经典]] |
| 专注力问题 | "容易分心""坐不住" | Skill3 深度工作 | [[番茄工作法]]、[[快速阅读]] |

## 五、LLM-WIKI专用提示词模板

```
仅依据Obsidian LLM-WIKI内学习方法MD笔记内容作答，不编造域外理论与方法。
1. 识别问题匹配对应 `skill_tag` 与 `category`，调取库内原有知识点
2. 多技能关联问题时，跨笔记整合内容
3. 复杂方法分层说明，实操类标注清晰步骤
4. 重要内容关联对应笔记名称（使用 [[wikilink]] 格式），方便溯源查阅
5. 回答中区分"该方法的理论依据"与"实操建议"
```

```
启用学习方法SKILL组合（高维思考+时间管理+清单系统+刻意练习+金字塔原理），

根据用户提供的以下信息：
- 学习目标：[用户输入]
- 可用时间：[用户输入]
- 当前水平：入门/进阶/专业
- 偏好风格：[用户输入]

输出包含以下部分：
1. 需求分析 — 拆解目标为可执行的技能点
2. 时间规划 — 建议每日/每周时间分配
3. 任务清单 — 阶段性任务拆解
4. 练习方法 — 每个阶段对应的刻意练习方案
5. 输出模板 — 结构化学习笔记模板
```

### 模板3：学习问题诊断提示词

```
启用学习方法诊断模式，综合调取6大SKILL模块内容。
根据用户描述的学习困难症状，进行跨技能问题诊断。

诊断流程：
1. 识别用户描述中匹配的问题类型（时间管理|方法效率|专注力|任务混乱|思维瓶颈|表达输出）
2. 调取对应技能模块的笔记内容
3. 输出诊断报告，包含：
   - 核心问题判断
   - 关联技能与推荐书籍（[[wikilink]]）
   - 具体改善方案（含实操步骤）
   - 推荐阅读链接

注意：不作非专业学习障碍判断，如疑似ADHD等建议用户寻求专业评估。
```

## 六、知识库迭代优化方案

1. **笔记增量更新**：新增学习方法书籍归入 `知识库/学习方法/`，同步更新 `.书库索引/` 和 `schema/kb.yaml`
