## 🍅 番茄5：~/.hermes/ 目录结构（25分钟）

### 5.1 用"房子"类比理解目录

想象 `~/.hermes/` 是一套专属于你的 AI 住宅：

```
~/.hermes/  ← 你的 AI 之家
│
├── config.yaml   ← 客厅（总控开关，控制全屋）
├── .env          ← 保险柜（密码/密钥）
├── MEMORY.md     ← 书房（长期记忆库）
├── USER.md       ← 卧室（你的个人档案）
├── SOUL.yaml     ← 衣帽间（AI 的人格设定）
│
├── skills/       ← 工具间（技能仓库）
│   ├── built-in/ ← 成品工具
│   └── custom/   ← 自创工具
│
├── profiles/     ← 分身房（多实例隔离）
└── cache/        ← 储藏室（临时缓存）
```

### 5.2 查看你的目录

```bash
# 查看 Hermes 配置目录
ls -la ~/.hermes/

# 查看目录树（Ubuntu/macOS）
find ~/.hermes -maxdepth 2 -type f | sort
```

### 5.3 各文件职责速查

| 文件/目录 | 类比 | 功能 | 修改频率 |
|:----------|:-----|:-----|:---------|
| `config.yaml` | 客厅开关 | 主配置：模型、记忆、技能开关 | 偶尔 |
| `.env` | 保险柜 | API Key 等敏感信息 | 极低 |
| `MEMORY.md` | 书房 | AI 的长期记忆 | 频繁（自动/手动） |
| `USER.md` | 卧室 | 你的偏好、风格、禁忌 | 低 |
| `SOUL.yaml` | 衣帽间 | AI 的人格设定 | 低 |
| `skills/` | 工具间 | 技能库存放 | 中 |
| `profiles/` | 分身房 | 多实例隔离 | 中 |
| `cache/` | 储藏室 | 临时缓存 | 自动清理 |

## 🍅 番茄6：config.yaml 详解（25分钟）

### 6.1 配置文件加载优先级

```
加载顺序（后者覆盖前者）：
1. 系统默认配置
2. ~/.hermes/config.yaml
3. --config 指定的配置文件
4. 命令行参数
```

### 6.2 完整配置模板

```yaml
# ==================== 模型配置 ====================
model:
  provider: openrouter           # 提供商：openrouter / openai / anthropic / deepseek
  name: deepseek/deepseek-v4-flash  # 模型名称（支持 OpenRouter 格式）
  api_key: ${OPENROUTER_API_KEY} # 从环境变量读取
  base_url: https://openrouter.ai/api/v1
  temperature: 0.7               # 创造性：0=保守，1=创新
  max_tokens: 4096               # 最大输出长度

# ==================== 记忆配置 ====================
memory:
  enabled: true                  # 是否启用记忆
  max_tokens: 10000              # 记忆窗口大小
  compression: true              # 是否压缩旧记忆
  curated_write: true            # 策展式写入（推荐开启）
  write_threshold: 0.7           # 写入阈值：重要性>0.7才写入

# ==================== 技能配置 ====================
skills:
  enabled: true                  # 是否启用技能系统
  auto_generate: true            # 是否自动生成技能
  path: ~/.hermes/skills         # 技能库存放路径

# ==================== 工具配置 ====================
tools:
  enabled: true                  # 是否启用工具
  default_toolset: full          # 默认工具集
  mcp_servers: []                # MCP 服务器列表

# ==================== 日志配置 ====================
logging:
  level: INFO                    # DEBUG / INFO / WARNING / ERROR
  file: ~/.hermes/hermes.log     # 日志文件路径
```

### 6.3 关键配置项调优建议

| 配置项 | 作用 | 推荐值 | 调大/调小的效果 |
|:-------|:-----|:-------|:---------------|
| `temperature` | 创造性程度 | 0.7 | ↑更创意但可能偏离主题 |
| `max_tokens` | 最大输出长度 | 4096 | ↑能回答更长内容但消耗更多 Token |
| `memory.max_tokens` | 记忆窗口 | 10000 | ↑记得更多但响应变慢 |
| `curated_write` | 策展式写入 | true | 开启防止记忆膨胀 |
| `write_threshold` | 写入阈值 | 0.7 | ↑记忆更精简但可能漏掉重要信息 |

### 6.4 查看当前配置

```bash
# 查看配置
hermes config show

# 验证配置语法
hermes config validate
```

## 🍅 番茄7：.env 与安全配置（25分钟）

### 7.1 为什么要用 .env？

```yaml
# ❌ 错误做法：API Key 直接写 config.yaml
model:
  api_key: sk-or-v1-xxxxxx    # 危险！可能被提交到 Git

# ✅ 正确做法：通过环境变量引用
model:
  api_key: ${OPENROUTER_API_KEY}  # 安全！Key 在 .env 中
```

**.env 的三大作用：**
1. **安全隔离**：敏感信息不进版本控制
2. **环境切换**：开发/生产环境使用不同的 Key
3. **密钥轮换**：换 Key 只需改 .env，不用改配置

### 7.2 .env 文件模板

```env
# ==================== 核心 API Keys ====================
# OpenRouter（推荐，聚合多模型）
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxxxxxx

# OpenAI（备选）
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx

# Anthropic（备选）
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxx

# DeepSeek（备选）
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx

# ==================== 扩展服务 ====================
# GitHub（MCP 连接用）
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxx

# ==================== 代理设置（选填）====================
HTTP_PROXY=http://127.0.0.1:7890
HTTPS_PROXY=http://127.0.0.1:7890
```

### 7.3 安全最佳实践

```bash
# 1. 设置 .env 文件权限（仅自己可读）
chmod 600 ~/.hermes/.env

# 2. 添加到全局 .gitignore
echo ".env" >> ~/.gitignore
echo "auth.json" >> ~/.gitignore

# 3. 验证 .env 不会被追踪
cd ~/.hermes
git init 2>/dev/null || true
echo ".env" >> .gitignore
echo "auth.json" >> .gitignore
```

```bash

```

## 🍅 番茄8：配置实践 + 练习（25分钟）

### 8.1 完整配置流程

```bash
# 1. 进入配置目录
cd ~/.hermes

# 2. 创建完整目录结构
mkdir -p skills/{built-in,custom} profiles/default cache/{models,conversations}

# 3. 创建基础配置文件
touch config.yaml .env MEMORY.md USER.md SOUL.yaml

# 4. 编辑 config.yaml（使用上面 6.2 节的模板）
nano config.yaml

# 5. 编辑 .env（填入你的 API Key）
nano .env

# 6. 验证配置
hermes config validate
```

### 8.2 配置完成后验证

```bash
# 验证配置语法
hermes config validate

# 查看生效配置
hermes config show

# 启动对话验证
hermes chat
```

### 💪 刻意练习——配置修改三轮循环

**练习目标**：在20分钟内对配置文件进行3轮"修改—验证—观察"的完整循环

```
===== 循环 1：入门任务 =====
1. 修改 config.yaml 中的 temperature 为 0.3
2. 运行 `hermes config validate` 验证
3. 运行 `hermes chat` 问一个创意类问题（如"讲个故事"）
4. 再改 temperature 为 1.0，问同样的问题
5. 对比两次回答的差异
✅ 验证方式：0.3 的回答更保守，1.0 更有创意

===== 循环 2：进阶任务 =====
1. 在 config.yaml 中切换模型（如从 deepseek 改为别的模型）
2. 运行 `hermes config validate` 通过
3. 运行 `hermes chat` 确认模型已切换
✅ 验证方式：hermes 使用新配置的模型

===== 循环 3：挑战任务 =====
1. 不依赖向导，手动创建一份完整的 config.yaml + .env 配置
2. 包含至少 3 个 Provider 的 API Key 占位
3. 配置完成后 `hermes config validate` 通过
✅ 验证方式：从头搭建的配置能正常启动对话
```
