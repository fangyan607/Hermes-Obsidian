## 🍅 番茄1：服务器选购与Ubuntu初始化

> ⏱ 25分钟 | 📌 核心技能：云服务器创建与系统安装

### 1.1 选择阿里云服务器

**推荐配置**（教程已设定）：
| 配置项 | 规格 | 说明 |
|:-----|:----|:----|
| CPU | 2vCPU | 足够运行Hermes + 多个Agent |
| 内存 | 2GiB | 轻量级AI任务，推荐≥4GiB |
| 系统盘 | 40GB SSD | 存储知识库和日志 |
| 操作系统 | Ubuntu 24.04 LTS | 长期支持，社区活跃 |
| 带宽 | 3Mbps | 够个人使用 |
| 地域 | 华东1（杭州） | 国内访问延迟低 |

**购买流程**：
1. 登录阿里云官网 → 产品 → 云服务器ECS
2. 按量付费或包年包月（推荐包月约¥70-100）
3. 选择配置后创建实例
4. **关键**：在安全组中放行以下端口

### 1.2 安全组规则配置

连接服务器前，先配置安全组（防火墙）：

| 端口 | 协议 | 用途 | 建议 |
|:---|:----|:----|:----|
| 22 | TCP | SSH连接 | 务必开启 |
| 18789 | TCP | Hermes Web界面 | 教程使用 |
| 80 | TCP | HTTP | 后续可能需要 |
| 443 | TCP | HTTPS | 后续可能需要 |

**阿里云控制台操作**：
```
ECS控制台 → 网络与安全 → 安全组 → 配置规则 → 入方向
→ 手动添加/快速添加以上端口
```

> 💡 **注意**：生产环境应将SSH端口改为非标准端口，18789端口应绑定域名+SSL

### 1.3 连接到服务器

#### Windows（推荐PowerShell）

```bash
ssh root@你的服务器公网IP
```

#### 首次登录

```bash
# 如果你用的是root用户（阿里云默认）
ssh root@114.215.208.216

# 创建普通用户（安全最佳实践）
adduser hermes
usermod -aG sudo hermes

# 切换到新用户
su - hermes
```

### 1.4 系统更新

```bash
# 更新软件包列表
sudo apt update

# 升级所有已安装软件
sudo apt upgrade -y

# 安装基础工具包
sudo apt install -y curl git ca-certificates build-essential

# 检查Python版本（Hermes需要Python ≥ 3.10）
python3 --version
# 应该显示 Python 3.12.x 或更高
```

## 🍅 番茄2：基础环境配置

> ⏱ 25分钟 | 📌 核心技能：Python环境、Node.js、服务器安全基础

### 2.1 安装Python环境

```bash
# 检查Python 3
python3 --version

# 安装pip（Python包管理器）
sudo apt install -y python3-pip python3-venv

# 验证pip
pip3 --version

# 安装虚拟环境管理
sudo pip3 install virtualenv
```

### 2.2 安装Node.js（可选，部分Agent需要）

```bash
# 使用NodeSource安装Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 验证
node --version   # 应显示 v20.x
npm --version    # 应显示 10.x
```

### 2.3 服务器安全基础

#### 配置UFW防火墙

```bash
# 安装UFW
sudo apt install -y ufw

# 设置默认策略
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 放行必要端口
sudo ufw allow 22/tcp       # SSH
sudo ufw allow 18789/tcp    # Hermes

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status verbose
```

#### 配置Fail2Ban（防暴力破解）

```bash
sudo apt install -y fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban
```

### 2.4 配置时区与系统语言

```bash
# 设置时区为北京时间
sudo timedatectl set-timezone Asia/Shanghai

# 确认
timedatectl

# 设置系统语言为英文（避免中文乱码）
sudo update-locale LANG=en_US.UTF-8
```

## 🍅 番茄3：SSH密钥配置与Git工作流

> ⏱ 25分钟 | 📌 核心技能：SSH密钥认证、Git全局配置

### 3.1 什么是SSH密钥？

```
公钥 🔑 → 放在服务器上（锁）
私钥 🔐 → 保存在本地（钥匙）

没有SSH密钥：每次连接都要输入密码
有了SSH密钥：自动验证，无需密码（也更安全）
```

### 3.2 在本地生成SSH密钥

#### Windows (PowerShell)

```powershell
# 生成SSH密钥对
ssh-keygen -t ed25519 -C "your_email@example.com"

# 全程回车（不设置密钥密码，方便自动同步）
# 生成的密钥位置：C:\Users\你的用户名\.ssh\
#   id_ed25519      → 私钥（自己保管）
#   id_ed25519.pub  → 公钥（放到服务器）
```

#### macOS/Linux

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

### 3.3 将公钥上传到服务器

```bash
# 查看公钥内容
cat ~/.ssh/id_ed25519.pub
# 复制输出的完整内容

# 登录服务器
ssh root@114.215.208.216

# 创建.ssh目录（如果不存在）
mkdir -p ~/.ssh
chmod 700 ~/.ssh

# 编辑authorized_keys文件
nano ~/.ssh/authorized_keys
# 粘贴刚才复制的公钥内容
# Ctrl+O 保存，Ctrl+X 退出

# 设置权限
chmod 600 ~/.ssh/authorized_keys
```

#### 验证配置

```bash
# 退出服务器后，重新连接（这次应该不用密码）
exit
ssh root@114.215.208.216
```

### 3.4 Git全局配置

#### 在服务器上配置Git

```bash
# 配置Git用户名和邮箱
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub邮箱"

# 配置GitHub代理（国内服务器推荐）
git config --global url."https://ghproxy.com/https://github.com/".insteadOf "https://github.com/"

# 验证配置
git config --global --list
```

#### 在本地电脑配置Git

```bash
# 同样配置本地Git
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub邮箱"
```

### 3.5 生成GitHub SSH密钥（本地↔GitHub通信用）

```powershell
# 本地电脑执行
ssh-keygen -t ed25519 -C "github-obsidian"
# 将公钥添加到GitHub：
# GitHub → Settings → SSH and GPG keys → New SSH key
```

> 💡 **最佳实践**：服务器使用一对SSH密钥，GitHub使用另一对，各司其职。

## 🍅 番茄4：GitHub仓库创建与团队协作基础

> ⏱ 25分钟 | 📌 核心技能：创建GitHub仓库、克隆、推送、拉取

### 4.1 创建GitHub仓库

1. 登录 [GitHub](https://github.com)
2. 点击右上角 `+` → `New repository`
3. 填写：
   - **Repository name**: `obsidian-knowledge-base`（或你喜欢的名字）
   - **Description**: "我的Obsidian知识库，同步到阿里云Hermes"
   - **Type**: Private（私有仓库，保护你的笔记隐私）
   - **Initialize with README**: ✅
4. 点击 `Create repository`

### 4.2 将GitHub仓库的SSH密钥添加到服务器

```bash
# 在服务器上生成专用于GitHub的SSH密钥
ssh-keygen -t ed25519 -f ~/.ssh/github_key -C "server-github"

# 查看公钥
cat ~/.ssh/github_key.pub
```

### 4.3 配置SSH Config（简化连接）

```bash
# 创建SSH配置文件
nano ~/.ssh/config
```

添加以下内容：

```
# GitHub
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/github_key
```

```bash
# 设置权限
chmod 600 ~/.ssh/config
```

### 4.4 测试GitHub连接

```bash
# 测试连接（服务器）
ssh -T git@github.com
# 成功提示：Hi 用户名! You've successfully authenticated...

# 克隆仓库（首次）
cd ~
git clone git@github.com:你的用户名/obsidian-knowledge-base.git
```

### 4.5 Git工作流速览

```
日常协作流程：

本地电脑                    GitHub                    服务器
   │                         │                         │
   │ --- git add/commit ---→│                         │
   │ --- git push ---------→│                         │
   │                         │ --- git pull ---------→│
   │                         │                         │
   │ ←--- git pull ---------│                         │
   │                         │ ←--- git push ---------│
```

### 4.6 常用Git命令速查

```bash
# 查看状态
git status

# 添加文件到暂存区
git add 文件名     # 添加指定文件
git add .          # 添加所有变更

# 提交
git commit -m "描述这次改动"

# 推送到远程
git push origin main

# 拉取最新
git pull origin main

# 查看提交历史
git log --oneline --graph
```
