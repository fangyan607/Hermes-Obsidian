# GBrain 知识库总览

## 什么是 GBrain

GBrain 是一种知识图谱型知识库系统，其核心设计理念是**实体中心**——每一页 Markdown 文件代表一个独立的概念或实体（Entity），实体之间通过类型化关系（Typed Relations）相互连接，形成一个可被自动构建和可视化的知识图谱。

GBrain 的核心特征：

- **页面即实体**：每个 `.md` 文件代表知识图谱中的一个节点
- **类型化关系**：Frontmatter 中的 `relations` 字段为每个 [wikilink] 标注了关系类型，使连接语义化
- **自动建图**：GBrain 插件自动扫描 wikilink，构建并可视化知识图谱
- **结构化元数据**：YAML frontmatter 中的 `type`、`domain`、`tags` 字段支持多维筛选和检索

## 如何使用这些页面

### 在 Obsidian 中使用
1. 将 `GBrain/` 目录复制到你的 Obsidian vault 根目录
2. 确保安装了 **Graph View** 插件（Obsidian 核心插件）
3. 在 Graph View 中启用 "Tags" 和 "Groups" 以按领域和类型着色节点
4. 推荐安装第三方插件 **Dataview** 以执行结构化查询

### 查询示例（Dataview）
```dataview
TABLE type, domain
FROM "GBrain"
WHERE domain = "创造性思维"
SORT type ASC
```

```dataview
TABLE relations
FROM "GBrain/心理学"
FLATTEN relations
```

### 扩展指南
你可以按照以下流程为 GBrain 添加新实体：
1. 在对应的领域目录下创建新的 `.md` 文件
2. 按照 frontmatter 模板填写 `type`、`domain`、`relations` 和 `tags`
3. 使用 `[[wikilink]]` 引用已有实体
4. 运行 `init.sh`（见下文）检查 frontmatter 格式
