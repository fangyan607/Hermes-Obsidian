## 核心方法论

### 番茄工作法（时间结构）

```
🍅 25分钟专注 → 5分钟休息 → 重复
每4个番茄钟后 → 15-30分钟长休息
```

### 费曼学习法（质量保障）

每个核心概念后嵌入「费曼自测」：

| 类型 | 用途 | 示例 |
|:-----|:-----|:-----|
| 复述型 | 检测基本理解 | "用三句话解释CDN是什么" |
| 对比型 | 检测概念辨析 | "Cloudflare 和传统DNS的区别是什么？" |
| 应用型 | 检测迁移能力 | "举一个需要配置WAF的场景" |
| 编程型 | 检测实践能力 | "独立配置一条DNS记录" |

### 刻意练习（能力提升）

```
明确目标 → 专注练习 → 即时反馈 → 纠正迭代
```

**每日刻意练习任务：**
- 明确的目标（"完成DNS配置"）
- 专注练习（排除干扰，单点突破）
- 即时反馈（检查配置是否生效）
- 纠正迭代（失败则分析原因再试）

## 前置准备

### 需要准备的资源

1. **域名**
   - 一个自己的域名（可在阿里云、腾讯云、Namecheap 购买）
   - 建议使用 `.com` 或 `.dev` 域名

2. **账号**
   - Cloudflare 账号（免费版即可）
   - GitHub 账号

3. **开发环境**
   - Node.js 18+
   - Git
   - 代码编辑器（VS Code 推荐）

4. **可选**
   - 一个静态网站项目（用于 Day 5 部署）
   - 一个 API 项目（用于 Day 6 Workers）

## 常见问题速查

| 问题 | 解决方案 |
|:-----|:---------|
| DNS 解析不生效 | 等待 TTL 时间，检查 NS 记录 |
| SSL 证书不生效 | 检查 SSL 模式设置 |
| Pages 部署失败 | 检查构建命令和输出目录 |
| Workers 报错 | 检查代码语法和权限配置 |
| GitHub 联动失败 | 检查 API Token 权限 |
