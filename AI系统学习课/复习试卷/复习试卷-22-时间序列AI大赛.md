## Section A：费曼学习法复习文档

### 一、核心概念（用自己的话解释）

**1. 时间序列预测为什么难？**
传统的 ARMA/ARIMA 模型有两个硬伤：(1) ARMA 要求数据平稳，但真实数据（如股价、流量）往往不平稳；(2) ARIMA 是线性模型，无法处理非线性关系，且要求时间间隔严格等长（如 X₁ 和 X₂ 间隔1小时，X₂ 和 X₃ 也必须间隔1小时）。数据一旦缺失，还得用插值预估，会引入噪音。

**2. Prophet 是什么？**
Facebook 开源的时间序列预测工具（https://facebook.github.io/prophet/），核心思想是用加法模型把时间序列拆成四个部分：

```
y(t) = g(t) + s(t) + h(t) + ε
```

- **g(t)** 趋势项：非周期性变化（饱和增长模型 或 分段线性模型）
- **s(t)** 周期项：年/周/日周期性（用傅里叶级数拟合）
- **h(t)** 活动效果项：节假日、大事件等异常活动
- **ε** 无法解释的误差

**3. Prophet 比传统模型强在哪？**
- 能处理数据缺失（不需要插值）
- 能自动处理趋势迁移（shifts in the trend）
- 能识别异常数据点（outliers）
- 对节假日/大事件建模更精准（如春节、双十一）

### 二、关键机制拆解

**4. 突变点（Changepoint）检测**
Prophet 默认在前 80% 的时间序列中均匀放置 25 个候选突变点，通过 Laplace 稀疏先验对突变幅度做正则化，筛选出显著点。未来预测时，趋势延续最后一个突变点后的斜率。关键参数：
- `changepoint_prior_scale`：控制趋势灵活性，越大越灵活（但可能过拟合）
- `n_changepoints`：候选突变点数量
- `changepoint_range`：在时间序列前多少范围内寻找突变点（如 0.9 表示前 90%）

**5. 季节性 = 傅里叶级数**
```
s(t) = Σ [aₙ·cos(2πnt/P) + bₙ·sin(2πnt/P)]
```
- P 是周期：年度=365.25，周=7
- N 是傅里叶阶数：N 越大，越能捕捉高频变化（但也可能拟合噪音）
- 通过参数 `yearly_seasonality` / `weekly_seasonality` / `daily_seasonality` 控制

**6. 节假日建模**
把节假日做成一个 DataFrame（两列：`holiday` 名称 + `ds` 日期），传入 Prophet。可以设置 `lower_window` 和 `upper_window` 来覆盖节假日前后的影响期。这个 DataFrame 必须同时包含历史数据和要预测的时期中的节假日。

### 三、实战项目关键步骤

**7. Prophet 的标准用法**
```python
model = Prophet()
model.fit(df)              # 输入两列: ds(日期), y(预测值)
future = model.make_future_dataframe(periods=365)  # 扩展未来日期
forecast = model.predict(future)
model.plot(forecast)        # 绘制预测结果
model.plot_components(forecast)  # 成分分解: trend / weekly / yearly
```

**8. 饱和增长（Logistic Growth）**
当增长有上限时（如 App 下载量 ≤ 手机用户总数），需设置承载能力 cap：
```python
df['cap'] = 8.5   # 每个时间点都要指定
m = Prophet(growth='logistic')
```

**9. 预测值组成**
因为是加法模型：
```
yhat = trend + weekly + yearly + holidays
```

### 四、AI 大赛实战：资金流入流出预测

**10. 任务**
天池大赛：根据 2013-07-01 到 2014-08-31 的数据（2.8万用户，284万行为数据），预测 2014年9月每天的申购总额和赎回总额。提交 30 行结果（每天一行），评估指标：申购得分×45% + 赎回得分×55%，误差 ≤ 0.3 才有分。

**11. 数据表（4张）**
| 表名 | 内容 |
|------|------|
| user_profile_table | 用户 ID、性别、城市、星座 |
| user_balance_table | 每日余额、申购量、赎回量、消费量等 |
| mfd_day_share_interest | 万份收益、七日年化收益率 |
| mfd_bank_shibor | 银行间拆借利率（隔夜到1年共8个期限）|

**12. 两种预测方法**

**方法一：ARIMA**
```python
ARIMA(purchase, order=(7,1,5)).fit().predict('2014-09-01', '2014-09-30')
```
缺点：太简单，没利用周内规律，没考虑节假日和利率波动。

**方法二：周期因子法**
1. 算每周几的均值 ÷ 整体均值 = 周期因子
2. 算 base（去掉周期因子影响的日均值）
3. 预测值 = base × 周期因子

#### 三、问答题（参考要点）

**21. 突变点检测机制**
- **(1) 发现突变点**：Prophet 默认在前 80% 时间序列中均匀放置 25 个候选突变点，通过 Laplace 稀疏先验（L1 正则化）对突变幅度进行正则化，只保留幅度显著的点。
- **(2) 未来趋势延续**：预测未来时，趋势不再发生新的突变，而是延续最后一个突变点之后的斜率。
- **(3) `changepoint_prior_scale`**：控制突变点选择的灵活度，默认 0.05。值越大，允许的突变幅度越大，拟合越灵活（但可能过拟合）；值越小，趋势越平滑（但可能欠拟合）。

**22. 大赛预测方案设计**
- **(1) 模型选择**：倾向于 Prophet 而非 ARIMA。原因：Prophet 不需要平稳性检验和数据插值，能自动处理缺失值和突变点，且能融入节假日因素。ARIMA 对这次任务过于简单，无法利用周内规律和外部事件。
- **(2) 周期因子分析**：先进行 STL 分解观察趋势和周期；按工作日/周末和周内规律计算周期因子；结合周因子和月因子，用 base × 周期因子预测每日值。
- **(3) 特殊日期**：需要考虑 2014年9月的节假日（如中秋节 9月6-8日），以及收益率变化和银行间拆借利率波动对资金流入流出的影响。还可以结合 Prophet 的 `holidays` 参数将这些节日纳入模型。
