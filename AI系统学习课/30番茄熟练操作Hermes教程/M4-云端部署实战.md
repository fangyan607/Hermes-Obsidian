## 🍅 番茄13：部署前的准备（25分钟）

### 13.1 为什么需要云服务器部署？

| 场景 | 本地部署 | 云服务器部署 |
|:-----|:---------|:------------|
| 尝鲜体验 | ✅ 够用 | ⛔ 没必要 |
| 24/7 在线 Agent | ❌ 关电脑就停 | ✅ 全天候运行 |
| 团队共享 | ❌ 仅本机 | ✅ 公网可访问 |
| Telegram 集成 | ❌ 需公网IP | ✅ 直接对接 |
| 定时任务（Cron） | ❌ 需一直开机 | ✅ 7x24 在线 |

**适合人群**：想跑 24 小时 Agent、集成消息平台、做自动化工作流的用户。

### 13.2 服务器选型

| 云厂商 | 配置 | 价格 | 实测效果 |
|:-------|:-----|:-----|:---------|
| **阿里云** | 2vCPU / 2GB RAM | ¥30-50/月 | 本地部署 Hermes 够用 |
| **腾讯云** | 2vCPU / 2GB RAM | ¥30-50/月 | 源码部署成功，Web 面板可访问 |
| **华为云** | 2vCPU / 4GB RAM | ¥50-80/月 | 未测试 |

> 💡 **选型建议**：2vCPU/2GiB 足够运行 Hermes + Web 面板，选择离家最近的区域延迟最低。

### 13.3 前置配置：安全组放行端口

**⚠️ 这是最容易忽略的一步！** 新服务器默认所有端口关闭，需要手动放行。

**阿里云操作：**
1. 控制台 → 云服务器 ECS → 实例 → 安全组
2. 入方向 → 添加规则
3. 放行端口：`18789`（Hermes Web 面板默认端口）
4. 授权对象：`0.0.0.0/0`（允许所有 IP 访问，后续可收紧）

**腾讯云操作：**
1. 控制台 → 轻量应用服务器 → 防火墙
2. 添加规则：
   - 端口：`18789`
   - 协议：TCP
   - 来源：`0.0.0.0/0`

> ![[Pasted image 20260620100918.png]] ← 安全组放行截图示例
> ![[Pasted image 20260620154303.png]] ← 腾讯云防火墙配置截图示例

也可在服务器内部配置：

```bash
# Ubuntu 内部防火墙放行
sudo ufw allow 18789/tcp
sudo ufw status
```

### 13.4 获取 OpenRouter API Key

可在本地操作，也可在服务器上操作：

1. 访问 https://openrouter.ai → 注册 → Settings → API Keys
2. 创建 Key → 充值 $5-10
3. 复制 Key（格式：`sk-or-v1-xxxxxxxx`）

> ⚠️ **安全提醒**：API Key 不要在聊天中直接输入，通过 `~/.hermes/.env` 管理。

## 🍅 番茄14：环境初始化与安装（25分钟）

> 📸 本节操作截图全部来自 [[日记/2026-06-21]] 的腾讯云实战部署记录。

### 14.1 SSH 登录服务器

```bash
# 在本地终端执行（替换为你的 IP）
ssh ubuntu@你的服务器IP

# 例如：ssh ubuntu@122.51.219.154
```

### 14.2 系统初始化

```bash
# 更新系统包
sudo apt update && sudo apt upgrade -y

# 安装基础依赖
sudo apt install -y curl git build-essential ca-certificates software-properties-common
```

> ⚠️ **注意**：系统更新后建议重启服务器以加载新内核。

### 14.3 配置镜像加速（关键！）

国内服务器访问 GitHub 和 PyPI 超时是最常见的坑。**以下配置请按顺序执行：**

```bash
# 1. GitHub 镜像加速（解决 git clone 超时）
git config --global url."https://ghproxy.com/https://github.com/".insteadOf "https://github.com/"
```

> ![[Pasted image 20260621115952.png]] ← 配置 GitHub 镜像截图

```bash
# 2. 安装 uv 包管理器（比 pip 更快，更好的依赖管理）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 配置环境变量
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 验证安装
uv --version
```

> ![[Pasted image 20260621120134.png]] ← 安装 uv 截图

### 14.4 安装 Python 3.11

> 📸 安装 Python 3.11 是踩坑最多的一步！大量依赖需要 Python 3.11，不要用系统默认 Python。

```bash
# 1. 添加 Python 官方源
sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt update

# 2. 安装 Python 3.11 及开发依赖
sudo apt install -y python3.11 python3.11-dev python3.11-venv python3.11-distutils

# 3. 验证安装
python3.11 --version
```

### 14.5 源码手动部署 Hermes（兜底方案）

> ⚠️ **关键决策**：一键安装脚本在国内环境大概率超时（443错误），**推荐直接源码手动部署**。

```bash
# 步骤1：创建独立虚拟环境（隔离系统环境，避免依赖冲突）
python3.11 -m venv ~/hermes-agent-venv
source ~/hermes-agent-venv/bin/activate
```

> ![[Pasted image 20260621134057.png]] ← 创建虚拟环境截图

```bash
# 步骤2：配置国内 PyPI 镜像
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
pip install --upgrade pip
```

> ![[Pasted image 20260621134903.png]] ← 配置 PyPI 清华源截图

```bash
# 步骤3：克隆 Hermes 源码（通过镜像加速）
git clone https://github.com/NousResearch/hermes-agent.git
# 或者使用 Gitee 镜像：git clone https://gitee.com/Nicias/hermes-agent.git
```

> ![[Pasted image 20260621140103.png]] ← 克隆源码截图

```bash
# 步骤4：进入目录，本地安装
cd hermes-agent
pip install .
```

> ![[Pasted image 20260621141336.png]] ← 注意 `pip install .` 后面的点不能少！
> ![[Pasted image 20260621141401.png]] ← 安装成功截图

> ![[Pasted image 20260621162044.png]] ← 配置模型截图

> ⚠️ **注意**：`hermes setup` 过程中会询问沙箱模式，选择 **Local（本机直接运行）**。

### 15.2 安装 Node.js 并构建前端资源（踩坑集中区！）

```bash
# 步骤1：安装 Node.js 20 LTS（⚠️ 不要用 apt 默认版本，太旧！）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 验证版本（必须 >= 20.19）
node -v
npm -v
```

> ⚠️ **踩坑记录**：apt 默认安装的 Node 18.20.8 不满足 Vite 要求（需要 ≥ 20.19），导致 `npm run build` 失败。必须手动安装 Node 20 LTS。

> ![[Pasted image 20260621170016.png]] ← 安装 Node.js 20 截图

```bash
# 步骤2：进入前端目录，安装依赖并打包
cd ~/hermes-agent/web
npm config set registry https://registry.npmmirror.com
npm install
npm run build
```

> ⚠️ **踩坑记录**：如果之前用低版本 Node 构建失败，记得先删除旧依赖：
> ```bash
> rm -rf node_modules package-lock.json
> ```

### 15.4 配置 systemd 服务（后台常驻）

创建 systemd 服务文件，让 Hermes 在后台 7x24 运行：

```bash
sudo nano /etc/systemd/system/hermes-agent.service
```

写入以下内容：

```ini
[Unit]
Description=Hermes Agent Dashboard Service
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/hermes-agent
Environment="PATH=/home/ubuntu/hermes-agent-venv/bin"
Environment="HERMES_WEB_STATIC=/home/ubuntu/hermes-agent/hermes_cli/web_dist"
ExecStart=/home/ubuntu/hermes-agent-venv/bin/hermes dashboard --host 0.0.0.0 --port 18789 --insecure
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 重载并启动服务
sudo systemctl daemon-reload
sudo systemctl enable hermes-agent.service
sudo systemctl start hermes-agent.service

# 查看状态
sudo systemctl status hermes-agent.service

# 查看日志
sudo journalctl -u hermes-agent.service -f
```

### 16.2 常见踩坑记录

> 📸 以下踩坑全部来自 [[日记/2026-06-20]] 和 [[日记/2026-06-21]] 的真实部署经历。

| 问题 | 原因 | 解决方案 |
|:-----|:-----|:---------|
| `git clone` 443 超时 | GitHub 国内访问慢 | 配置 ghproxy 镜像 |
| `pip install` 超时 | PyPI 国内访问慢 | 配置清华源镜像 |
| Python 版本冲突 | 系统默认 Python 太低 | 手动安装 Python 3.11 |
| `npm run build` 失败 | Node 版本 < 20.19 | 安装 Node.js 20 LTS |
| Web 面板 404 | 前端静态目录路径错误 | 设置 `HERMES_WEB_STATIC` 环境变量 |
| `hermes serve` 不存在 | 命令已变更 | 使用 `hermes dashboard` 替代 |
| 公网无法访问 | 安全组/防火墙未放行 | 检查云厂商安全组和系统 ufw |
| `--insecure` 错误 | 未加此参数 | 公网绑定需加 `--insecure` |
