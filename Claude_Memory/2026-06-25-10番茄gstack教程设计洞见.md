# 会话洞见 — 10番茄gstack教程设计

## 任务背景
使用"五法合一教程设计"技能，为gstack浏览器自动化工具设计10番茄精通教程。

## 核心洞见

### 1. 工具教程的特殊设计挑战
不同于概念型教程，gstack browse是一个CLI工具，教程设计面临"展示 vs 动手"的平衡。解决方案：每个🍅都以一条真实的bash命令开场，让学习者在第一分钟就开始敲命令。

### 2. 抽象层的选择
gstack browse有≈50条命令。教程没有逐一列举，而是按**能力维度**分组（导航/读取/交互/快照/断言），每组选2-3条核心命令深入讲解。金字塔原理在这里起了关键作用——先给结论"你能做什么"，再展开"具体怎么做"。

### 3. QA黄金三角的设计
"baseline → action → diff" 这个模式（🍅4）是系统思考的应用——建立初始状态，执行干预，观察变化。这个模式可以推广到任何测试场景。

### 4. 刻意练习的"从脚本到流水线"路径
三级练习的设计遵循"单条命令 → 多步脚本 → 可复用流水线"的递进逻辑。第10🍅的综合实战其实是一个shell脚本模板，学习者可以直接拿来修改使用。

### 5. 费曼自测的定位
在工具教程中，费曼自测不测试"记忆"(记住命令参数)，而是测试"理解"(为什么要这样设计)。例如"为什么snapshot -D比人工对比更可靠？"——这引导学习者思考工具的设计哲学而非机械记忆。
