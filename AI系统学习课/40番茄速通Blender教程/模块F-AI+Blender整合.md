## \U0001F34531 MCP协议与AI+3D概述（25分钟）

### 费曼输入（概念解释）

**MCP（Model Context Protocol / 模型上下文协议）是什么？**

MCP 是 Anthropic 开源的一个开放协议——让 AI 应用与外部工具和数据源通信。它的设计哲学是"标准化连接"。

> 类比：MCP = "AI 世界的 USB 接口"
>
> 就像 USB 让任何设备都能即插即用，MCP 让任何 AI 应用都能即插即用任何工具。

核心架构有三层角色：

```
┌─────────────┐      MCP协议       ┌──────────────┐      TCP:9876      ┌──────────┐
│  AI客户端    │ ◄──────────────► │  MCP服务器    │ ◄──────────────► │  Blender  │
│ (Claude等)   │                   │ (blender-mcp) │                   │  插件     │
└─────────────┘                   └──────────────┘                   └──────────┘
                                                                          │
                                                                    ┌─────▼─────┐
                                                                    │ bpy API   │
                                                                    │ (Blender  │
                                                                    │  Python)   │
                                                                    └───────────┘
```

完整工作流：Claude AI ↔ MCP 协议 ↔ blender-mcp 服务器 ↔ TCP:9876 ↔ Blender 插件 ↔ bpy ↔ Blender 场景

**AI+Blender 的三种层次：**

| 层次 | 方式 | 说明 | 难度 |
|:---:|:----|:----|:---:|
| 1 | Python 脚本自动化 | 传统方式，手写 bpy 脚本 | 中 |
| 2 | **MCP 协议接入** | AI 直接调用 Blender 命令，本模块核心 | 低 |
| 3 | AI 生 3D 模型 | 文本/图片 → 3D 网格（Hyper3D/Hunyuan3D） | 低 |

**blender-mcp 生态（2026）：**

- **社区版**（ahujasid/blender-mcp）：20000+ ⭐，v1.5.0，基于 socket 通信
- **官方版**（Blender Lab）：2026 年 4 月发布，一键安装，实时 API 文档查询
- **Bonsai Skill Package**：73 个 Blender/BIM/IFC 专用 Claude AI Skills
- **Anthropic 赞助**：Anthropic 成为 Blender 开发基金赞助人

**适用场景 vs 局限性：**

| 场景 | 适用性 | 说明 |
|:----|:-----:|:----|
| 批量操作（材质/灯光/移动） | ✅ 极佳 | 全场景统一改材质、调灯光、移动物体 |
| 快速原型（"创建低多边形森林"） | ✅ 优秀 | 一句话生成场景框架 |
| 复杂有机雕刻 | ❌ 不适用 | AI 不知道细分拓扑怎么走 |
| 精确工程建模 | ❌ 不适用 | 缺少工程约束求解器 |

## \U0001F34532 blender-mcp 安装与配置（25分钟）

### 费曼输入（概念解释）

**社区版 vs 官方版安装对比：**

| 步骤 | 社区版 v1.5.0 | 官方版（Blender Lab） |
|:---|:-------------|:-------------------|
| 1 | `pip install uv` | 从 Claude Desktop connector 下载 |
| 2 | `uv pip install blender-mcp` | 拖拽安装到 Blender |
| 3 | 手动安装 Blender 插件 | 自动完成 |
| 4 | 启用插件 + 启动服务器 | 即装即用 |
| **总步骤** | **4 步** | **2 步** |

**Claude Desktop 配置原理：**

编辑 `claude_desktop_config.json` 文件告诉 Claude Desktop："当你需要操作 Blender 时，通过这个 MCP 服务器来操作"。配置中的 `command` + `args` 定义了如何启动 blender-mcp 服务器进程。

**验证连接的逻辑：**

```
AI 提问："Blender 里现在有什么？"
    ↓
Claude 调用 get_scene_info 工具
    ↓
MCP 协议将请求发送给 blender-mcp
    ↓
blender-mcp 通过 bpy 查询当前场景
    ↓
返回场景信息（物体列表、属性等）
    ↓
Claude 理解结果并用自然语言回答
```

### 刻意练习（具体操作步骤 + 代码/命令）

**步骤 1：安装 uv（Python 包管理器）**

```bash
pip install uv
```

验证安装：
```bash
uv --version
```

**步骤 2：安装 blender-mcp**

```bash
uv pip install blender-mcp
```

**步骤 3：安装 Blender 插件**

1. 打开 Blender → Edit → Preferences → Add-ons
2. 点击右上角 "Install..."
3. 找到并选择 `blender-mcp.zip`
4. 或者手动复制到 Blender 的 addons 目录：
   - Windows: `%APPDATA%\Blender Foundation\Blender\5.1\scripts\addons\`
   - macOS: `~/Library/Application Support/Blender/5.1/scripts/addons/`
   - Linux: `~/.config/blender/5.1/scripts/addons/`

**步骤 4：启用并启动**

1. 在 Add-ons 搜索栏输入 "MCP"
2. 勾选启用 blender-mcp
3. 在 3D View 中按 N 打开 Sidebar
4. 切换到 MCP 选项卡
5. 点击 "Start Server"
6. 确认状态显示 "Running on port 9876"

**步骤 5：配置 Claude Desktop**

找到或创建配置文件：

- Windows: `%APPDATA%\Claude\claude_desktop_config.json`
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Linux: `~/.config/Claude/claude_desktop_config.json`

写入以下内容：

```json
{
  "mcpServers": {
    "blender": {
      "command": "uvx",
      "args": ["blender-mcp"]
    }
  }
}
```

**步骤 6：验证连接**

1. 保持 Blender 打开且 MCP 服务器运行
2. 打开 Claude Desktop
3. 发送消息："Blender 里现在有什么？"
4. 如果配置正确，Claude 会返回当前场景中的物体列表

**常见问题排查：**

| 问题 | 原因 | 解决方案 |
|:----|:----|:--------|
| 连接失败 | 端口 9876 被占用 | 换个端口，或在 MCP 面板修改端口号 |
| uv 找不到 | PATH 没配置 | 将 uv 所在目录加入系统 PATH |
| 插件不显示 | 版本不匹配 | 确认 Blender 5.1 对应 Python 3.13 |
| Claude 无响应 | 配置文件路径错误 | 检查 `claude_desktop_config.json` 位置 |

## \U0001F34533 AI 驱动建模实战（25分钟）

### 费曼输入（概念解释）

**execute_blender_code 工具：**

这是 blender-mcp 最核心的工具。它允许 AI 生成任意 bpy 代码并在 Blender 中执行。本质上，AI 充当了"编程助手 + 执行者"的双重角色。

工作流程：
```
用户用自然语言描述需求
    ↓
Claude 理解需求，生成对应的 bpy 代码
    ↓
通过 MCP 协议发送 execute_blender_code 请求
    ↓
blender-mcp 服务器接收代码
    ↓
在 Blender 的 Python 环境中执行
    ↓
返回执行结果/错误信息
    ↓
Claude 分析结果，可能需要进一步调整
```

**⚠️ 安全警告：**
- execute_blender_code 没有安全检查！AI 生成的代码可能有 bug
- AI 生成的代码可能造成不可逆操作（删除物体、合并网格等）
- **黄金法则：操作前先保存 .blend 文件**

**提示词技巧：**

| 原则 | 好例子 | 差例子 |
|:----|:------|:------|
| 具体 | "添加一个红色金属茶壶，位置(0,0,0)" | "加点东西" |
| 分步 | "先建桌子 → 再放椅子 → 最后加灯光" | "帮我做个办公室" |
| 明确风格 | "低多边形风格，每个面不超过 20 个多边形" | "做好看一点" |

## \U0001F34534 AI 辅助材质/灯光/渲染（25分钟）

**材质系统基础回顾：**

这个工具让 AI 能"看"到当前视口的内容——AI 获取截图后可以基于视觉反馈进行调整，形成闭环：

```
AI 修改材质参数 → 截图 → AI 分析效果 → 再次修改 → 再截图 → ...
```

这就是 **AI 驱动的循环反馈工作流**。

## \U0001F34535 Hyper3D Rodin 与 AI 生 3D 模型（25分钟）

### 费曼输入（概念解释）

**Hyper3D Rodin：文本/图片 → 3D 模型**

Hyper3D Rodin 是集成在 blender-mcp v1.5.0 中的 AI 生 3D 工具。你只需要输入一段文字描述，它就能自动生成对应的 3D 网格，并支持纹理自动生成。

**支持的 AI 生 3D 工具：**

| 工具 | 类型 | 输入 | 集成方式 | 质量 |
|:----|:----|:----|:--------|:----|
| Hyper3D Rodin | 文本/图片 → 3D | 文字描述或图片 | blender-mcp 内置 | ★★★☆☆ |
| Hunyuan3D | 图片 → 3D | 单张图片 | blender-mcp v1.4.0+ | ★★★☆☆ |
| Sketchfab 搜索 | 搜索 → 下载 | 关键词 | 直接集成 | ★★★★★（已有模型） |
| Poly Haven | 一键导入 | HDRI/贴图/模型 | 直接集成 | ★★★★★ |

**AI + Blender 全流程工作流：**

```
第1步 🎨 概念设计：AI 生成参考图（Midjourney / DALL·E / Stable Diffusion）
第2步 🏗 模型生成：Hyper3D Rodin 根据描述生成初步 3D 模型
第3步 🔧 手动优化：在 Blender 中调整拓扑、增加细节
第4步 🎭 材质+灯光：AI 辅助批量设置材质和灯光
第5步 📸 渲染输出：AI 设置渲染参数并输出
第6步 📤 导出发布：AI 批量导出多种格式（FBX/OBJ/GLB）
```

**当前局限性（2026）：**

- AI 生成的面数多但拓扑乱——不适合直接用于生产
- 需要手动再拓扑才能用于游戏/影视资产管线
- 最适合概念设计和快速原型阶段
