### 二、贝叶斯原理（逆向概率）

**核心思想**：在没有掌握全部客观事实的情况下，先估计一个值（先验概率），再根据新证据更新判断（后验概率）。

**贝叶斯公式**：
$$P(C_i|A) = \frac{P(A|C_i)P(C_i)}{\sum_j P(A|C_j)P(C_j)}$$

**关键术语**：
- **先验概率**：通过经验判断事件发生的概率
- **后验概率**：发生结果之后，推测原因的概率
- **似然函数**：衡量模型参数与数据匹配程度的函数
- **条件概率**：P(A|B) 表示 B 已发生前提下 A 发生的概率

**经典案例——"贝叶死"**：
- 发病率 1/10000，检测准确率 99.9%，误报率 0.1%
- 查出阳性后实际患病概率 ≈ 9%（而非99.9%！）
- 原因：10000人中1人患病且大概率检出（≈1人）+ 9999人中0.1%误报（≈10人）→ 阳性者共约11人，其中真正患病的只有1人

### 三、朴素贝叶斯分类器

**假设**：每个输入变量相互独立（"朴素"的由来）

**两种概率**：
1. 类别概率 P(Cj)
2. 条件概率 P(Ai|Cj)

**训练三步**：
1. 给出训练数据及对应分类
2. 计算类别概率和条件概率
3. 使用贝叶斯原理对新数据预测

**离散值分类示例**：根据身高(高/中/矮)、体重(重/中/轻)、鞋码(大/中/小)判断性别

**连续值分类**：假设服从正态分布 → 计算均值和方差 → 用密度函数求概率值

**Sklearn工具**：
| 类型 | 类 | 适用场景 |
|------|-----|---------|
| 高斯朴素贝叶斯 | GaussianNB | 连续变量（身高、长度） |
| 多项式朴素贝叶斯 | MultinomialNB | 离散变量（单词出现次数） |
| 伯努利朴素贝叶斯 | BernoulliNB | 布尔变量（单词是否出现） |

**应用场景**：文本分类、文本过滤、情感预测、推荐系统

### 四、决策树与随机森林

**决策树**：把经验总结出来，关键问题是选择哪个属性作为根节点

**常用算法**：C4.5、ID3、CART

**信息熵**：衡量不确定性，熵越大→不确定性越大
- 信息增益 = 原有熵 H(D) - 分裂后熵 H(D|A)
- 信息增益越大→该特征选择性越好

**随机森林**：
- bagging思想：多个弱分类器投票组成强分类器
- 每棵树独立生长、无剪枝
- 优点：不易过拟合，减少预测方差
- 关键参数：m（特征选择个数）— m越大，树的相关性和分类能力提升

**Sklearn工具**：
```python
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
DecisionTreeClassifier(criterion='entropy')
RandomForestClassifier(n_estimators=10, criterion='gini')
```

### 五、SVM 支持向量机

**核心思想**：将低维线性不可分问题映射到高维空间→变为线性可分→找到分离超平面

**三种方法**：SVC、NuSVC、LinearSVC（均基于 libsvm）

**关键参数**：
- kernel：RBF(默认)、Linear、Poly、Sigmoid
- C：惩罚系数（越大惩罚越大）
- gamma：核函数系数
- degree：多项式核的最高次数

**核选择技巧**：
- 样本数 < 特征数：用线性核 or 先降维再用非线性核
- 样本数 ≥ 特征数：可用非线性核映射到更高维

### 六、逻辑回归

**假设条件**：
1. 数据服从伯努利分布
2. 正类的概率由 sigmoid 函数计算

**损失函数**：基于最大似然估计推导，用梯度下降求解

**决策边界**：默认阈值 0.5，判断正类概率是否大于 0.5

**优点**：
- 形式简单，可解释性好
- 工程上的 baseline
- 训练速度快，资源占用小
- 方便输出概率分数调整阈值

**缺点**：
- 准确率不高，难拟合真实分布
- 难处理样本不均衡
- 难处理非线性数据（不引入其他方法时）

**为什么去掉高度相关的特征？** ① 可解释性更好 ② 提高训练速度

### 七、二手车价格预测项目要点

**评价标准**：MAE（Mean Absolute Error）— 越小越准确

**建模流程**：数据加载 → 数据探索 → 特征选择 → XGBoost模型训练 → 预测

**提分思路**：
1. 处理时间类型数据（regDate → year/month/day + 日期diff）
2. 模型融合（XGBoost + LightGBM）
3. 增加品牌统计特征（brand_amount, brand_price_mean等）
4. 调整学习率（0.1 → 0.01）
5. 增加 n_estimators

#### 三、问答题（参考答案要点）

**1. 贝叶斯原理与"贝叶死"案例**
- 贝叶斯原理 = 先验概率 × 似然函数 / 证据因子
- 核心：用新证据更新对事件的判断
- "贝叶死"案例：检测准确率99.9%针对的是"患病者检出阳性"的概率（P(阳性|患病)=99.9%），而我们关心的是"阳性者患病"的概率（P(患病|阳性)）
- 发病率极低（万分之一），即使检测准确率高，也会因大量健康人群的误报（假阳性）导致阳性者中真正患病比例大幅降低
- 计算：P(患病|阳性) = (0.01%×99.9%) / (0.01%×99.9% + 99.99%×0.1%) ≈ 9%

**2. 二手车价格预测提分思路**
- **时间特征处理**：将 regDate/creatDate 拆分为 year/month/day，并增加基于基准日期的日期diff。原因：时间特征蕴含车辆使用时长信息，对价格有直接影响
- **模型融合**：XGBoost + LightGBM。原因：不同模型有不同偏置和优势，融合可降低方差和偏差
- **品牌统计特征**：对 brand 做 groupby 统计（brand_amount, brand_price_mean/median/std等）。原因：品牌是关键价格影响因素，统计特征可提取品牌整体价格水平信息
- **调整学习率**：learning_rate 从 0.1 降至 0.01，同时增加 n_estimators。原因：小学习率配合更多迭代可以让模型更精细地收敛
- **特征工程文档化**：将特征工程思路总结到文档中，便于系统化迭代优化
